## 1. 랜딩페이지 구조 변경 요약

1. **메인 Hero 영역 리뉴얼**
   - 첫 화면 카피를 핵심 콘셉트인 **"함께 성장하는 AI 소규모 크루 작업실"**로 전면 개편하여, 첫 방문자가 서비스의 핵심 가치를 단번에 인지할 수 있도록 하였습니다.
   - 설명글(lead)을 핵심만 담은 짧은 문장("마음이 맞는 동료들과 소규모 크루를 이루어 AI 학습과 프로젝트를 끝까지 완주하는 최적의 협업 공간입니다.")으로 개선하여 가독성을 높였습니다.
   - 주요 CTA 버튼의 문구를 요구사항에 맞춰 **"인기 크루 둘러보기"** 및 **"크루 만들기"**로 정돈했습니다.

2. **인기 크루 전면 배치 및 다변화**
   - 기존의 '진행 중인 크루' 섹션을 랜딩 본문의 최우선 핵심 콘텐츠인 **'현재 인기 크루'** 섹션으로 개편하였습니다.
   - 인기 크루 카드를 총 6개(스터디, 콘텐츠 제작, 웹빌드/코딩, 자동화/부업, 디자인, 엔지니어링)로 세밀하게 확장하여, 다양한 AI 활용 목적에 맞춘 크루들을 한눈에 탐색하고 참여할 수 있도록 설계했습니다.
   - 데스크톱 화면에서 6개 카드가 3열 레이아웃(`grid-template-columns: repeat(3, 1fr)`)으로 최적화되어 보일 수 있도록 그리드 설정을 변경하였으며, 태블릿 2열, 모바일 1열 반응형 레이아웃을 빈틈없이 구현했습니다.

3. **마지막 CTA 영역**
   - "나에게 맞는 크루 찾기"와 "새 크루 만들기"로 마지막 동선을 단순화하고 직관적으로 개편했습니다.

## 2. 인기 크루 선정 기준 설계 및 UI 반영

단순 조회수 기반 랭킹의 한계를 극복하기 위해, **사용자 행동 및 신뢰성 지표를 복합적으로 결합**한 6대 인기 크루 선정 기준을 마련하고, 이를 직관적인 UI 배지(Badge)와 데이터 영역으로 시각화했습니다.

| 인기 선정 기준 | UI 배지 및 표현 방식 | 구체적 매칭 조건 및 데이터 설계 |
| :--- | :--- | :--- |
| **최근 7일 참여 신청 수** | `🔥 신청 급상승` (`badge-trend`) | 최근 7일 내의 가입 신청이 급격히 증가한 크루 (예: 최근 7일 신청 12건) |
| **모집 마감 임박 여부** | `⏰ 마감 임박` (`badge-trend`) | 정원 대비 공석이 1석 이하여서 조기 마감이 확실시되는 크루 (예: 남은 자리 1석) |
| **완주율 및 활동 지속률** | `🏆 완주율 높음` (`badge-success`) | 이전 기수의 수료/완주 비율이 압도적으로 높은 검증된 크루 (예: 이전 완주율 95%) |
| **크루장 신뢰도/평점** | `⭐ 크루장 인증` (`badge-success`) | 누적 활동 및 이력이 검증된 우수 크루장 파트너십 (예: 크루장 신뢰 평점 4.9/5.0) |
| **최근 활동 빈도** | `⚡ 최근 활동 활발` (`badge-info`) | 크루 내부 공유 파일, 노트 작성, 실시간 대화 등의 실시간 활성 지수 (예: 오늘 업로드 에셋 24건) |
| **누적 찜/저장 수** | `⭐ 저장 많은 크루` (`badge-warn`) | 예비 지원자들의 위시리스트 누적 북마크 수 (예: 누적 관심/저장 120회 이상) |

- **UI 스타일링**: 다크 모드 톤에 어울리는 은은한 불투명 배경색(Pill 배경)과 뚜렷한 식별 색상을 배지 스타일로 정의하여 눈의 피로를 낮추고 직관성을 극대화했습니다.

## 3. 서비스 소개 / 요금 플랜 이동 방식

기획서 및 부가 설명 성격을 지닌 본문 요소들을 분리하여 랜딩 본문을 가볍게 유지하되, 사용자가 언제든 탐색할 수 있도록 **싱글 페이지 웹앱(SPA) 스타일의 탭 브라우징 환경**을 구축했습니다.

- **상단 내비게이션 바 구성**: `홈`, `서비스 소개`, `인기 크루`, `요금/플랜` 탭 메뉴로 재정리하였으며, 활성화된 메뉴에는 투명도 있는 테두리와 액티브 컬러(`var(--accent)`)를 주어 직관적인 피드백을 제공합니다.
- **탭 분할**:
  - **홈 탭**: Hero 섹션 + 인기 크루 섹션 + 신뢰 정책 섹션 + 마지막 CTA 섹션
  - **서비스 소개 탭**: 기존 문제 제기('왜 CrewUp인가요?') + '사용 흐름' 섹션을 간결하게 구성
  - **요금/플랜 탭**: 기존 3단계 요금제('Free / Plus / Pro') 정보 배치
- **동적 탭 전환 및 스무스 스크롤 연동**:
  - JavaScript로 탭 전환(`.tab-section.active`)과 CSS 큐빅 베지어 트랜지션을 연동하여, 메뉴 클릭 시 탭 내용이 부드럽게 나타나도록(`fadeInSlideUp` 애니메이션) 처리했습니다.
  - 사용자가 다른 탭에 머물러 있는 상태에서 상단 메뉴의 **"인기 크루"**나 우측의 **"시작하기"** 버튼, 또는 모바일 고정 CTA 버튼을 누를 시, 자동으로 **홈 탭으로 전환한 후 해당 섹션(`id="crews"`, `id="join"`)으로 자연스럽게 스무스 스크롤**을 실행합니다.
  - 모바일용 하단 고정 CTA바 역시 탭 전환 로직과 매끄럽게 연결됩니다.

## 4. 에러 및 특이사항

- **기존 데모의 격리**: 모바일 폰 미리보기 화면 내의 내부 탭 기능(`.phone .tab`)과 전체 페이지용 탭 전환 기능(`[data-tab]`)이 충돌하지 않도록 이벤트 핸들러 선택자에서 정확히 분리 및 격리 처리하였습니다.
- **오동작 최소화**: 로고를 클릭하면 즉시 메인 홈 탭으로 복귀하고 화면 맨 위로 이동하게 설계하여 내비게이션 완성도를 높였습니다.
