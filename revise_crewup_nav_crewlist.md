# [Context]
CrewUp AI 홈페이지 시안(`/workspace/MA/crewup_web_prototype.html`)은 현재 홈/서비스 소개/인기 크루/요금 플랜 탭 구조로 수정되어 있다.
사용자가 추가 피드백을 주었다.

중요 피드백:
- 상단 탭 이름이 "인기 크루"이면 안 된다.
- 탭에는 "크루 목록"이 있어야 한다.
- 랜딩페이지 홈에는 "현재 인기 크루"를 보여주는 것은 맞지만, 내비게이션/탭 개념은 전체 크루 탐색으로 보여야 한다.

즉:
- 홈 화면 섹션명: "현재 인기 크루" 유지 가능
- 상단 탭/메뉴명: "인기 크루" → "크루 목록"으로 변경
- 사용자가 탭을 눌렀을 때는 인기 크루만 보는 느낌이 아니라 전체 크루 목록/탐색 페이지로 들어가는 느낌이어야 함

## [New Task]

### 1. 상단 내비게이션 수정
현재 상단 메뉴에 `인기 크루`가 있다면 `크루 목록`으로 변경한다.

권장 메뉴 구조:
- 홈
- 서비스 소개
- 크루 목록
- 요금/플랜
- 시작하기

### 2. 탭/섹션 의미 정리
- 홈 탭 안에는 Hero + 현재 인기 크루 일부를 보여줘도 된다.
- 하지만 `크루 목록` 탭/메뉴는 전체 크루를 탐색하는 의미로 보여야 한다.
- 클릭 시 현재 인기 크루 섹션으로 스크롤하더라도, 화면 문맥은 "크루 목록"이어야 한다.
- 가능하면 섹션 제목을 다음처럼 정리한다:
  - 큰 제목: "크루 목록"
  - 서브 영역/배지: "현재 인기 크루"
  - 설명: "최근 신청, 활동성, 완주율 등을 기준으로 주목받는 크루를 먼저 보여드립니다."

### 3. 인기 크루 개념은 하위 큐레이션으로 표현
"인기 크루"는 탭 이름이 아니라 크루 목록 안의 큐레이션 기준이어야 한다.
예:
- 크루 목록 상단 필터/칩:
  - 전체
  - 인기순
  - 마감 임박
  - 입문 추천
  - 프로젝트형
- 현재 화면은 그중 "인기순"이 선택된 상태처럼 표현 가능

### 4. CTA/문구 정리
- Hero 버튼 "인기 크루 둘러보기"는 "크루 목록 보기"로 바꾼다.
- 마지막 CTA도 필요하면 "나에게 맞는 크루 찾기" 유지 가능.
- 문구 전체에서 탭 이름으로 쓰인 "인기 크루"는 "크루 목록" 또는 "크루 탐색"으로 바꾼다.
- 단, 섹션 내부 배지나 큐레이션 문구로 "현재 인기 크루"는 유지 가능.
