# [Context]
Crew Up 홈페이지 시안(`/workspace/MA/crewup_web_prototype.html`)에는 Hero 영역 오른쪽에 스마트폰 화면 미리보기가 있다.
현재 그 스마트폰 안의 예시 내용이 사용자가 이전에 했던 특정 프로젝트(예: G304 제품 캡처, 숏폼 후킹, PC방 장면, 가격대별 후보 등)처럼 보인다.

사용자 피드백:
- 랜딩페이지 스마트폰 화면 내용이 사용자의 실제/이전 프로젝트처럼 보이면 안 된다.
- 특정 개인 프로젝트, 쿠팡/제품/광고 숏폼 작업처럼 보이는 내용은 제거해야 한다.
- 더 범용적인 Crew Up 서비스 예시로 바꿔야 한다.
- 단, 실제 구현 시 이 역시 더미데이터/placeholder임을 구분해야 한다.

참고 가드 문서:
- `/workspace/MA/crewup_dummy_data_guardrail.md`

## [New Task]
`/workspace/MA/crewup_web_prototype.html`의 Hero 스마트폰 미리보기 내용을 범용적인 Crew Up 예시로 수정하라.

### 1. 제거해야 할 특정 프로젝트성 표현
스마트폰 목업 및 관련 JS/HTML에서 아래처럼 특정 개인 프로젝트나 쿠팡/광고 작업처럼 보이는 표현을 제거한다.

- G304
- 제품 캡처
- 상세페이지
- 숏폼 후킹
- PC방 장면
- 가격대별 후보
- 광고 숏폼 3개
- 블로그 랜딩
- 제품 후보 조사
- 이미지 프롬프트 테스트
- 특정 상품/마케팅 캠페인처럼 보이는 내용

### 2. 범용적인 스마트폰 목업 내용으로 교체
스마트폰 화면은 Crew Up의 일반적인 크루 활동 예시처럼 보여야 한다.

권장 방향:
- 크루명: `AI 프로젝트 성장 크루` 또는 `AI 업무 자동화 크루` 같은 범용 이름
- 목표: `이번 주 목표 정리와 역할 분담을 완료해요`처럼 일반적인 협업 목표
- 공유 자료 예시:
  - `온보딩 가이드`
  - `이번 주 체크리스트`
  - `참고 자료 모음`
  - `회의 기록`
- 진행 상태 예시:
  - `공유 자료`
  - `이번 주 할 일`
  - `참여 멤버`
- 탭 예시:
  - `홈`
  - `자료`
  - `노트`

### 3. 더미데이터 표시 원칙
시안 화면 안에서 너무 노골적으로 `더미데이터`라고 쓰지는 않아도 되지만, 코드 주석 또는 완료 보고서에는 명확히 적는다.
- 스마트폰 목업 내용은 서비스 이해를 위한 범용 placeholder이다.
- 실제 구현 시 DB/API/CMS/관리자 입력 데이터로 교체되어야 한다.
- 하드코딩된 예시를 실제 서비스 콘텐츠로 사용하면 안 된다.

### 4. 브랜드/표현 유지
