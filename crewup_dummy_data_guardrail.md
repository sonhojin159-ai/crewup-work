# Crew Up 더미데이터/예시 콘텐츠 사용 방지 가이드

## 목적
Crew Up 홈페이지/서비스 구현 과정에서 기획용 예시 문구나 더미데이터가 실제 웹페이지 콘텐츠로 그대로 들어가는 문제를 방지한다.

## 핵심 원칙
- 예시 크루, 예시 유저, 예시 가격, 예시 통계는 실제 콘텐츠가 아니다.
- 시안에서 쓰는 예시 데이터는 반드시 `placeholder`, `sample`, `mock`, `예시`로 명확히 구분한다.
- 실제 구현 단계에서는 예시 데이터를 그대로 노출하지 않는다.
- 실제 데이터가 없으면 아래 중 하나로 처리한다.
  1. 빈 상태 UI 표시
  2. “아직 등록된 크루가 없습니다” 문구 표시
  3. CMS/DB/API 연결 전용 자리로 남김
  4. 관리자 입력 후 노출되도록 설계

## 금지 사례
아래 같은 내용이 실제 홈페이지에 그대로 들어가면 안 된다.

- `AI 숏폼 & 릴스 제작 크루`
- `AI 부업 실전 크루`
- `노코드 & AI 웹페이지 제작 크루`
- `프롬프트 & AI 도구 입문 크루`
- 예시 크루장 이름
- 예시 신청 수/완주율/저장 수
- 예시 인기 배지
- 예시 가격/요금제 수치

위 항목은 현재 시안 이해를 돕기 위한 샘플일 수는 있지만, 실제 서비스 콘텐츠로 확정된 것이 아니다.

## 에이전트 작업 지시 시 필수 문구
Crew Up 관련 구현 지시서에는 아래 문구를 반드시 포함한다.

```md
주의: 현재 HTML/기획서에 들어간 크루명, 크루장명, 신청 수, 완주율, 저장 수, 인기 배지, 요금제 수치 등은 모두 시안용 더미데이터일 수 있다. 실제 구현 시 이를 실제 서비스 콘텐츠처럼 하드코딩하지 말 것. 실제 데이터 소스가 없으면 빈 상태 UI, 관리자 입력 영역, DB/API 연동 자리, 또는 명확한 placeholder 상태로 처리할 것.
```

## 구현 전 체크리스트
실제 개발/배포 전 반드시 확인한다.

- [ ] 화면에 보이는 크루 목록이 실제 DB/API/CMS에서 오는가?
- [ ] 하드코딩된 예시 크루명이 남아 있지 않은가?
- [ ] 예시 크루장/멤버/통계가 실제 데이터처럼 보이지 않는가?
- [ ] 인기 크루 선정 기준은 실제 계산 로직 또는 관리자 선정 로직으로 분리되어 있는가?
- [ ] 데이터가 없을 때 빈 상태 UI가 준비되어 있는가?
- [ ] 더미데이터가 필요하면 `sample`, `mock`, `placeholder`로 명확히 표시되어 있는가?

## 향후 지시서 작성 규칙
Hermes가 Crew Up 작업 지시서를 만들 때는 다음을 구분해야 한다.

### 시안/프로토타입 단계
- 예시 데이터 사용 가능
- 단, 반드시 더미데이터임을 보고서와 코드 주석에 명시
- 실제 콘텐츠로 오해될 수 있는 표현은 피함

### 실제 구현 단계
- 예시 데이터 하드코딩 금지
- DB/API/CMS 연결 또는 빈 상태 UI 우선
- 운영자가 등록한 크루만 노출
- 인기 기준은 실제 데이터 기반 또는 관리자 큐레이션 기반으로 분리
