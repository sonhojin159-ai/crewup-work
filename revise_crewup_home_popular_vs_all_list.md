# [Context]
Crew Up 홈페이지 시안(`/workspace/MA/crewup_web_prototype.html`)은 현재 홈/서비스 소개/크루 목록/요금 플랜 탭 구조다.
사용자가 중요한 정보구조 피드백을 주었다.

핵심 피드백:
- 홈에서는 “인기 크루”만 일부 보여줘야 한다.
- 상단 탭의 “크루 목록”에서는 전체 크루 목록이 다 보여야 한다.
- 현재처럼 홈의 인기 크루 섹션과 크루 목록 탭/메뉴가 같은 섹션으로 연결되면 안 된다.
- “인기 크루”와 “전체 크루 목록”은 화면/섹션/탭 의미가 분리되어야 한다.

더미데이터 주의:
- 현재 크루 카드들은 시안용 placeholder이다.
- 실제 구현 시 예시 크루명/통계/배지는 DB/API/CMS/관리자 입력 데이터로 대체해야 한다.
- `/workspace/MA/crewup_dummy_data_guardrail.md` 원칙을 따른다.

## [New Task]

### 1. 홈 탭: 인기 크루 섹션
홈 탭에는 Hero 아래에 “현재 인기 크루” 섹션을 둔다.
- 목적: 첫 방문자가 바로 흥미를 느끼게 하는 큐레이션 영역
- 카드 수: 3~4개 정도만 노출
- 제목 예시: `현재 인기 크루`
- 설명 예시: `최근 신청과 활동성이 높은 크루를 먼저 보여드립니다.`
- CTA: `전체 크루 목록 보기`
  - 클릭 시 크루 목록 탭으로 이동해야 한다.

### 2. 크루 목록 탭: 전체 크루 목록
상단 메뉴의 `크루 목록`은 별도 탭/섹션이어야 한다.
- `tab-content-crews` 같은 별도 탭 컨테이너를 만든다.
- 전체 크루 목록이 이 탭에 표시되어야 한다.
- 카테고리 필터는 이 전체 크루 목록 탭 안에 둔다.
- 카테고리:
  - 전체
  - 콘텐츠 제작
  - 부업
  - 웹페이지 제작
  - 자동화
  - AI 학습/스터디

### 3. 내비게이션 동작 수정
상단 메뉴 구조:
- 홈
- 서비스 소개
- 크루 목록
- 요금/플랜
- 시작하기

동작:
- 홈 클릭 → Hero + 현재 인기 크루 섹션이 있는 홈 탭
- 크루 목록 클릭 → 전체 크루 목록 탭
- 홈의 `전체 크루 목록 보기` CTA → 크루 목록 탭으로 이동
- 시작하기/크루 만들기 CTA → 홈의 신청/CTA 섹션 또는 적절한 시작 위치로 이동

### 4. 문구 구분
- 홈 섹션에서는 `현재 인기 크루`
- 크루 목록 탭에서는 `전체 크루 목록` 또는 `크루 목록`
- 인기 배지는 카드 내부 하위 정보로 유지 가능
- 크루 목록 탭 제목이 `현재 인기 크루`처럼 보이면 안 된다.
