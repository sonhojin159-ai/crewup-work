- 개선 권고:
  1. **만능 동료 밴 적용 시 동일 카드 중복 처리 검토**:
     - 현재 `apply_bans_and_reselect` 함수는 개별 카드 인스턴스를 기준으로 밴을 적용합니다. 예를 들어, 카드풀에 `D` 카드가 2장 남아있고 마왕이 `[A, B, D]`를 선택한 상태에서 용사가 `D` 1장을 밴하면, 마왕의 선택에서 `D` 1장이 제거되어 `[A, B]`가 남고, 남은 1장의 대체 카드를 카드풀 `[B, D]`에서 무작위로 선택하게 됩니다. 이 과정에서 70% 확률의 마왕 회피 경향이 동작하더라도, 재선택 결과로 다시 `D`가 뽑힐 수 있습니다.
     - 용사가 특정 시련 카드를 밴했을 때, 해당 라운드에서 해당 카드 **타입 자체**가 등장하지 못하게 막는 것이 기획 의도라면, `apply_bans_and_reselect`에서 밴된 카드 타입(`bans` 목록에 있는 카드들)의 모든 인스턴스를 임시 카드풀(`temp_cards`) 및 `kept`에서 완전히 배제하도록 코드를 보완해야 합니다.
  2. **터미널 출력 한글 깨짐 호환성 대응**:
     - `CodexResult.py`는 UTF-8로 정상 작성되었으나, Windows 터미널(기본 CP949/MS949) 환경에서 오케스트레이터(`tracker.py`) 등을 통해 비대화형 실행을 하거나 일반 콘솔에서 실행 시 출력 한글이 깨질 수 있습니다. 향후 사용자 경험을 개선하기 위해 실행 진입점(`Game/prototype.py`) 또는 시스템 레벨에서 터미널 출력 인코딩을 UTF-8로 강제 변환하는 코드(`import sys; sys.stdout.reconfigure(encoding='utf-8')`)를 추가하는 것을 권장합니다.
