# [Context]
CrewUp AI는 AI 스터디/프로젝트/크루 모집을 위한 모바일 웹 중심 서비스다.
현재 `/workspace/MA/crewup_web_prototype.html`은 홈페이지 시안으로 만들어졌지만, 사용자 피드백상 기획서 문장과 내부 설명이 과도하게 노출되어 실제 이용자용 랜딩페이지로 적합하지 않다.

중요한 방향 전환:
- 홈페이지는 내부 기획서 요약본이 아니다.
- 방문자가 처음 봤을 때 바로 이해하고 행동할 수 있는 공개 랜딩페이지여야 한다.
- “3개의 실험”, “맥락을 쌓는다”, “컨테이너”, “프로토타입”처럼 내부 기획/운영진 관점의 표현은 제거하거나 이용자 언어로 바꾼다.
- 서비스 설명은 짧고 직관적으로 한다.

## [New Task]
`/workspace/MA/crewup_web_prototype.html`을 실제 이용자에게 보여줄 홈페이지 시안으로 전면 수정하라.

### 수정 목표
기획서 느낌을 제거하고, 방문자 기준의 랜딩페이지로 바꾼다.

### 반드시 제거/축소할 것
- “먼저 3개의 실험”, “실험”, “프로토타입”, “컨테이너”, “맥락” 등 내부 기획 용어
- 서비스 운영 구조를 길게 설명하는 문장
- 크루 내부 기능을 과도하게 보여주는 대시보드식 설명
- 기획서 문장처럼 보이는 긴 설명문
- 방문자가 당장 알 필요 없는 운영/확장 계획

### 홈페이지에 필요한 핵심 내용만 남길 것
1. 첫 화면 Hero
   - CrewUp이 무엇인지 한 문장으로 설명
   - 예: “AI를 같이 배울 사람과 프로젝트 크루를 찾는 곳”
   - CTA: “크루 둘러보기”, “크루 만들기”

2. 이용자 관점 문제 제기
   - 혼자 공부하면 쉽게 끊긴다
   - 같이 할 사람을 찾기 어렵다
   - 오픈채팅은 금방 흐트러진다
   - 단, 문장은 짧고 이용자 언어로 작성

3. 핵심 사용 흐름
   - 크루 찾기
   - 크루장/소개 확인
   - 참여 신청
   - 함께 학습/프로젝트 진행

4. 크루 예시 섹션
   - AI 입문 스터디
   - 숏폼/콘텐츠 제작 크루
   - 웹서비스 만들기 크루
   - 부업/자동화 실험 크루
   단, “실험”이라는 표현이 부자연스러우면 “자동화 프로젝트 크루”처럼 순화

5. 신뢰 요소
   - 크루장 소개
   - 진행 방식
   - 모집 인원
   - 난이도/목표 기간

6. 마지막 CTA
   - “나에게 맞는 크루 찾기”
   - “새 크루 만들기”

### 문체 가이드
- 기획서 문체 금지
- 내부 회의자료 같은 설명 금지
- 이용자에게 직접 말하는 짧은 카피 사용
- 모바일에서 한눈에 읽히는 구조
- 문장 길이는 짧게
- 기능 설명보다 “내가 여기서 뭘 할 수 있는지”를 우선

### 디자인 가이드
- 기존 어두운 톤과 현대적인 분위기는 유지 가능
- 하지만 정보량을 줄이고 여백을 늘릴 것
- 홈페이지답게 Hero → 문제 → 사용 흐름 → 크루 예시 → CTA 흐름으로 단순화
- 내부 앱 대시보드처럼 보이는 복잡한 카드/지표는 줄일 것
