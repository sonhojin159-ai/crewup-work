# User Operating Profile

이 노트는 Obsidian을 사용자의 맞춤형 작업 위키로 운영하기 위한 기준 노트입니다.

## 핵심 목적

Obsidian의 가장 중요한 역할은 단순 프로젝트 기록이 아니라, 사용자를 점점 더 잘 파악해 매 작업 결과를 개인화하는 것입니다.

Hermes와 에이전트는 프로젝트를 진행하면서 다음 내용을 구조화해 저장하고, 이후 작업에서 참고합니다.

- 사용자의 행동 패턴
- 사용자가 자주 확인하는 항목
- 사용자의 작업 방향성
- 사용자가 선호하는 디자인 방향
- 사용자가 선호하는 UI/UX 요소
- 사용자가 반복해서 지적하는 문제
- 사용자가 중요하게 보는 검수 기준
- 프로젝트를 넘나들며 재사용할 수 있는 취향/판단 기준

## 운영 원칙

- 모든 대화를 원문 그대로 저장하지 않는다.
- 사용자를 이해하는 데 도움이 되는 반복 패턴과 선호를 짧고 구조적으로 저장한다.
- 단일 프로젝트에만 해당하는 내용은 해당 프로젝트 폴더에 저장한다.
- 여러 프로젝트에 걸쳐 반복되는 사용자의 취향/작업 방식은 이 노트 또는 `01_User` 폴더에 저장한다.
- 다음 작업을 시작할 때, 필요한 경우 이 노트와 관련 프로젝트 노트를 먼저 참고해 사용자의 선호에 맞춘 결과를 만든다.

## 현재 확인된 사용자 선호

- 사용자는 작업 내용을 쉽게 잊을 수 있으므로, 중요한 결정과 액션 아이템은 Hermes가 먼저 Obsidian에 저장해주길 원한다.
- 사용자는 Obsidian을 CrewUp뿐 아니라 모든 프로젝트의 개인/프로젝트 위키로 쓰길 원한다.
- 사용자는 다음 프로젝트에서도 이전 프로젝트의 결정, 패턴, 체크리스트, 디자인 선호를 참고하는 방식을 원한다.
- 사용자는 Obsidian이 자신을 파악해서 매 작업마다 점점 더 맞춤형 결과를 내는 시스템이 되길 원한다.
