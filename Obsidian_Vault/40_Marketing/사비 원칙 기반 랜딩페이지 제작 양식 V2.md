## 이번 버전의 핵심

사비 원칙의 17단계를 고객에게 그대로 노출하지 않고, 소상공인이 채우기 쉬운 6개 입력 묶음으로 압축했다.

1. Hero / Hook: 대상 고객, 큰 약속, 지역/신뢰 한 줄
2. Problem: 고객이 문의 전 막히는 이유, 기존 방식의 실패
3. Mechanism: 해결 방식, 진행 순서, 차별점
4. Proof: 후기, 사진, 수치, 위치, 자격
5. Offer: 가격, 보너스, 한정 조건, 리스크 제거
6. Action: 문의 버튼, 무엇을 보내야 하는지, 마지막 P.S.

## 구현된 데모 기능

- 좌측 입력 패널
  - 업종 선택: 네일/뷰티, 청소/방문서비스, PT/필라테스, 클리닉/상담
  - 매장명, 지역/신뢰 한 줄, 대상 고객, 헤드라인, 설명문, 이미지 URL
  - 문제/기존 실패, 해결 방식/진행 순서
  - 증거/후기/이미지 갤러리
  - 오퍼명, 가격, 한정 조건, 보너스, 보장, CTA, P.S.
- 중앙 내부 설계 블록
  - 17단계 → 6개 전환 블록으로 압축한 구조 표시
  - 필요 시 추가하는 선택 블록: 가격표/메뉴판, FAQ/반박 제거, 위치/운영시간
- 우측 모바일 미리보기
  - 입력값 변경 즉시 반영
  - 고객 화면에는 내부 용어를 노출하지 않고 자연스러운 모바일 안내페이지로 표시
  - 안내형 / 이벤트 포스터형 / 후기·사례형 스타일 버튼
  - 워터마크 `CREWUP SAMPLE`
  - 정규화 JSON 표시 및 복사
  - JSON 저장 / HTML 다운로드 버튼

## 구조 결정

- 내부 프레임워크명: `sabri_style_direct_response_compressed`
- 기본 블록: `hero`, `problem`, `mechanism`, `proof`, `offer`, `action`
- 추가 블록: `priceList`, `faq`, `location`
- 고객 판매 표현은 “사비 17단계”가 아니라 아래처럼 바꿔 말한다.
  - 사진과 가격표만 주시면 인스타 프로필용 모바일 안내페이지를 제작합니다.
  - 문의 전에 고객이 궁금해하는 가격·혜택·예약 방법을 한 화면으로 정리합니다.
  - 완성형 대행이 아니라, 입력한 정보가 바로 모바일 페이지로 바뀌는 저가형 안내페이지 양식입니다.
