# 사비 기준 최종 모바일 편집 사이트 수정본

날짜: 2026-06-12

## 수정 이유

직전 `final-mobile-builder-site`는 편집 사이트처럼 보이도록 정리했지만, 오른쪽 모바일 미리보기와 예시 자료의 본문이 너무 축약되어 있었다. 사용자가 지적한 핵심 문제는 다음과 같다.

- 미리보기 자료가 사비/Sabri 기준의 직판 흐름을 충분히 반영하지 못함.
- Problem, Mechanism, Proof, Offer, Risk reversal, P.S. 등의 세일즈 블록이 줄어들어 단순 안내페이지처럼 보임.
- 양식 선택은 있었지만, 예시 모바일 페이지 안에 17단계 기반의 전환 구조가 명확히 남아있지 않음.

## 구조

고객 화면에는 `사비 17단계`라는 용어를 직접 노출하지 않는다. 대신 편집기 내부와 JSON에는 다음 압축 구조를 명시했다.

1. Hook / Hero: 1-4단계
2. Problem: 5-6단계
3. Mechanism: 7-9단계
4. Proof: 10-12단계
5. Offer: 13-15단계
6. Action: 16-17단계

## 미리보기 반영 방식

모든 양식 미리보기는 아래 내용을 포함한다.

- 대상 고객 호출
- 큰 약속 헤드라인
- 약속을 뒷받침하는 설명
- 고객이 막히는 이유
- 이미 해본 실패한 대안
- 해결 방식/차별점
- 진행 순서
- 신뢰 지표
- 후기
- 갤러리 이미지
- 대표 오퍼
- 가격/범위
- 한정 조건
- 보너스/포함 혜택
- 리스크 제거/안심 장치
- CTA
- P.S. 마지막 리마인더

## 양식별 차이

- 룩북형: Hero → Proof/Gallery → Problem → Mechanism → Price → Offer → FAQ → Action
- 포스터형: Hero/Event → Offer → Problem → Proof → Mechanism → FAQ → Action
- 인스타형: Profile/Links → Problem → Mechanism → Proof → Offer → FAQ → Action
- 예약형: Hero → Calendar → Problem → Mechanism → Proof → Offer → FAQ → Action
- 사례형: Dark case style → Problem → Mechanism → Proof → Offer → FAQ → Action

## 검증

브라우저에서 다음을 확인했다.

- 5개 양식 모두 Hook, Problem, Mechanism, Proof, Offer, Action 콘텐츠 유지.
- `step_map`과 `base_blocks`가 JSON에 표시됨.
- 입력값 변경 시 미리보기와 JSON에 즉시 반영됨.
- 고객 전달용 HTML 생성 함수가 각 양식별 HTML을 생성함.
- Chromium headless로 5개 양식 스크린샷 생성 완료.

## 2026-06-12 추가 UX 수정: 업종 선택과 양식 선택 분리

사용자 피드백: 업종 선택 시 양식이 랜덤하게 배정되는 흐름은 문제가 있다. 양식은 사용자가 직접 선택해야 한다.

반영 사항:

- 상단 설명을 `업종은 내용 프리셋, 양식은 사용자가 직접 선택`으로 변경.
- 좌측 업종 버튼을 `네일샵`, `청소업체`, `PT샵`으로 유지해 사용자가 자연스럽게 업종을 고르게 함. `데이터` 단어는 제거.
- 중앙 설명을 `업종 프리셋과 양식 선택은 분리합니다`로 변경.
- 업종 버튼은 문구/사진/오퍼 데이터만 바꾸고 현재 선택된 양식은 유지하도록 검증.
- JSON `template.selection_rule`에 `industry_preset_changes_content_only_format_is_user_selected_no_random` 추가.
- 검증 결과: 포스터형 선택 후 청소업체를 눌러도 포스터형 유지, 내용만 청소업체로 변경됨.

## 다음 기준

앞으로 이 프로젝트의 편집기/미리보기는 단순 예쁜 안내페이지가 아니라, 반드시 `사비 17단계 → 6개 고객 친화 블록` 기준으로 본문을 유지해야 한다. 축약하더라도 최소한 문제, 메커니즘, 증거, 오퍼, 보장, 행동 요청, P.S.는 빠지면 안 된다.

또한 업종 선택과 양식 선택은 반드시 분리한다. 업종 선택은 데이터/문구 프리셋만 바꾸고, 양식은 랜덤 배정하지 않으며 사용자가 직접 선택해야 한다.
