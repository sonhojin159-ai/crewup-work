# AI 학습 크론 점검 기록 2026-06-09

## 점검 배경

사용자가 2026-06-09 아침에 크론잡이 작동하지 않은 것 같다고 확인 요청.

## 확인 결과

크론 자체는 작동했다. 다만 05:00 KST 실행분 2개가 스크립트 제한 시간 120초에 걸려 실패했다.

   - last_status: error

## 원인

Hermes cron의 script-only job 기본 스크립트 제한 시간이 120초였다. 사용자가 원하는 학습 루프는 약 55분이므로, cron이 정상 시작하더라도 2분 뒤 강제 종료됐다.

## 조치

`hermes config set cron.script_timeout_seconds 4200` 실행.

검증:

```text
cron.scheduler._get_script_timeout() -> 4200
```

4200초는 70분이므로 55분 학습 루프와 저장/정리 시간을 감당할 수 있다.

## 참고

`hermes gateway restart`는 gateway 내부 실행이라 거부되었지만, scheduler의 `_get_script_timeout()`은 실행 시 config를 읽도록 되어 있어 다음 cron 실행부터 4200초가 적용될 것으로 확인했다.
