# 클로드 디자인형 편집 가능한 모바일 페이지 빌더 구상

작성일: 2026-06-09

## 핵심 사용자 구상

사용자는 중계 웹사이트를 단순한 템플릿 판매 페이지가 아니라, `클로드 디자인처럼 한눈에 보고 편집하는 웹 기반 모바일 페이지 빌더`로 구상했다.

핵심 문장:

> 사진이랑 판매 품목, 가격 정보, 이벤트 등을 입력 받아서 클로드 디자인처럼 한눈에 보고 편집하는 거지.

## 제품 정의 수정

기존 표현:

```text
엑셀 수정형 모바일 안내페이지 양식 마켓
```

수정된 표현:

```text
사진·품목·가격·이벤트를 입력하면 모바일 홍보페이지를 한눈에 보면서 편집할 수 있는 템플릿 빌더
```

또는:

```text
소상공인용 클로드 디자인식 모바일 홍보페이지 편집기
```

## 핵심 UX

사용자는 복잡한 관리자 페이지가 아니라 다음처럼 한 화면에서 작업해야 한다.

```text
왼쪽: 입력/편집 패널
오른쪽: 모바일 페이지 실시간 미리보기
상단: 양식 타입 선택
하단 또는 우측: 구매/다운로드/공유 버튼
```

모바일 또는 좁은 화면에서는:

```text
상단: 양식 선택
중간: 입력 카드
하단: 미리보기 전환/저장 버튼
```

## 입력받을 정보

필수 입력:

1. 사진
   - 메인 사진
   - 갤러리 사진
   - 이벤트 사진
   - 전후 사진
   - 품목/메뉴 사진

2. 판매 품목
   - 품목명
   - 카테고리
   - 설명
   - 대표 이미지
   - 노출 여부

3. 가격 정보
   - 가격
   - 할인 가격
   - 추가 비용 조건
   - 패키지 가격

4. 이달의 이벤트
   - 이벤트명
   - 기간
   - 혜택
   - 적용 조건
   - 강조 여부

5. 랜딩페이지 문구
   - 메인 제목
   - 설명문
   - CTA 문구
   - 안내 문구
   - 예약 전 주의사항

6. 링크 정보
   - 카카오톡 링크
   - 네이버지도 링크
   - 인스타그램 링크
   - 전화번호

## 편집 방식

클로드 디자인처럼 “결과물을 보면서 직접 수정하는 느낌”이 중요하다.

필요한 편집 UX:

- 텍스트 클릭 후 바로 수정
- 사진 클릭 후 교체
- 가격표 항목 추가/삭제
- 이벤트 카드 추가/삭제
- 섹션 순서 변경
- 양식 타입 즉시 변경
- 색상/폰트 프리셋 변경
- 모바일 미리보기 고정

## 양식 선택 방식

같은 데이터로 양식 타입만 바꿀 수 있어야 한다.

```text
룩북형
포스터형
인스타 피드형
견적 문의형
예약 캘린더형
전후 사례형
가격표형
리뷰형
쿠폰형
```

중요한 점:

```text
데이터는 하나
표현 방식은 여러 개
고객은 미리보면서 마음에 드는 양식을 고름
```

## 판매 구조

완성품 대행이 아니라 편집기/양식 사용권을 파는 구조다.

가격은 낮게 시작해야 한다.

예시:

```text
무료: 워터마크 미리보기
9,900원: 양식 1개 다운로드/공유
19,900원: 양식 1개 + 30일 수정 가능
29,000원: 양식 3개 비교/다운로드
49,000원: 맞춤 세팅 대행
월 9,900~29,000원: 호스팅/수정 유지
```

## 차별점

단순 템플릿 판매와 다른 점:

```text
템플릿 파일만 파는 것이 아니라, 입력 데이터가 즉시 디자인에 반영된다.
```

단순 랜딩페이지 대행과 다른 점:

```text
비싼 맞춤 제작이 아니라, 고객이 직접 보면서 고르고 수정한다.
```

노션/미리캔버스와 다른 점:

```text
소상공인 모바일 홍보페이지에 필요한 입력 항목과 양식만 좁혀서 제공한다.
```

## MVP 화면 구성

1. 히어로
   - “사진과 가격표만 넣으면 모바일 홍보페이지 완성”

2. 편집 데모
   - 좌측 입력 패널
   - 우측 모바일 미리보기

3. 양식 선택 바
   - 룩북형 / 포스터형 / 인스타형 / 예약형 / 사례형

4. 입력 탭
   - 기본정보
   - 사진
   - 판매 품목
   - 가격표
   - 이벤트
   - 문구
   - 링크

5. 구매 CTA
   - 미리보기 저장
   - HTML 다운로드
   - 링크 발급
   - 카톡으로 구매 문의

## 구현 우선순위

1. JSON 샘플 데이터로 편집기 UI 만들기.
2. 사진/가격/이벤트/문구를 입력하면 미리보기에 즉시 반영.
3. 양식 타입 변경 버튼 구현.
4. HTML 다운로드 기능 구현.
5. 워터마크/구매 제한 상태 구현.
6. 엑셀 업로드는 그 다음 단계로 추가.

## 중요한 제품 판단

초기에는 엑셀만 강조하기보다, 웹 편집기를 전면에 세우는 편이 더 직관적이다.

엑셀은 보조 입력 방식으로 두고, 핵심 경험은 다음이어야 한다.

```text
사진·품목·가격·이벤트 입력
→ 오른쪽에서 바로 보기
→ 양식 바꾸기
→ 마음에 드는 양식 구매
```

이 구조가 사용자가 말한 “클로드 디자인처럼 한눈에 보고 편집”하는 방향과 가장 잘 맞는다.
