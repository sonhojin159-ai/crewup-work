# 엑셀 기반 수정 가능 랜딩페이지 상품 구조

작성일: 2026-06-09

## 핵심 아이디어

수정 가능한 랜딩페이지를 판매하려면, 단순히 HTML을 만들어주는 것이 아니라 `엑셀 입력값에 따라 랜딩페이지 내용과 양식이 바뀌는 구조`가 되어야 한다.

사용자의 핵심 문장:

> 엑셀이랑 랜딩페이지를 입력받았을 때 랜딩페이지 양식이 바뀌어야 한다. 엑셀에는 가격 정보, 이달의 이벤트, 랜딩페이지 문구 등이 적혀 있어야 한다.

## 제품 방향성

이건 단순 랜딩페이지 대행이 아니라 다음에 가깝다.

```text
엑셀로 직접 수정 가능한 모바일 안내페이지 시스템
```

또는 판매 문구로는:

```text
사장님이 엑셀만 고치면 가격표·이벤트·문구·예약 안내가 자동 반영되는 모바일 홍보페이지
```

## 왜 중요한가

기존 HTML 납품형의 문제:

- 사장님이 가격을 바꾸려면 제작자에게 다시 요청해야 한다.
- 이달의 이벤트가 바뀌면 HTML 수정이 필요하다.
- 예약 가능 시간, 가격표, 안내문구가 자주 바뀌는 업종에는 유지보수 부담이 생긴다.
- “수정 가능”이라고 말하려면 실제로 사장님이 다룰 수 있는 입력 수단이 필요하다.

엑셀 기반 구조의 장점:

- 사장님에게 익숙하다.
- 별도 관리자 페이지보다 초기 구현이 쉽다.
- 네이버폼/구글시트/엑셀 업로드 등으로 확장 가능하다.
- 월관리 상품으로 연결하기 좋다.
- 양식 타입별 카탈로그와 결합하면 상품성이 좋아진다.

## 권장 구조

### 1. 엑셀 입력 파일

엑셀에는 단순 텍스트만이 아니라 페이지 구조 제어 정보도 들어가야 한다.

필수 시트:

1. `기본정보`
   - 매장명
   - 업종
   - 주소
   - 카카오톡 링크
   - 네이버지도 링크
   - 전화번호
   - 영업시간

2. `랜딩설정`
   - 양식 타입
   - 톤앤매너
   - 메인 컬러
   - 포인트 컬러
   - 폰트 스타일
   - 노출 섹션 순서

3. `히어로문구`
   - 상단 작은 문구
   - 메인 제목
   - 설명문
   - 대표 CTA
   - 보조 CTA

4. `가격표`
   - 메뉴명
   - 가격
   - 설명
   - 노출 여부

5. `이달의이벤트`
   - 이벤트명
   - 기간
   - 혜택
   - 설명
   - 노출 여부

6. `이미지`
   - 용도
   - URL 또는 파일명
   - 대체텍스트
   - 노출 여부

7. `예약가능시간`
   - 요일
   - 시간
   - 상태
   - 설명

8. `안내문구`
   - 제목
   - 내용
   - 노출 여부

## 양식 타입 제어

엑셀의 `랜딩설정.template_type` 값에 따라 랜딩페이지 양식이 바뀌어야 한다.

예시:

```text
lookbook      → 룩북/매거진형
poster        → 포스터/이벤트형
instagram     → 인스타 스토리/피드형
quote         → 디자인 선택/견적 문의형
calendar      → 예약 캘린더형
case          → 전후 사례형
price-menu    → 메뉴판/가격표형
```

중요한 점:

- 같은 엑셀 데이터라도 `template_type`이 바뀌면 HTML 레이아웃이 달라져야 한다.
- 가격표/이벤트/이미지는 공통 데이터로 쓰되, 보여주는 방식은 템플릿별로 달라야 한다.
- 고객에게는 “양식 선택 + 엑셀 입력” 구조로 보여주는 것이 좋다.

## 구현 구조

권장 파이프라인:

```text
엑셀 입력
→ 데이터 파서
→ 표준 JSON 변환
→ 템플릿 선택
→ HTML 렌더링
→ 모바일 페이지 생성
```

기술적으로는:

```text
.xlsx
→ parser
→ landing-data.json
→ template renderer
→ index.html
```

## 상품화 관점

초기 MVP는 관리자 페이지보다 엑셀 기반이 낫다.

1단계:

```text
엑셀 템플릿 제공
제작자가 엑셀을 받아 HTML 생성
```

2단계:

```text
구글시트 연동
시트 수정 시 페이지 재생성
```

3단계:

```text
간단한 관리자 화면 제공
사장님이 직접 수정
```

처음부터 관리자 페이지를 만들면 구현 범위가 커진다. 먼저 엑셀 기반으로 “수정 가능한 상품”의 가치를 검증하는 게 좋다.

## 판매 문구 후보

```text
가격표, 이달의 이벤트, 예약 안내를 엑셀로 직접 수정할 수 있는 모바일 홍보페이지
```

```text
사장님이 엑셀만 고치면 이번 달 아트와 이벤트가 바뀌는 인스타 프로필용 안내페이지
```

```text
매달 바뀌는 가격표·이벤트·예약 안내를 직접 수정 가능한 모바일 랜딩페이지로 만들어드립니다
```

## 중요한 제품 판단

이 아이디어는 단순한 부가 기능이 아니라 상품의 핵심 차별점이 될 수 있다.

기존 포지션:

```text
랜딩페이지 만들어드립니다
```

수정 후 포지션:

```text
엑셀로 직접 수정 가능한 소상공인 모바일 홍보페이지를 만들어드립니다
```

이쪽이 더 팔릴 가능성이 높다.
