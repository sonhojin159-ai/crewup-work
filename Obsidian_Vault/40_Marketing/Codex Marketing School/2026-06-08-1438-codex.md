## 1. 오늘의 결론

- 월 100~150만원은 “큰 프로젝트 1건”보다 `30만~50만원 제작 패키지 3건 + 10만~20만원 월관리 3~5건`이 현실적이다.
- 지금 팔아야 할 것은 “AI 작업물”이 아니라 `소상공인이 문의를 받을 수 있는 홍보 페이지와 문의 연결 구조`다.
- 첫 상품은 랜딩페이지, 카카오톡/전화/폼 연결, 기본 카피, 모바일 최적화, 직접 수정 가능한 필드까지 포함한 소형 패키지로 잡는다.
- 첫 고객은 포트폴리오보다 `무료 홍보 진단 + 개선 예시 1장`으로 접근하는 것이 유리하다.
- 이번 주 목표는 공부가 아니라 `실제 제안 30명, 진단 5명, 유료 전환 1명`이다.

## 2. 수익화 관점의 핵심 개념

### 개념 정의

이번 상품의 핵심은 “랜딩페이지 제작”이 아니라 `문의 전환용 홍보 구조 제작`이다.

즉, 고객에게 이렇게 팔아야 한다.

> “홈페이지 하나 만들어드립니다”  
> 보다  
> “네이버, 인스타, 블로그에서 보고 들어온 사람이 바로 전화/카톡 문의할 수 있는 1페이지 홍보 구조를 만들어드립니다.”

소상공인은 디자인 자체보다 다음 문제에 돈을 낸다.

- 가게 설명이 산만하다.
- 가격/서비스가 한눈에 안 보인다.
- 전화나 카카오톡 문의로 연결이 약하다.
- 모바일에서 보기 불편하다.
- 인스타/블로그 링크를 눌렀을 때 보여줄 깔끔한 페이지가 없다.
- 본인이 직접 문구, 가격, 사진을 바꾸기 어렵다.

### 왜 월 100~150만원 목표에 중요한가

초기에는 기술 난이도가 낮고, 고객이 이해하기 쉬우며, 납기가 짧은 상품이 필요하다.

추천 구조는 다음과 같다.

| 상품 | 가격 | 월 목표 |
|---|---:|---:|
| 홍보 랜딩페이지 제작 | 30만~50만원 | 월 2~3건 |
| 월관리 | 10만~20만원 | 누적 3~5곳 |
| 수정/추가 섹션 | 5만~15만원 | 선택 매출 |

예시:

- 제작 40만원 × 2건 = 80만원
- 월관리 15만원 × 4곳 = 60만원
- 합계 = 140만원

이 구조가 중요한 이유는 매달 새 고객만 찾아다니지 않아도 되기 때문이다.

### 초보자가 빠지기 쉬운 함정

- “AI로 멋진 페이지 제작”처럼 공급자 언어로 말한다.
- 고객 업종을 너무 넓게 잡는다.
- 가격을 너무 낮게 잡아 수정 요청에 끌려다닌다.
- 제작 후 월관리 상품으로 연결하지 않는다.
- CTA, 전화, 카카오톡, 폼 연결 없이 예쁜 페이지만 만든다.
- 모바일 화면을 먼저 설계하지 않는다.
- 고객이 직접 바꿔야 할 필드와 건드리면 안 되는 영역을 구분하지 않는다.

## 히어로 섹션 방향

히어로는 첫 화면에서 바로 다음 정보가 보여야 한다. 모바일 기준으로 너무 큰 이미지나 긴 문구를 쓰면 안 된다.

구성:

- 상단: 업종명 또는 브랜드명
- 메인 카피: 고객이 얻는 결과
- 서브 카피: 지역/대상/특징
- CTA: 카카오톡 문의, 전화하기
- 보조 정보: 운영시간, 지역, 첫 상담 가능 여부

예시 카피:

> 강남 1:1 체형교정 PT  
> 통증과 자세 고민을 상담부터 운동 루틴까지 맞춤 관리합니다.  
> 첫 상담 예약 가능 · 카카오톡 문의 1분 연결

## 문제 제기 섹션

고객의 고객이 느끼는 불안을 짚어야 한다.

예시:

- 가격이 어떻게 되는지 모르겠어요.
- 나에게 맞는 서비스인지 판단하기 어려워요.
- 상담 전에 실제 후기를 보고 싶어요.
- 문의하려면 어디로 연락해야 하는지 헷갈려요.

이 섹션은 “그래서 이 페이지가 필요합니다”가 아니라, 해당 사업자의 서비스 구매 장벽을 낮추는 역할이다.

## 서비스/가격 섹션

가격 공개가 가능한 업종은 가격을 보여준다.

```text
기본 상담
30분 / 30,000원
처음 방문하시는 분께 추천

대표 프로그램
4회 / 220,000원
가장 많이 선택하는 구성

정기 관리
월 4회 / 상담 후 안내
개인 상태에 맞춘 관리
```

가격 공개가 어려운 업종은 이렇게 쓴다.

```text
상담 후 견적
현장 상태, 요청 범위, 일정에 따라 안내드립니다.
카카오톡으로 사진을 보내주시면 대략적인 비용을 먼저 안내드립니다.
```

## 후기/신뢰 요소

초기 고객은 후기가 없을 수 있다. 이때는 대체 신뢰 요소를 쓴다.

- 대표자 얼굴 또는 작업 사진
- 작업 과정 사진
- 자격증/경력
- 실제 사용하는 도구
- 상담 전후 체크리스트
- 샘플 결과물
- 네이버 플레이스/블로그 링크

후기 문구 예시:

> “가격과 진행 방식이 한눈에 보여서 문의하기 편했어요.”

## CTA 버튼

CTA는 최소 3곳에 반복 배치한다.

- 히어로 하단
- 가격 섹션 하단
- 마지막 섹션

버튼 문구:

- 카카오톡으로 상담하기
- 전화 문의하기
- 30초 신청폼 작성
- 사진 보내고 견적 받기
- 첫 상담 예약하기

연결 방식:

## 1. 고객 수정형 홍보페이지의 상품 명세

소상공인에게 팔 상품은 “홈페이지 제작”보다 “직접 수정 가능한 문의페이지 세팅”으로 정의하는 편이 좋습니다.


### 상품 설명


## 3. 업종별 CTA 우선순위

업종마다 자연스러운 문의 방식이 다릅니다. 모든 버튼을 같은 비중으로 두면 전환이 흐려집니다.

| 업종 | 1순위 CTA | 2순위 CTA | 이유 |
|---|---|---|---|
| PT샵 | 카카오톡 상담 | 전화 | 상담 전 질문이 많음 |
| 피부관리실 | 네이버 예약 | 카카오톡 | 시간 예약이 핵심 |
| 공방 | 신청폼 | 카카오톡 | 날짜, 인원, 클래스 선택 필요 |
| 청소업체 | 사진 보내고 견적 | 전화 | 현장 상태 확인 필요 |
| 수리업체 | 전화 | 사진 견적 | 긴급성이 높음 |
| 과외/학원 | 상담 신청폼 | 카카오톡 | 학생 정보 수집 필요 |
| 세무/노무 | 상담폼 | 전화 | 사전 정보가 많을수록 효율적 |

페이지 안에서는 버튼을 이렇게 나눕니다.

```text
메인 CTA: 업종에서 가장 자연스러운 행동 1개
보조 CTA: 급한 고객을 위한 전화 또는 카카오톡
숨은 CTA: 폼, 예약, 지도, 인스타 등 보조 링크
```

예를 들어 청소업체라면 “카카오톡 상담하기”보다 “사진 보내고 견적 받기”가 더 직접적입니다.

## 4. 무료 진단 보고서 템플릿

첫 고객 확보용 무료 진단은 길면 안 됩니다. 1장짜리로 끝내야 합니다.

```markdown
# 무료 문의페이지 진단

업체명: ○○○
확인일: 2026-06-08

## 1. 현재 좋은 점
- 네이버 플레이스에 기본 정보가 등록되어 있습니다.
- 실제 사진이 있어 신뢰를 만들 수 있습니다.

## 2. 문의 전 막히는 지점
- 가격 또는 대표 서비스가 한눈에 보이지 않습니다.
- 카카오톡/전화 문의 버튼이 바로 보이지 않습니다.
- 처음 방문자가 어떤 순서로 상담받는지 알기 어렵습니다.

## 3. 개선하면 좋은 구조
- 첫 화면: 대표 서비스 + 지역 + 문의 버튼
- 중간: 가격/프로그램 3개 정리
- 하단: 후기/사진/FAQ/카카오톡 연결

## 4. 제안
모바일에서 바로 문의할 수 있는 1페이지 홍보페이지를 제작해드릴 수 있습니다.

- 제작비: 29만원 체험가
- 포함: 페이지 제작, 카카오톡/전화 연결, 가격표, FAQ, 직접 수정 기능
- 무료관리: 제작 후 1개월
- 이후 관리: 월 10만원부터 선택
```

진단의 목적은 “문제 지적”이 아니라 “바로 만들면 좋아지는 그림”을 보여주는 것입니다.

## 5. 첫 영업 메시지 카피

너무 판매색이 강하면 답장이 줄어듭니다. 첫 메시지는 짧고 구체적이어야 합니다.

### 인스타 DM

```text
안녕하세요. ○○ 지역 소상공인 홍보페이지를 만들고 있습니다.

플레이스/인스타를 보고 들어왔는데,
서비스 가격이랑 상담 연결이 한 화면에 정리되면 문의가 더 편해질 것 같아서요.

원하시면 무료로 3가지 개선 포인트만 짧게 정리해드릴게요.
홍보나 광고 제안은 아니고, 모바일 문의 동선 진단입니다.
```

### 카카오톡/문자

```text
안녕하세요. 소상공인용 모바일 문의페이지를 제작하고 있습니다.

○○을 보고 연락드렸습니다.
현재 정보는 잘 올라와 있는데, 가격/서비스/문의 버튼이 한 페이지에 정리되면 신규 고객이 더 쉽게 문의할 수 있을 것 같았습니다.

괜찮으시면 무료로 간단한 개선안 1장 보내드려도 될까요?
```

### 답장 왔을 때

```text
감사합니다. 아래 3가지만 확인해서 짧게 정리해드리겠습니다.

1. 첫 화면에서 어떤 서비스인지 바로 보이는지
2. 가격/상담 방식이 충분히 쉬운지
3. 카카오톡/전화 문의까지 막히는 부분이 있는지

오늘 안에 1장으로 보내드릴게요.
```

## 6. 29만원 체험가 제안 구조

무료 진단 후 바로 “제작하실래요?”라고 묻기보다, 선택지를 좁혀 제안해야 합니다.

```text
진단해보니 새 홈페이지를 크게 만들 필요는 없어 보이고,
모바일 문의용 1페이지면 충분할 것 같습니다.

제가 체험가로 아래 구성까지 29만원에 제작해드릴 수 있습니다.

- 첫 화면 문구 정리
- 서비스/가격표 구성
- 후기/신뢰 요소 배치
- FAQ 정리
- 카카오톡/전화/신청폼 연결
- 사장님이 직접 수정 가능한 입력 화면
- 제작 후 1개월 무료관리

이후에는 직접 수정하셔도 되고,
```

|  |  | 인스타 | 가격 없음 | A안 | Y/N | Y/N | 대기/거절/계약 |

초기에는 매출보다 “어떤 문장에 답장이 오는지”가 더 중요한 데이터입니다.

```markdown
12. 직접 수정할 부분과 맡기고 싶은 부분을 나누면 운영이 편할까요?
```

이 질문의 목적은 “홈페이지가 필요하시죠?”라고 설득하는 것이 아닙니다. 고객 스스로 “문의 전에 설명할 페이지가 없어서 반복 응대가 생긴다”고 느끼게 하는 것입니다.

## 3. 29만원 체험가의 작업 범위 고정

체험가는 싸게 해주는 상품이 아니라, 범위를 제한한 첫 거래 상품이어야 합니다. 범위가 느슨하면 첫 고객부터 손해가 납니다.

### 포함 범위

| 항목 | 포함 |
|---|---|
| 모바일 문의페이지 1개 | 포함 |
| 섹션 수 | 최대 8개 |
| 대표 이미지 | 최대 1장 |
| 서비스/가격 카드 | 최대 3개 |
| 사진 갤러리 | 최대 6장 |
| FAQ | 최대 5개 |
| 문의 버튼 | 카카오톡, 전화, 예약 중 최대 2개 |
| 수정 라운드 | 1회 |
| 무료관리 | 제작 후 1개월 |

### 제외 범위

| 항목 | 처리 방식 |
|---|---|
| 로고 제작 | 별도 견적 |
| 사진 보정 | 별도 견적 또는 고객 제공 |
| 긴 블로그형 글 작성 | 월관리 또는 추가 작업 |
| 예약 시스템 개발 | 외부 링크 연결까지만 |
| 결제 기능 | 초기 상품에서는 제외 |
| 다국어 페이지 | 별도 견적 |
| 여러 지점 페이지 | 지점당 추가 비용 |

### 고객에게 설명하는 문장

```text
29만원 체험가는 처음부터 큰 홈페이지를 만드는 상품이 아니라,
고객이 문의 전에 꼭 봐야 할 정보를 모바일 1페이지로 정리하는 구성입니다.

카카오톡 문의, 전화, 예약 링크까지 연결해드리고,
제작 후 1개월 동안 가격/사진/공지 같은 운영 수정도 같이 봐드립니다.
```

가격을 낮출수록 작업 범위는 더 선명해야 합니다. 그래야 월관리 전환 전까지 에너지를 남길 수 있습니다.

## 4. 월관리 상품을 3단계로 나누기

월관리 10만원 하나만 두면 고객 상황에 따라 애매해질 수 있습니다. 처음에는 단순하게 3단계로 나누되, 주력은 10만원 상품으로 잡습니다.

| 상품 | 가격 | 대상 | 포함 내용 |
|---|---:|---|---|
| 셀프 운영 | 0원 | 직접 수정 가능한 고객 | 수정폼 제공, 기본 사용 안내 |
| 기본 관리 | 월 10만원 | 바쁜 소상공인 | 월 2회 문구/사진/공지/FAQ 수정 |
| 성장 관리 | 월 20만원 | 이벤트와 후기 관리가 필요한 곳 | 월 4회 수정, 후기 정리, 이벤트 문구, 섹션 개선 제안 |

### 월 10만원 상품 설명

```text
직접 수정도 가능하지만, 매달 가격표, 이벤트, 사진, FAQ가 조금씩 바뀌면
페이지가 금방 오래된 느낌이 납니다.

월 10만원 기본 관리는 한 달에 2번까지
공지, 가격, 사진, 자주 묻는 질문, 문의 버튼을 정리해드리는 상품입니다.
```

월관리의 핵심은 “내가 대신 고쳐준다”가 아니라 “페이지가 방치되지 않게 한다”입니다.

## 5. 첫 고객 제안 메시지 A/B 테스트

초기 영업에서는 메시지를 감으로 바꾸면 안 됩니다. 문제 지적형, 링크 정리형, 체험가 제안형으로 나눠서 테스트해야 합니다.

### A안: 문제 지적형

```text
안녕하세요. 근처 업체 홍보페이지를 살펴보다가 연락드렸습니다.

현재 고객이 가격, 작업 범위, 문의 방법을 한 번에 확인하기가 조금 어려워 보여서요.
특히 모바일에서 카카오톡 문의까지 이어지는 동선이 더 짧아지면 좋을 것 같습니다.

원하시면 “문의 전에 막히는 지점 3개”만 간단히 정리해서 보내드릴 수 있습니다.
```

### B안: 링크 정리형

```text
안녕하세요.

인스타, 네이버 플레이스, 문의 링크가 따로 흩어져 있으면
고객이 문의 전에 필요한 정보를 찾다가 이탈하는 경우가 많습니다.

업체 소개, 대표 상품, 가격 안내, 후기, 카카오톡 문의를
모바일 1페이지로 정리하는 세팅을 하고 있습니다.

원하시면 현재 페이지 기준으로 개선 포인트 3개를 무료로 정리해드릴게요.
```

### C안: 체험가 직접 제안형

```text
안녕하세요.

소상공인용 모바일 문의페이지를 29만원 체험가로 제작하고 있습니다.
고객이 가격, 사진, FAQ를 보고 바로 카카오톡으로 문의할 수 있게 만드는 페이지입니다.

제작 후 1개월 동안 공지/사진/가격 수정도 같이 봐드립니다.
괜찮으시면 현재 홍보 동선을 보고 맞는 구성인지 먼저 무료로 진단해드리겠습니다.
```

### 기록 기준

| 항목 | 기준 |
|---|---|
| 발송 수 | 메시지별 10곳 |
| 1차 성공 | 답장 |
| 2차 성공 | 무료 진단 수락 |
| 3차 성공 | 29만원 제안 수락 |
| 실패 기록 | 읽씹, 가격 부담, 필요 없음, 나중에 |

처음에는 어떤 메시지가 “잘 썼는지”보다 어떤 메시지가 “답장을 만드는지”만 봐야 합니다.

## 누적 지식에 추가할 메모

- 상품은 “홍보페이지 제작”이 아니라 업종별 “문의 전환 패키지”로 보여야 구매 이유가 선명해진다.
- 첫 상담은 설명보다 진단 질문으로 시작해야 고객이 자신의 문의 동선 문제를 직접 인식한다.
- 29만원 체험가는 저가 맞춤 제작이 아니라 범위가 제한된 첫 거래 상품이어야 한다.
- 월관리는 단일 상품보다 셀프 운영, 기본 관리, 성장 관리로 나누면 고객 상황별 제안이 쉬워진다.
- 초기 영업 메시지는 감으로 바꾸지 말고 문제 지적형, 링크 정리형, 체험가 제안형으로 나눠 답장률을 기록해야 한다.

## 1. 업종별 “수정 가능한 정보”를 먼저 제한하기

## 3. 29만원 체험가의 “범위 고정” 문서

## 4. 첫 고객 확보용 업종 우선순위

처음부터 모든 업종을 대상으로 하면 메시지가 흐려집니다.  
첫 5명은 “문의 전 정보 확인이 많은 업종”에 집중하는 편이 좋습니다.

### 우선순위 기준

| 기준 | 설명 |
|---|---|
| 가격 질문이 많다 | 페이지가 반복 응대 시간을 줄여줄 수 있음 |
| 사진 신뢰가 중요하다 | 전후 사진, 사례 사진으로 설득 가능 |
| 예약/상담 전환이 있다 | 카카오톡 문의 버튼 효과가 분명함 |
| 사장님이 직접 운영한다 | 의사결정자가 가까워 영업이 빠름 |
| 기존 채널이 흩어져 있다 | 인스타, 플레이스, 블로그 정보를 정리할 명분이 있음 |

### 추천 1순위 업종

| 업종 | 제안 포인트 |
|---|---|
| 청소/정리수납 | 전후 사진, 가격 기준, 가능 지역 정리가 중요 |
| 피부/왁싱/네일 | 가격표, 이벤트, 예약 문의가 자주 바뀜 |
| PT/필라테스 | 체험권, 프로그램, 상담 흐름 정리가 필요 |
| 공방/클래스 | 일정, 모집 인원, 준비물, 수강료가 자주 바뀜 |
| 출장 서비스 | 지역, 비용, 작업 사례, 문의 버튼이 중요 |

초기에는 “홈페이지 필요하세요?”보다 “카톡 문의 전에 고객이 확인할 가격표와 FAQ를 한 페이지로 정리해드린다”가 더 잘 먹힙니다.

## 5. 첫 10곳 영업 실험안

영업은 감으로 하지 말고 Obsidian에 실험으로 기록해야 합니다.  
목표는 당장 대량 판매가 아니라 “어떤 업종이 반응하는지”를 찾는 것입니다.

```markdown
# 첫 10곳 영업 실험

## 실험 목표
29만원 체험가 제안에 반응하는 업종과 메시지를 찾는다.

## 대상
- 청소업체 3곳
- 피부/왁싱/네일 3곳
- PT/필라테스 2곳
- 공방/클래스 2곳

## 보낼 제안
A안: 무료 진단 중심
B안: 카톡 문의 동선 개선 중심
C안: 직접수정 홍보페이지 중심

## 기록 항목
| 날짜 | 업체 | 업종 | 채널 | 메시지안 | 반응 | 다음 액션 |
|---|---|---|---|---|---|---|
|  |  |  | 인스타/카톡/문자 | A/B/C | 읽음/답장/거절/상담 |  |

## 성공 기준
- 10곳 중 답장 2곳 이상
- 상담 1곳 이상
- 무료 진단 요청 1곳 이상

## 실패 기준
- 답장 0곳
- 메시지 읽힘은 있으나 반응 없음
- 가격 언급 전에 거절 발생
```

### 메시지 A안: 무료 진단 중심

```text
안녕하세요. 인스타와 플레이스 정보를 보고 연락드렸습니다.

판매 제안보다 먼저,
고객이 문의 전에 어디서 막힐 수 있는지만 간단히 정리해드리는 방식입니다.
```

### 메시지 B안: 카톡 문의 동선 중심

```text
안녕하세요. 현재 운영하시는 채널을 보니
고객이 가격과 자주 묻는 질문을 확인한 뒤 카톡으로 문의하면 더 편할 것 같아 연락드렸습니다.

```

### 메시지 C안: 직접수정 중심

```text
안녕하세요.

사장님이 공지, 가격, 사진, FAQ 정도는 직접 바꿀 수 있고

초기 사례를 만드는 중이라 체험가 29만원으로 진행하고 있습니다.
```

## 3. 29만원 상품의 납품물 구조

```markdown
## 2. 직접수정 입력폼
- 공지 수정
- 가격/상품 수정

- 문의 링크 수정

```

고객에게는 “페이지 제작”보다 “문의받기 전 필요한 정보 정리 + 수정 가능한 페이지 + 한 달 운영 확인”으로 보여줘야 가격 저항이 줄어듭니다.

## 4. 월관리 전환 설계

무료관리 1개월 뒤에 갑자기 “월관리 하실래요?”라고 물으면 부담스럽습니다.  
처음부터 무료관리 기간을 “운영 점검 기간”으로 설계해야 합니다.

```markdown
# 월관리 전환 시나리오

## 오픈 당일
전달 메시지:
페이지 오픈했습니다.
이번 한 달은 실제 문의 흐름을 보면서 공지, FAQ, 가격 표현을 조금씩 다듬어보면 좋습니다.

## 1주차
점검 항목:
- 카카오톡 버튼 클릭 문제 없는지
- 고객이 가격을 다시 묻는지
- FAQ에 없는 질문이 들어오는지

## 2주차
점검 항목:
- 사진 순서가 적절한지
- 첫 화면 문장이 이해되는지
- 문의가 어느 버튼에서 많이 발생하는지

## 3주차
점검 항목:
- 반복 질문 추가
- 이벤트/공지 업데이트
- 가격 표현 보완

## 4주차
월관리 제안:
이번 한 달 동안 공지, FAQ, 사진, 가격 표현처럼 운영 중에 바뀌는 부분이 있었습니다.
앞으로도 월 2회 정도 정리해드리는 관리형으로 두시면 페이지가 방치되지 않습니다.

월관리 비용은 월 9만원이고,
간단 수정, 문의 버튼 확인, FAQ 보완, 공지 교체까지 포함됩니다.
```

월관리 상품은 “서버 유지비”처럼 말하면 약합니다.  
“페이지가 방치되지 않게 월 2회 운영 정리”라고 말하는 편이 더 현실적입니다.

핵심은 매월 3건 제작이 아니라, 3개월 뒤에도 관리 고객이 남는 구조입니다.

아래 순서가 가장 무난합니다.

```markdown
# 모바일 홍보페이지 기본 와이어프레임

## 1. 첫 화면
- 대표 사진
- 매장명
- 한 줄 핵심 문장
- 주요 혜택 또는 특징 2개
- 카카오톡 문의 버튼

## 2. 고객 상황 공감
- 이런 분께 적합합니다
- 방문/예약 전 자주 생기는 고민 3개

## 3. 대표 상품/서비스
- 상품명
- 기준 가격
- 소요시간
- 포함사항
- 추천 대상

## 4. 사진/결과물
- 실제 공간 또는 결과 사진
- 작업 전후 또는 완성 예시
- 클래스/시술/상담 장면

## 5. 자주 묻는 질문
- 가격 관련 질문
- 예약 관련 질문
- 준비물/주의사항
- 환불/변경 기준

## 6. 위치/운영 정보
- 주소
- 영업시간
- 주차/대중교통
- 예약 가능 시간

## 7. 마지막 문의 영역
- “상담 전 아래 내용을 보내주세요”
- 카카오톡 문의 버튼
- 전화 또는 예약 링크
```

첫 화면과 마지막 화면에 문의 버튼을 넣고, 중간에는 판단 정보를 배치합니다.

## 3. 카카오톡 문의 유도 문구

문의 버튼은 “상담하기”보다 구체적이어야 클릭률이 높습니다.

```markdown
# 카카오톡 버튼 문구 예시

## 가격형
- 가격 상담하기
- 내 상황에 맞는 비용 물어보기
- 예약 가능 금액 확인하기

## 예약형
- 이번 주 예약 가능 시간 보기
- 방문 일정 문의하기
- 클래스 자리 확인하기

## 상담형
- 사진 보내고 상담받기
- 증상/상황 설명하고 문의하기
- 맞춤 안내 받기

## 클래스형
- 수업 일정 문의하기
- 준비물 확인하고 신청하기
- 원하는 날짜로 문의하기
```

문의 직후 자동 안내 문구도 준비해두면 좋습니다.

```markdown
안녕하세요. 빠른 안내를 위해 아래 내용을 함께 보내주세요.

1. 원하시는 서비스 또는 클래스
2. 희망 날짜와 시간
3. 궁금한 점
4. 참고 사진이 있다면 첨부
```

## 4. 제작 전 자료 요청 체크리스트

자료 요청이 느슨하면 제작 일정이 밀립니다.  
처음부터 “없으면 대체 가능”까지 정해두는 편이 좋습니다.

```markdown
# 제작 전 자료 요청 체크리스트

## 필수 자료
- 매장명
- 주소
- 영업시간
- 대표 서비스 3개
- 가격표 또는 기준 가격
- 카카오톡 문의 링크
- 대표 사진 3장 이상

## 있으면 좋은 자료
- 후기
- 전후 사진
- 공간 사진
- 시술/작업 과정 사진
- 기존 인스타그램 또는 블로그 링크
- 네이버 플레이스 링크

## 없을 때 대체 방식
- 가격표 없음: 기준 가격 또는 “상담 후 안내” 문구 사용
- 후기 없음: 자주 듣는 고객 반응을 짧은 문장으로 정리
- 사진 부족: 우선 보유 사진으로 오픈 후 1개월 내 교체
- FAQ 없음: 상담 질문지에서 반복 문의를 뽑아 구성
```

이 체크리스트는 고객에게 그대로 보내도 되고, 내부 제작 명세로 써도 됩니다.

## 5. 첫 고객 확보용 실험안

초기에는 광고보다 직접 제안이 빠릅니다.  
단, “홈페이지 만들어드릴게요”가 아니라 “문의 전환 정보 정리해드릴게요”로 접근해야 합니다.

```markdown
# 첫 고객 확보 2주 실험안

## 대상
- 인스타는 운영하지만 링크가 정리되지 않은 매장
- 네이버 플레이스 정보는 있으나 가격/FAQ가 부족한 매장
- 클래스, 예약, 상담 문의가 반복되는 업종
- 사진 자료가 어느 정도 있는 소상공인

## 1일차
- 후보 30곳 리스트업
- 업종, 지역, 현재 링크, 부족한 정보 기록

## 2~3일차
- 10곳에 1차 메시지 발송
- 답변 여부 기록

## 4~5일차
- 관심 보인 곳에 간단 제안서 전달
- 무료 샘플 구조 1개 작성

## 6~7일차
- 1명에게 체험가 제안
- 자료 요청 체크리스트 발송

## 2주차
- 제작 1건 완료 목표
- 제작 과정 기록
- 전후 비교 자료 확보
- 다음 영업 메시지에 사례로 활용
```

영업 기록은 아래처럼 남기면 됩니다.

```markdown
# 영업 기록

## 업체명
- 업종:
- 지역:
- 현재 채널:
- 부족한 정보:
- 제안 포인트:
- 발송일:
- 반응:
- 다음 액션:
```

- 첫 고객 확보는 대량 광고보다 후보 리스트, 직접 제안, 샘플 구조, 전후 사례 확보 순서로 진행하는 편이 현실적이다.

# 추가 심화 라운드 8 / 경과 423초

## 1. 월관리 전환을 위한 “1개월 무료관리” 설계

무료관리는 호의가 아니라 유료관리 전환을 검증하는 기간입니다.  
따라서 “무엇을 해주는지”보다 “무엇을 기록해서 다음 달 비용을 정당화할지”가 중요합니다.

```markdown
# 1개월 무료관리 운영 명세

## 목적
- 페이지 오픈 후 실제 문의 흐름 확인
- 누락된 정보 보완
- 반복 문의를 FAQ와 버튼 문구에 반영
- 월관리 전환 근거 확보

## 무료관리 기간
- 페이지 오픈일 기준 30일

## 포함 작업
- 문구 수정 월 2회
- 가격/상품 정보 수정 월 2회
- 공지 추가 월 2회
- FAQ 추가 월 3개
- 카카오톡 문의 문구 개선 1회
- 간단 이미지 교체 월 3장

## 제외 작업
- 전체 디자인 변경
- 신규 섹션 대규모 추가
- 상세페이지 추가 제작
- 사진 촬영
- 블로그/인스타 콘텐츠 제작
- 광고 집행

## 기록할 데이터
- 카카오톡 문의 수
- 자주 들어온 질문
- 문의 전 이탈 원인 추정
- 고객이 헷갈려 한 표현
- 가격 문의 비율
- 예약 가능 여부 문의 비율
- 수정 요청 횟수
- 실제 반영한 수정 내역
```

무료관리 종료 시점에는 아래 문장으로 전환합니다.

```markdown
지난 1개월 동안 문의 흐름을 보니 가격, 예약 가능 시간, 준비사항 관련 질문이 반복되었습니다.

다음 달부터는 이 부분을 계속 정리해서
고객 문의 전에 필요한 정보를 줄이고,
사장님이 반복 답변하는 시간을 줄이는 방향으로 관리해드릴 수 있습니다.
```

## 2. 월관리 상품 3단계 패키지

월 100~150만원 목표라면 고가 1건보다 소액 월관리 여러 건이 안정적입니다.  
초기에는 아래처럼 단순한 3단계가 좋습니다.

```markdown
# 월관리 패키지

## A. 기본 관리형 / 월 5만원
적합 대상:
- 가격, 공지, FAQ 정도만 자주 바뀌는 매장
- 문의량은 적지만 정보 최신성이 필요한 매장

포함:
- 텍스트 수정 월 2회
- 공지 등록 월 1회
- FAQ 추가 또는 수정 월 2개
- 버튼 링크 점검 월 1회

제외:
- 디자인 변경
- 신규 섹션 추가
- 이미지 대량 교체

## B. 문의 개선형 / 월 10만원
적합 대상:
- 카카오톡 문의를 늘리고 싶은 매장
- 가격, 예약, 후기, 사진 업데이트가 있는 매장

포함:
- 텍스트 수정 월 4회
- 공지 등록 월 2회
- FAQ 추가 또는 수정 월 5개
- 이미지 교체 월 5장
- 카카오톡 문의 문구 개선 월 1회
- 월 1회 간단 리포트

## C. 성장 관리형 / 월 15만원
적합 대상:
- 클래스, 예약제, 상담형 업종
- 상품 구성이 자주 바뀌는 매장
- 문의 흐름을 계속 개선하고 싶은 매장

포함:
- 텍스트 수정 월 6회
- 공지 등록 월 4회
- FAQ 추가 또는 수정 월 8개
- 이미지 교체 월 10장
- 상품/서비스 섹션 개선 월 1회
- 후기/사례 섹션 업데이트 월 1회
- 월 1회 개선 제안서
```

목표 조합은 이렇게 잡을 수 있습니다.

```markdown
# 월 100~150만원 조합 예시

## 안정형
- 월 5만원 고객 10명 = 50만원
- 월 10만원 고객 7명 = 70만원
- 합계 120만원

## 소수 집중형
- 월 10만원 고객 8명 = 80만원
- 월 15만원 고객 4명 = 60만원
- 합계 140만원

## 초기 현실형
- 제작비 20만원 고객 3명 = 60만원
- 월 5만원 고객 4명 = 20만원
- 월 10만원 고객 3명 = 30만원
- 합계 110만원
```

그래서 “사장님이 직접 수정 가능 + 필요하면 대신 관리” 구조가 더 현실적입니다.

```markdown
# 수정 요청 양식

## 요청 유형
- 공지 추가
- 가격 변경
- 상품 설명 수정
- 사진 교체
- FAQ 추가
- 버튼 링크 변경
- 운영시간 변경
- 기타

## 수정 위치
- 첫 화면
- 상품/서비스
- 공지
- FAQ
- 후기/사례
- 문의 버튼
- 위치/운영 정보

## 수정할 내용
- 기존 내용:
- 변경할 내용:

## 반영 희망일
- 오늘
- 이번 주 내
- 특정 날짜:

## 첨부 자료
- 사진:
- 링크:
- 참고 문구:

## 긴급 여부
- 일반
- 긴급

## 긴급 사유
- 가격 오류
- 예약 마감
- 휴무/영업시간 변경
- 이벤트 종료
- 잘못된 링크
```

## 4. 오픈 후 7일 점검 체크리스트

페이지 제작 후 바로 끝내면 월관리 전환 명분이 약합니다.  
오픈 7일 안에 점검을 넣어야 “관리의 필요성”이 자연스럽게 드러납니다.

```markdown
# 오픈 후 7일 점검 체크리스트

## 1. 링크 점검
- 카카오톡 버튼 정상 작동
- 전화 버튼 정상 작동
- 네이버 예약 링크 정상 작동
- 인스타그램 링크 정상 작동

## 2. 모바일 화면 점검
- 첫 화면에서 매장명 확인 가능
- 핵심 문장이 잘리지 않음
- 버튼이 바로 보임
- 가격 정보가 너무 아래에 있지 않음
- 사진 로딩이 어색하지 않음

## 3. 문의 흐름 점검
- 고객이 어떤 버튼을 눌러야 할지 명확함
- 문의 전 준비해야 할 내용이 있음
- 가격 문의를 줄일 정보가 있음
- 예약 가능 여부를 확인할 방법이 있음

## 4. 정보 누락 점검
- 주소
- 영업시간
- 휴무일
- 주차 정보
- 소요시간
- 가격 기준
- 예약 변경 기준
- 준비물 또는 주의사항

## 5. 월관리 후보 항목
- 자주 바뀔 공지
- 계절별 상품
- 이벤트
- 후기 업데이트
- 가격 변동
- 예약 마감 안내
- 신규 사진 교체
```

점검 후에는 고객에게 아래처럼 보낼 수 있습니다.

```markdown
오픈 후 7일 기준으로 확인해보니
현재 페이지는 기본 문의 동선은 잘 작동합니다.

다만 앞으로 문의 품질을 높이려면
FAQ, 공지, 후기, 가격 안내를 계속 업데이트하는 쪽이 중요합니다.

특히 예약제 업종은 정보가 오래되면
고객이 문의 전에 이탈하거나 같은 질문을 반복하게 됩니다.
```

## 5. 첫 고객 제안 메시지 3종

초기 영업에서는 “홈페이지 제작”보다 “반복 문의 줄이기”가 더 잘 먹힙니다.

```markdown
# 제안 메시지 1: 인스타 운영 매장용

안녕하세요. 인스타그램 보고 연락드립니다.

현재 게시물에는 서비스 사진이 잘 정리되어 있는데,
처음 보는 고객 입장에서는 가격, 예약 방법, 자주 묻는 질문을 한 번에 확인하기가 조금 어려워 보였습니다.

인스타 프로필 링크에 연결할 수 있는
모바일용 간단 홍보페이지를 만들어드리고 있습니다.

카카오톡 문의 버튼, 대표 서비스, 가격 안내, FAQ를 한 페이지로 정리해서
고객이 문의 전에 필요한 내용을 먼저 확인할 수 있게 만드는 방식입니다.
```

```markdown
# 제안 메시지 2: 네이버 플레이스 부족 매장용

안녕하세요. 네이버 플레이스 보고 연락드립니다.

현재 기본 정보는 있지만
고객이 예약 전에 궁금해할 가격, 소요시간, 준비사항, FAQ가 조금 더 정리되면 문의 전환에 도움이 될 것 같았습니다.

복잡한 홈페이지가 아니라
모바일에서 바로 보는 1페이지 홍보페이지를 제작하고 있습니다.

카카오톡 문의 버튼까지 연결해서
고객이 확인하고 바로 문의할 수 있는 구조입니다.
```

```markdown
# 제안 메시지 3: 체험가 제안용

초기 사례를 만들고 있어서
이번 달에는 체험가로 1페이지 홍보페이지를 제작해드리고 있습니다.

포함되는 내용은
대표 서비스, 가격 안내, 사진, FAQ, 카카오톡 문의 버튼, 운영 정보입니다.

오픈 후 1개월 동안은 간단한 문구 수정과 공지 수정도 무료로 도와드립니다.

대신 제작 전후 비교 사례로 활용할 수 있으면 좋겠습니다.
상호 공개 여부는 협의 가능합니다.
```

## 누적 지식에 추가할 메모

- 1개월 무료관리는 단순 서비스가 아니라 월관리 전환 근거를 수집하는 검증 기간으로 설계해야 한다.
- 월관리 상품은 수정 횟수, 이미지 교체 수, FAQ 업데이트 수, 리포트 제공 여부로 구분하면 설명이 쉽다.
- 직접수정 페이지라도 고객이 모든 수정을 직접 하지는 않으므로 수정 요청 양식과 대행 관리 옵션이 필요하다.
- 오픈 후 7일 점검은 링크, 모바일 화면, 문의 흐름, 정보 누락, 월관리 후보 항목을 기준으로 진행한다.
- 첫 영업 메시지는 “홈페이지 제작”보다 “가격, 예약, FAQ, 카톡문의 흐름 정리”를 앞세우는 편이 소상공인에게 더 실용적으로 전달된다.

# 추가 심화 라운드 9 / 경과 470초

## 라운드 9: 월관리 전환을 만드는 운영 설계

이번 라운드는 “무엇을 포함할지”가 아니라, **무료관리 1개월 동안 어떤 근거를 쌓아 월관리로 전환할지**에 집중합니다.

핵심은 고객에게 이렇게 느끼게 만드는 것입니다.

```markdown
페이지는 만들어졌지만,
장사가 돌아가려면 정보가 계속 바뀐다.

그리고 그 변경을 제때 반영해주는 사람이 있으면
문의가 덜 꼬이고, 사장님 시간이 줄어든다.
```

## 1. 무료관리 1개월을 월관리 전환 실험으로 설계하기

1개월 무료관리는 “서비스 제공”이 아니라 **전환 관찰 기간**으로 봐야 합니다.

무료관리 기간에 봐야 할 것은 디자인 만족도가 아닙니다.

```markdown
# 무료관리 1개월 관찰 항목

## 문의 관련
- 고객이 가장 많이 묻는 질문은 무엇인가?
- 페이지에 있는데도 다시 묻는 정보는 무엇인가?
- 문의 전에 가격을 확인하는 흐름이 있는가?
- 문의 버튼 문구가 충분히 구체적인가?

## 정보 변경 관련
- 한 달 안에 바뀐 공지가 있었는가?
- 가격, 시간, 일정, 예약 가능 여부가 바뀌었는가?
- 사진 교체 요청이 있었는가?
- 후기나 사례를 추가할 일이 있었는가?

## 사장님 업무 관련
- 같은 답변을 반복해서 보내고 있는가?
- 문의 응대 시간이 줄었는가?
- 수정 요청을 직접 처리하기 어려워했는가?
- 페이지를 직접 수정할 의지가 실제로 있었는가?

## 월관리 필요 신호
- “이것도 바꿔주세요” 요청이 2회 이상 발생
- 문의 내용 중 FAQ로 만들 수 있는 질문이 3개 이상 발생
- 가격/예약/공지 변경이 1회 이상 발생
- 사장님이 직접수정보다 대행을 선호
```

무료관리 종료 전에는 단순히 “연장하시겠어요?”라고 묻지 말고, 관찰된 내용을 근거로 제안해야 합니다.

```markdown
# 무료관리 종료 제안 구조

지난 1개월 동안 확인해보니
페이지 자체는 정상적으로 운영되고 있습니다.

다만 실제 운영 중에 아래 항목들이 계속 바뀌었습니다.

- 공지 수정:
- FAQ 추가 후보:
- 사진 교체 필요:
- 문의 문구 개선 필요:
- 반복 문의 내용:

그래서 앞으로는 직접 관리하셔도 되지만,
월 1회 이상 정보가 바뀌는 업종이라면
월관리로 맡기시는 쪽이 더 편하실 수 있습니다.
```

## 2. 업종별 월관리 전환 가능성 분류표

모든 업종에 같은 월관리 상품을 팔면 설득력이 약합니다.  
월관리 필요성은 “정보 변화 빈도”와 “문의 복잡도”에 따라 달라집니다.

```markdown
# 업종별 월관리 전환 가능성

## 전환 가능성 높음

### 예약제 뷰티
예시:
- 네일
- 속눈썹
- 피부관리
- 왁싱
- 두피관리

월관리 이유:
- 가격표 변경 가능성 있음
- 예약 가능 시간 안내 필요
- 시술 전 주의사항 반복 문의 많음
- 사진과 후기 업데이트 중요

추천 상품:
- 월 10만원 문의 개선형
- 월 15만원 성장 관리형

---

### 클래스/공방
예시:
- 원데이클래스
- 베이킹 클래스
- 플라워 클래스
- 도자기 공방

월관리 이유:
- 일정이 계속 바뀜
- 모집 마감 공지가 필요함
- 준비물, 소요시간, 인원 안내가 중요함
- 시즌 상품이 있음

추천 상품:
- 월 10만원 문의 개선형
- 월 15만원 성장 관리형

---

### 상담형 서비스
예시:
- 심리상담
- 퍼스널컬러
- 체형교정
- PT 상담
- 입시/진로 상담

월관리 이유:
- 고객이 문의 전에 불안 요소를 많이 확인함
- FAQ 품질이 문의 전환에 영향 큼
- 후기/사례 업데이트가 중요함
- 상담 절차 설명이 계속 개선될 수 있음

추천 상품:
- 월 15만원 성장 관리형

---

## 전환 가능성 중간

### 음식점/카페
월관리 이유:
- 메뉴 변경
- 휴무 공지
- 시즌 메뉴
- 단체 예약 안내

주의점:
- 단순 음식점은 월관리 설득이 약할 수 있음
- 예약제, 단체주문, 클래스, 케이터링이 있어야 유리함

추천 상품:
- 월 5만원 기본 관리형
- 월 10만원 문의 개선형

---

## 전환 가능성 낮음

### 정보 변화가 거의 없는 업종
예시:
- 단순 소개 목적 매장
- 가격 고정형 서비스
- 문의가 적은 업종
- 인스타/네이버만으로 충분한 매장

대응:
- 월관리 강매 금지
- 제작비 중심으로 접근
- 필요 시 수정 건별 과금 제안

추천 구조:
- 제작비 20만원
- 수정 건별 1~3만원
```

이 분류표는 영업 우선순위를 정할 때 중요합니다.  
초기에는 “페이지가 필요한 업종”보다 **월관리 전환 가능성이 높은 업종**을 먼저 공략해야 합니다.

## 3. 문의 버튼 카피 설계

카카오톡 버튼은 단순히 “문의하기”라고 쓰면 약합니다.  
사장님에게도 고객에게도 어떤 문의인지 불분명하기 때문입니다.

버튼 문구는 업종별로 구체화해야 합니다.

```markdown
# 카카오톡 문의 버튼 문구 예시

## 뷰티/관리 업종
- 예약 가능 시간 문의하기
- 시술 가격 상담하기
- 내게 맞는 관리 문의하기
- 첫 방문 상담하기
- 사진 보고 시술 상담하기

## 클래스/공방
- 클래스 일정 문의하기
- 남은 자리 확인하기
- 단체 클래스 문의하기
- 준비물 확인하기
- 예약 가능 날짜 문의하기

## 상담형 서비스
- 상담 가능 시간 문의하기
- 첫 상담 절차 문의하기
- 내 상황에 맞는 상담 문의하기
- 비용과 진행 방식 문의하기
- 상담 전 궁금한 점 물어보기

## 음식점/카페
- 단체 예약 문의하기
- 메뉴/가격 문의하기
- 대관 가능 여부 문의하기
- 케이크 주문 문의하기
- 포장/배달 문의하기
```

버튼 주변 문구도 중요합니다.

```markdown
# 버튼 위 안내 문구

## 나쁜 예
궁금한 점은 문의해주세요.

## 좋은 예
예약 가능 시간, 가격, 준비사항이 궁금하시면
카카오톡으로 바로 문의하실 수 있습니다.

## 더 좋은 예
원하시는 날짜와 서비스를 함께 보내주시면
가능 시간과 가격을 빠르게 안내해드립니다.
```

카카오톡 첫 자동응답 문구까지 같이 설계하면 월관리 상품의 가치가 올라갑니다.

```markdown
# 카카오톡 첫 응답 문구 템플릿

안녕하세요. 문의 감사합니다.

빠른 안내를 위해 아래 내용을 함께 보내주세요.

1. 원하시는 서비스:
2. 희망 날짜/시간:
3. 방문 인원:
4. 궁금한 점:

확인 후 가능한 시간과 가격을 안내드리겠습니다.
```

## 4. 직접수정 페이지의 관리자 화면 설계

직접수정 기능은 “사장님이 모든 것을 바꿀 수 있음”보다  
**사장님이 자주 바꾸는 것만 안전하게 바꿀 수 있음**이 중요합니다.

수정 가능한 영역과 수정 불가 영역을 분리해야 합니다.

```markdown
# 직접수정 가능 영역

## 기본 정보
- 매장명
- 한 줄 소개
- 영업시간
- 휴무일
- 주소
- 주차 안내

## 문의 정보
- 카카오톡 링크
- 전화번호
- 네이버 예약 링크
- 인스타그램 링크

## 공지
- 공지 제목
- 공지 내용
- 노출 여부
- 시작일
- 종료일

## 서비스/가격
- 서비스명
- 짧은 설명
- 가격
- 소요시간
- 예약 필요 여부

## FAQ
- 질문
- 답변
- 노출 여부

## 이미지
- 대표 이미지
- 서비스 이미지
- 후기 이미지
```

수정하지 못하게 막아야 할 영역도 정해야 합니다.

```markdown
# 직접수정 제한 영역

## 제한 권장
- 전체 레이아웃
- 글자 크기
- 색상 조합
- 버튼 디자인
- 섹션 순서
- 여백
- 반응형 구조
- 문의 동선 구조

## 제한 이유
- 사장님이 직접 바꾸면 화면이 깨질 가능성이 높음
- 모바일 가독성이 나빠질 수 있음
- 문의 버튼 위치가 흐트러질 수 있음
- 디자인 수정 요청이 월관리 범위를 침범할 수 있음
```

관리자 화면 문구는 쉬워야 합니다.

```markdown
# 관리자 화면 필드명 예시

나쁜 표현:
- Hero Title
- CTA URL
- Section Visibility
- Description Body

좋은 표현:
- 첫 화면 제목
- 문의 버튼 링크
- 화면에 보이기
- 서비스 설명
```

직접수정 상품의 핵심은 기능이 아니라 **수정 범위 통제**입니다.  
그래야 유지보수 부담이 폭발하지 않습니다.

## 5. 첫 고객 확보용 진단표

초기 영업은 무작정 메시지를 보내기보다, 매장별로 간단한 진단을 남기고 접근하는 편이 좋습니다.

```markdown
# 소상공인 홍보페이지 진단표

## 매장명
-

## 업종
-

## 현재 채널
- 인스타그램
- 네이버 플레이스
- 블로그
- 카카오채널
- 없음

## 현재 문제로 보이는 항목
- 가격 정보가 찾기 어려움
- 예약 방법이 불명확함
- 자주 묻는 질문이 없음
- 서비스별 차이가 설명되지 않음
- 사진은 많지만 정리된 안내가 없음
- 문의 버튼이 바로 보이지 않음
- 영업시간/휴무 정보가 부족함
- 고객 후기나 사례가 흩어져 있음

## 제안할 페이지 구성
- 첫 화면
- 대표 서비스
- 가격 안내
- 예약 방법
- FAQ
- 후기/사진
- 위치/운영 정보
- 카카오톡 문의 버튼

## 월관리 가능성
- 낮음
- 중간
- 높음

## 월관리 근거
- 공지가 자주 바뀔 것 같음
- 사진 업데이트가 중요함
- 예약 일정이 바뀜
- FAQ가 계속 추가될 가능성 있음
- 후기/사례 업데이트가 필요함

## 제안 메시지 발송 여부
- 발송 전
- 발송 완료
- 답변 옴
- 거절
- 상담 진행
- 계약
```

이 진단표는 Obsidian에서 고객 후보 DB로 쓸 수 있습니다.

```markdown
제안상태:
예상상품:
마지막연락일:
다음액션:

```

---

## 6. 체험가 고객을 유료 고객으로 바꾸는 종료 리포트

체험가 고객에게는 “잘 만들어드렸습니다”보다  
**운영 중 발견된 개선 포인트**를 보여줘야 합니다.

```markdown
# 1개월 무료관리 종료 리포트

## 고객명
-

## 운영 기간
-

## 반영한 수정
- 공지:
- 문구:
- FAQ:
- 이미지:
- 링크:
- 기타:

## 운영 중 발견한 문제
- 고객이 반복해서 묻는 질문:
- 페이지에 추가하면 좋은 정보:
- 문의 버튼 문구 개선 필요:
- 오래되면 위험한 정보:
- 업데이트가 필요한 사진/후기:

추천 상품:
- 기본 관리형 / 월 5만원
- 문의 개선형 / 월 10만원
- 성장 관리형 / 월 15만원

추천 이유:
-

- FAQ 추가:
- 공지 정리:
- 문의 문구 개선:
- 사진 교체:
- 후기/사례 업데이트:
```

고객에게 보내는 문구는 이렇게 정리할 수 있습니다.

```markdown
지난 1개월 동안 운영해보니
페이지를 한 번 만들어두는 것보다
공지, FAQ, 문의 문구를 계속 정리하는 쪽이 더 중요해 보였습니다.

특히 아래 항목은 다음 달에도 관리하면 좋겠습니다.

- 자주 묻는 질문 추가
- 예약/가격 안내 정리
- 공지 최신화
- 사진 또는 후기 업데이트

원하시면 다음 달부터는 월관리로 이어서 관리해드릴 수 있습니다.
```

## 7. 월관리 상품을 팔 때 피해야 할 표현

월관리는 “홈페이지 유지보수”라고 말하면 작게 느껴집니다.  
소상공인 입장에서는 기술 관리보다 장사에 도움 되는 말이 더 중요합니다.

```markdown
# 피해야 할 표현

- 유지보수
- 서버 관리
- 페이지 관리
- HTML 수정
- 월 구독
- 관리 대행

# 더 나은 표현

- 문의 흐름 관리
- 공지 최신화
- 가격/예약 안내 정리
- 자주 묻는 질문 정리
- 카카오톡 문의 전환 개선
- 고객이 헷갈리지 않게 정보 정리
```

비교하면 차이가 큽니다.

```markdown
나쁜 제안:
월 10만원에 홈페이지 유지보수해드립니다.

좋은 제안:
월 10만원에 공지, 가격, FAQ, 사진, 카카오톡 문의 문구를 정리해서
고객이 문의 전에 필요한 정보를 더 쉽게 확인할 수 있게 관리해드립니다.
```

## 누적 지식에 추가할 메모

- 무료관리 1개월은 고객 만족 기간이 아니라 월관리 필요성을 관찰하고 기록하는 전환 실험 기간으로 설계해야 한다.
- 월관리 전환 가능성은 업종보다 정보 변경 빈도, 문의 복잡도, 예약 흐름 유무에 따라 판단하는 것이 더 정확하다.
- 카카오톡 버튼은 “문의하기”보다 “예약 가능 시간 문의하기”, “남은 자리 확인하기”처럼 고객 행동을 구체화해야 전환 흐름이 좋아진다.
- 직접수정 페이지는 모든 영역을 열어주는 방식보다 공지, 가격, FAQ, 링크, 이미지처럼 안전한 영역만 수정 가능하게 제한해야 유지보수 부담이 줄어든다.
- 체험가 고객을 월관리로 전환하려면 무료관리 종료 시 “수정 내역”이 아니라 “운영 중 발견된 반복 문의와 다음 달 관리 필요 항목”을 리포트로 제시해야 한다.

## 1. 월관리 전환을 위한 “변경 빈도 지도”

월관리 상품은 “많이 수정해드림”이 아니라  
**바뀌는 정보를 안정적으로 관리해드림**으로 설계해야 합니다.

소상공인 페이지에서 월관리 전환 가능성이 높은 정보는 보통 아래 4가지입니다.

```markdown
# 변경 빈도 지도

## 거의 안 바뀌는 정보
- 매장 소개
- 대표 서비스 설명
- 위치
- 기본 운영시간
- 브랜드 문구

관리 전략:
- 제작 시 완성도를 높인다.
- 월관리 범위의 핵심으로 팔지 않는다.

## 가끔 바뀌는 정보
- 가격표
- 시술/서비스 구성
- 대표 사진
- 후기
- FAQ

관리 전략:
- 월 1회 점검 항목으로 넣는다.
- 기본 관리형 또는 문의 개선형에 포함한다.

## 자주 바뀌는 정보
- 휴무 공지
- 예약 가능 시간
- 이벤트
- 시즌 메뉴
- 재고/수업 일정
- 프로모션

관리 전략:
- 월관리 핵심 가치로 제안한다.
- “고객이 헷갈리지 않게 최신 상태 유지”로 설명한다.

## 문의에 따라 계속 늘어나는 정보
- 자주 묻는 질문
- 예약 전 확인사항
- 환불/변경 규정
- 준비물
- 방문 전 주의사항

관리 전략:
- 카카오톡 문의 내용을 FAQ로 전환한다.
- 월관리 리포트의 핵심 근거로 쓴다.
```

이 지도는 고객 상담 때 바로 쓸 수 있습니다.

```markdown
사장님 페이지에서 월관리 필요성이 있는 부분은
전체 디자인 수정이 아니라 자주 바뀌는 정보입니다.

특히 휴무, 예약 가능 시간, 이벤트, FAQ는
오래된 상태로 남아 있으면 고객 문의가 꼬일 수 있어서
월 1회 정도 정리해두는 게 좋습니다.
```

## 2. 직접수정 관리자 화면의 입력 구조

직접수정 페이지는 필드가 많아질수록 사장님이 어려워합니다.  
관리자 화면은 “웹사이트 편집기”가 아니라 **매장 정보 입력표**처럼 보여야 합니다.

```markdown
# 관리자 화면 기본 구조

## 1. 오늘의 공지
필드:
- 공지 제목
- 공지 내용
- 공지 보이기 / 숨기기
- 공지 종료일

수정 가능:
- 텍스트
- 노출 여부
- 종료일

수정 불가:
- 공지 위치
- 글자 크기
- 배경 색상
```

```markdown
## 2. 대표 서비스

필드:
- 서비스명
- 짧은 설명
- 가격
- 소요 시간
- 예약 전 확인사항
- 대표 이미지

수정 가능:
- 서비스 텍스트
- 가격
- 이미지 교체

수정 불가:
- 카드 개수 무제한 추가
- 카드 레이아웃
- 버튼 위치
```

```markdown
## 3. FAQ

필드:
- 질문
- 답변
- 화면에 보이기

운영 규칙:
- 최대 8개까지 노출
- 긴 답변은 3~4줄 권장
- 환불/예약/주차/소요시간 관련 질문 우선
```

중요한 점은 “사장님이 마음대로 꾸미는 화면”이 아니라  
**정해진 구조 안에서 장사 정보를 바꾸는 화면**이어야 한다는 것입니다.

## 3. 첫 고객용 제안서 1페이지 구조

초기 영업에서는 긴 제안서보다  
“문제 발견 → 개선안 → 체험 제안”이 한 장에 보여야 합니다.

```markdown
# 소상공인 홍보페이지 개선 제안서

## 1. 현재 고객이 헷갈릴 수 있는 부분

매장명:
업종:
확인 채널:

발견한 문제:
- 가격 정보를 찾기 어려움
- 예약 방법이 바로 보이지 않음
- 카카오톡 문의 버튼이 눈에 잘 띄지 않음
- 자주 묻는 질문이 정리되어 있지 않음
- 사진은 있지만 서비스별 설명이 부족함

## 2. 제안하는 페이지 구성

- 첫 화면: 어떤 매장인지 바로 이해
- 대표 서비스: 가격/소요시간/특징 정리
- 예약 안내: 문의 전 필요한 정보 정리
- FAQ: 반복 질문 줄이기
- 후기/사진: 신뢰 보강
- 카카오톡 문의 버튼: 바로 문의 가능

## 3. 체험 제안

제작비:
- 체험가 15만원

포함:
- 홍보페이지 1개 제작
- 모바일 최적화
- 카카오톡 문의 버튼 연결
- 1개월 무료관리

1개월 후 선택:
- 종료
- 기본 관리형 월 5만원
- 문의 개선형 월 10만원
- 성장 관리형 월 15만원
```

짧은 DM 문구는 이렇게 쓸 수 있습니다.

```markdown
안녕하세요. 매장 안내 페이지 관점에서 채널을 살펴보다가
고객이 예약 전에 확인하면 좋을 정보가 조금 흩어져 보여 연락드렸습니다.

가격, 예약 방법, FAQ, 카카오톡 문의 버튼을 한 페이지에 정리하면
문의 전환이 더 깔끔해질 것 같아 간단한 개선안을 만들어봤습니다.

원하시면 체험가로 홍보페이지 제작 + 1개월 무료관리까지 진행해드릴 수 있습니다.
```

## 4. 월관리 상품별 작업 한도 명세

월관리에서 가장 위험한 것은  
고객이 “월관리니까 아무 수정이나 가능하다”고 이해하는 것입니다.

처음부터 작업 한도를 문서로 고정해야 합니다.

```markdown
# 월관리 상품 작업 범위

## 기본 관리형 / 월 5만원

목적:
- 페이지 정보가 오래되지 않게 유지

포함:
- 월 1회 수정
- 공지 문구 수정
- 운영시간/휴무일 수정
- 가격표 일부 수정
- 카카오톡 링크 확인

제외:
- 디자인 변경
- 섹션 추가
- 전체 문구 재작성
- 새 페이지 제작
```

```markdown
## 문의 개선형 / 월 10만원

목적:
- 고객 문의 전에 필요한 정보를 더 잘 보이게 정리

포함:
- 월 2회 수정
- FAQ 추가/정리
- 가격/예약 안내 문구 개선
- 문의 버튼 문구 개선
- 후기/사진 일부 교체
- 월 1회 간단 리포트

제외:
- 광고 운영
- 블로그 작성
- 상세페이지 추가 제작
- 브랜드 디자인 개편
```

```markdown
## 성장 관리형 / 월 15만원

목적:
- 페이지를 지속적으로 개선해 문의 흐름을 관리

포함:
- 월 4회 수정
- 이벤트/공지 업데이트
- FAQ 지속 보강
- 서비스 설명 개선
- 후기/사례 업데이트
- 월 1회 개선 리포트
- 다음 달 개선 항목 제안

제외:
- 촬영
- 광고비
- 외부 채널 운영 대행
- 대규모 리뉴얼
```

계약 전 안내 문구는 이렇게 정리합니다.

```markdown
월관리는 홈페이지를 매번 새로 디자인하는 서비스가 아니라
고객이 보는 공지, 가격, FAQ, 사진, 문의 문구를 최신 상태로 정리하는 서비스입니다.

디자인 변경이나 새 섹션 제작은 별도 작업으로 안내드립니다.
```

## 누적 지식에 추가할 메모

- 월관리 전환 가능성은 “수정량”보다 “정보 변경 빈도”와 “오래된 정보가 문의에 주는 영향”으로 판단해야 한다.

- Obsidian은 고객 후보 저장소가 아니라 진단, 제안, 계약, 관리 기록, 종료 리포트까지 이어지는 영업 운영 DB로 설계해야 한다.

## 1. “직접수정 홍보페이지”의 상품 경계 명세

```markdown
고객이 예약/문의 전에 궁금해하는 정보를 한 페이지에 정리하고

사장님은 운영에 자주 바뀌는 정보만 직접 수정할 수 있습니다.

직접 수정 가능:
- 공지
- 운영시간
- 가격
- 대표 메뉴/서비스
- 자주 묻는 질문
- 예약 전 안내
- 이벤트 문구

직접 수정 불가:
- 전체 디자인
- 색상 체계
- 레이아웃
- 버튼 위치
- 폰트
- 섹션 구조
- 애니메이션
```

이렇게 나눠야 나중에 월관리 고객이 “제가 디자인도 바꾸고 싶은데요”라고 할 때 자연스럽게 추가 작업으로 분리할 수 있습니다.

## 2. 고객 입력 필드 설계표

관리자 화면을 만들기 전에 고객이 직접 수정할 필드를 먼저 정해야 합니다.  
필드가 많으면 고객도 헷갈리고, 페이지 품질도 무너집니다.

```markdown
# 직접수정 필드 명세

## 1. 기본 정보
| 필드명 | 입력 방식 | 제한 | 페이지 반영 위치 |
|---|---|---|---|
| 매장명 | 텍스트 | 20자 이내 | 상단 제목 |
| 한 줄 소개 | 텍스트 | 40자 이내 | 첫 화면 설명 |
| 전화번호 | 텍스트 | 숫자/하이픈 | 문의 영역 |
| 카카오톡 링크 | URL | 필수 | 고정 문의 버튼 |
| 주소 | 텍스트 | 80자 이내 | 위치 안내 |

## 2. 운영 정보
| 필드명 | 입력 방식 | 제한 | 페이지 반영 위치 |
|---|---|---|---|
| 영업시간 | 요일별 텍스트 | 200자 이내 | 운영 안내 |
| 휴무일 | 텍스트 | 50자 이내 | 운영 안내 |
| 주차 안내 | 텍스트 | 100자 이내 | 위치 안내 |
| 예약 가능 시간 | 텍스트 | 100자 이내 | 예약 안내 |

## 3. 상품/서비스 정보
| 필드명 | 입력 방식 | 제한 | 페이지 반영 위치 |
|---|---|---|---|
| 서비스명 | 반복 입력 | 최대 8개 | 가격표 |
| 가격 | 텍스트 | 항목별 20자 이내 | 가격표 |
| 설명 | 텍스트 | 항목별 60자 이내 | 가격표 |
| 소요시간 | 텍스트 | 항목별 20자 이내 | 가격표 |

## 4. 공지/이벤트
| 필드명 | 입력 방식 | 제한 | 페이지 반영 위치 |
|---|---|---|---|
| 공지 제목 | 텍스트 | 30자 이내 | 공지 영역 |
| 공지 내용 | 긴 텍스트 | 200자 이내 | 공지 영역 |
| 노출 여부 | 토글 | 켜기/끄기 | 공지 영역 |

## 5. FAQ
| 필드명 | 입력 방식 | 제한 | 페이지 반영 위치 |
|---|---|---|---|
| 질문 | 텍스트 | 50자 이내 | FAQ |
| 답변 | 텍스트 | 150자 이내 | FAQ |
| 노출 순서 | 숫자 | 1~10 | FAQ |
```

관리자 화면에서는 “자유 편집”보다 “정해진 칸 채우기”가 좋습니다.  
소상공인은 디자인을 수정하고 싶은 것이 아니라, 바뀐 장사 정보를 빨리 반영하고 싶은 경우가 많습니다.

## 3. 첫 제작 전 고객에게 받는 자료 요청서

초기 제작 시간을 줄이려면 상담 후 바로 자료 요청서를 보내야 합니다.  
자료가 흩어져 있으면 제작자가 계속 물어보게 되고, 고객도 피곤해집니다.

```markdown
# 홍보페이지 제작 자료 요청서

사장님 페이지 제작을 위해 아래 자료를 보내주세요.

## 1. 기본 정보
- 매장명:
- 주소:
- 전화번호:
- 카카오톡 문의 링크:
- 인스타그램 또는 블로그 링크:
- 영업시간:
- 휴무일:
- 주차 가능 여부:

## 2. 대표 서비스
아래 형식으로 5~8개 정도 보내주세요.

예시:
- 젤네일 기본: 40,000원 / 약 60분 / 컬러 1가지 기준
- 원데이 클래스: 45,000원 / 약 2시간 / 재료비 포함
- 첫 방문 커트: 25,000원 / 약 40분 / 예약 필수

## 3. 고객이 자주 묻는 질문
예시:
- 당일 예약 가능한가요?
- 주차 가능한가요?
- 예약금이 있나요?
- 아이와 함께 방문 가능한가요?
- 소요시간은 얼마나 걸리나요?

## 4. 사진
가능하면 아래 사진을 보내주세요.

- 외관 사진 1장
- 내부 사진 2~3장
- 대표 상품/시술/작품 사진 5장 이상
- 가격표가 있다면 사진 또는 텍스트
- 위치 설명에 도움이 되는 사진

## 5. 강조하고 싶은 내용
아래 중 해당되는 내용을 알려주세요.

- 첫 방문 할인
- 예약제 운영
- 1인샵
- 프라이빗 공간
- 단체 예약 가능
- 키즈 가능
- 주차 가능
- 당일 예약 가능
```

자료 요청서는 구글폼, 카카오톡 메시지, 노션 폼 중 하나로 시작하면 됩니다.  
처음부터 복잡한 고객 포털을 만들 필요는 없습니다.

## 4. 카카오톡 문의 전환 문구 설계

페이지의 목표는 예쁜 소개가 아니라 문의입니다.  
그래서 버튼 문구도 “문의하기” 하나로 끝내지 말고, 고객 상황별로 나눠야 합니다.

```markdown
# 카카오톡 문의 버튼 문구

## 기본형
- 카카오톡으로 문의하기
- 예약 가능 시간 물어보기
- 가격 상담하기
- 방문 전 궁금한 점 묻기

## 네일샵
- 이달의 아트 예약 문의
- 제거 포함 가격 물어보기
- 당일 예약 가능 여부 확인

## 공방
- 클래스 일정 문의
- 단체 수업 가능 여부 묻기
- 커플 클래스 예약 문의

## 미용실
- 시술 상담하기
- 원하는 스타일 사진 보내기
- 첫 방문 예약 문의

## 피부관리실
- 내 피부에 맞는 관리 상담
- 첫 방문 프로그램 문의
- 예약 가능 시간 확인

## 필라테스
- 체험 수업 문의
- 시간표 확인하기
- 개인/그룹 수업 상담
```

버튼은 페이지에 최소 3곳 이상 배치합니다.

```markdown
# 문의 버튼 배치 기준

1. 첫 화면
- 방문자가 바로 문의할 수 있어야 함
- 문구: 카카오톡으로 예약 문의

2. 가격표 아래
- 가격 확인 후 바로 문의 가능
- 문구: 이 서비스로 문의하기

3. FAQ 아래
- 의문 해소 후 행동 유도
- 문구: 더 궁금한 점 카톡으로 묻기

4. 모바일 하단 고정 버튼
- 스크롤 중 언제든 문의 가능
- 문구: 카톡 문의
```

## 5. 1개월 무료관리 기간의 전환 설계

무료관리는 그냥 “한 달 봐드릴게요”가 되면 월관리로 이어지기 어렵습니다.  
처음부터 월관리 전환을 위한 관찰 항목을 정해야 합니다.

```markdown
# 1개월 무료관리 운영표

## 목적
무료관리 기간 동안 사장님의 수정 요청 패턴과 문의 개선 가능성을 확인하고,
월관리 상품으로 전환할 근거를 만든다.

## 관찰 항목
| 항목 | 기록 내용 |
|---|---|
| 수정 요청 횟수 | 한 달 동안 몇 번 요청했는가 |
| 수정 요청 유형 | 가격, 공지, 사진, 운영시간, 이벤트, FAQ |
| 긴급 요청 여부 | 당일 반영 요청이 있었는가 |
| 문의 증가 체감 | 사장님이 문의 변화를 느꼈는가 |
| 고객 질문 반복 | 같은 질문이 계속 들어오는가 |
| 추가 개선 가능성 | FAQ, 가격표, 사진, 버튼 문구 개선 여지 |

## 무료관리 종료 7일 전 메시지
사장님, 이번 달 무료관리 기간 동안
운영시간/가격/공지/문의 안내를 몇 차례 정리해드렸습니다.

앞으로도 이런 정보가 자주 바뀐다면
월 5만원 관리형으로 유지하시는 편이 편합니다.

월관리에는 아래 작업이 포함됩니다.

- 월 2회 정보 수정
- 공지/이벤트 문구 반영
- 가격표 일부 수정
- FAQ 추가/수정
- 카카오톡 문의 버튼 점검
```

월관리 전환 제안은 감정적으로 “계속 맡겨주세요”가 아니라,  
한 달 동안 실제로 발생한 수정 기록을 근거로 해야 합니다.

```markdown
# 월관리 전환 제안 템플릿

사장님, 무료관리 기간 동안 총 {수정횟수}회 수정이 있었습니다.

주로 바뀐 내용은
{주요수정유형} 쪽이었습니다.

이 정도면 페이지를 만들어두고 끝내기보다
매달 조금씩 관리하면서 최신 상태를 유지하는 편이 좋습니다.

기본 관리형은 월 {금액}원이고,
월 {수정한도}회까지 반영해드립니다.
```

---

## 누적 지식에 추가할 메모

- 직접수정 홍보페이지는 자유 편집 도구가 아니라, 소상공인의 장사 정보만 안전하게 바꾸는 제한형 정보 관리 도구로 정의해야 한다.
- 고객 입력 필드는 매장명, 운영시간, 가격, 공지, FAQ, 카카오톡 링크처럼 문의 전환에 직접 영향을 주는 항목으로 제한한다.
- 제작 전 자료 요청서를 표준화하면 상담 이후의 반복 질문이 줄고, 첫 제작 시간을 예측 가능하게 만들 수 있다.
- 카카오톡 버튼 문구는 업종과 고객 상황에 맞게 달라져야 하며, 첫 화면·가격표 아래·FAQ 아래·모바일 하단에 반복 배치하는 것이 좋다.
- 1개월 무료관리는 월관리 전환을 위한 수정 패턴 관찰 기간이며, 종료 제안은 실제 수정 횟수와 요청 유형을 근거로 해야 한다.

# 라운드 13: 월 100~150만원 모델을 위한 운영형 상품 설계

이번 라운드는 “페이지를 만들어준다”에서 한 단계 더 들어가서, **반복 판매 가능한 상품 구조**와 **초기 고객 확보용 영업 시스템**을 설계합니다.

---

## 1. 업종별 최소 패키지 구조

소상공인에게 처음부터 “맞춤 제작”이라고 말하면 상담 범위가 커집니다.  
반대로 **업종별 기본 패키지**로 제안하면 고객은 이해하기 쉽고, 제작자는 반복 작업으로 처리할 수 있습니다.

```markdown
# 업종별 홍보페이지 기본 패키지

## 네일샵 패키지
### 필수 섹션
- 첫 화면: 매장명, 위치, 대표 강점
- 이달의 아트/대표 시술
- 가격표
- 예약 전 안내
- 자주 묻는 질문
- 카카오톡 예약 문의

### 핵심 전환 문구
- 이달의 아트 예약 문의
- 제거 포함 가격 물어보기
- 당일 예약 가능 여부 확인

### 고객에게 꼭 받아야 할 자료
- 시술 사진 8장 이상
- 가격표
- 예약 가능 시간
- 제거/연장/케어 추가 비용
```

```markdown
## 공방/클래스 패키지
### 필수 섹션
- 첫 화면: 클래스명, 지역, 대상
- 클래스 종류
- 수업 시간/가격
- 완성 작품 사진
- 단체/커플/기업 출강 안내
- 카카오톡 일정 문의

### 핵심 전환 문구
- 클래스 일정 문의
- 커플 클래스 예약 문의
- 단체 수업 가능 여부 묻기

### 고객에게 꼭 받아야 할 자료
- 완성 작품 사진 6장 이상
- 수업 공간 사진
- 수업 소요 시간
- 최소/최대 인원
- 재료비 포함 여부
```

```markdown
## 피부관리실/1인샵 패키지
### 필수 섹션
- 첫 화면: 관리실 강점, 위치, 예약제 여부
- 대표 프로그램
- 첫 방문 추천 코스
- 가격 안내
- 방문 전 주의사항
- 카카오톡 상담 문의

### 핵심 전환 문구
- 내 피부에 맞는 관리 상담
- 첫 방문 프로그램 문의
- 예약 가능 시간 확인

### 고객에게 꼭 받아야 할 자료
- 공간 사진
- 관리 프로그램명
- 가격/소요 시간
- 첫 방문 추천 메뉴
- 예약 전 주의사항
```

핵심은 업종별로 페이지 구조를 새로 고민하지 않는 것입니다.  
처음에는 **네일샵, 공방, 피부관리실, 필라테스, 미용실** 정도만 템플릿화하면 충분합니다.

## 2. 상품 가격표와 월 100~150만원 조합

월 100~150만원을 만들려면 단가와 반복 수익을 같이 봐야 합니다.  
처음부터 고가 웹사이트 제작으로 접근하기보다, **초기 제작비 + 월관리비** 조합이 현실적입니다.

```markdown
# 가격 패키지 초안

## 스타터형
가격: 29만원
대상: 처음 홍보페이지가 필요한 1인샵/소규모 매장

포함:
- 모바일 중심 홍보페이지 1개
- 기본 섹션 5개
- 카카오톡 문의 버튼
- 가격표/FAQ 반영
- 1개월 무료관리

제외:
- 로고 제작
- 전문 촬영
- 블로그/인스타 운영
- 광고 대행
```

```markdown
## 표준형
가격: 49만원
대상: 사진과 서비스 구성이 어느 정도 있는 매장

포함:
- 모바일 중심 홍보페이지 1개
- 기본 섹션 7개
- 업종별 문의 문구 설계
- FAQ 5개 정리
- 공지/이벤트 영역
- 1개월 무료관리
- 월관리 전환 상담

추천 판매 상품으로 둔다.
```

```markdown
## 관리형
가격: 월 5만원
```

## 3. 첫 고객 확보용 영업 리스트 구조

첫 고객은 광고보다 직접 접촉이 빠릅니다.  
다만 아무 매장이나 연락하면 반응률이 낮습니다. **페이지 필요성이 보이는 매장**을 골라야 합니다.

```markdown
# 첫 고객 후보 선별 기준

## 우선순위 높음
- 인스타그램은 운영하지만 링크용 소개 페이지가 없음
- 가격표가 피드 이미지에만 있음
- 카카오톡 문의 링크가 찾기 어려움
- 네이버 플레이스 정보와 인스타 정보가 다름
- 공지, 휴무, 이벤트가 자주 바뀌는 업종
- 1인샵, 예약제, 클래스형 매장

## 우선순위 낮음
- 이미 잘 만든 홈페이지가 있음
- 대형 프랜차이즈
- 가격 공개를 아예 하지 않는 업종
- 온라인 문의보다 전화 중심인 매장
```

```markdown
# 영업 후보 기록표

| 매장명 | 업종 | 지역 | 현재 채널 | 문제점 | 제안 포인트 | 연락일 | 반응 | 다음 액션 |
|---|---|---|---|---|---|---|---|---|
| 예: OO네일 | 네일샵 | 마포 | 인스타 | 가격표 찾기 어려움 | 예약 문의 버튼 정리 | 6/8 | 미응답 | 3일 후 재연락 |
```

영업 대상은 하루에 100곳이 아니라,  
**문제가 명확히 보이는 10곳**을 고르는 쪽이 낫습니다.

## 4. 첫 제안 메시지 카피

처음 연락할 때는 “홈페이지 만들어드립니다”보다,  
상대 매장의 현재 문제를 짚어주는 방식이 반응률이 좋습니다.

```markdown
# 첫 DM 템플릿: 인스타용

안녕하세요 사장님.

인스타 보고 연락드렸습니다.
시술/상품 사진은 잘 올라와 있는데,
처음 보는 손님 입장에서는 가격표, 예약 방법, 위치 안내를 한 번에 보기 조금 어려울 수 있어 보여서요.

제가 소상공인용으로
카카오톡 문의까지 바로 연결되는 모바일 홍보페이지를 제작하고 있습니다.

예를 들면 아래 내용을 한 페이지에 정리합니다.

- 대표 서비스/가격
- 예약 전 안내
- 자주 묻는 질문
- 위치/주차 안내
- 카카오톡 문의 버튼

처음 제작 후 1개월은 무료로 수정 관리해드리고,
이후 필요하시면 월관리로 유지하는 방식입니다.

혹시 지금 인스타 링크에 연결할 간단한 홍보페이지가 필요하시면
예시 화면 보내드려도 될까요?
```

```markdown
# 첫 DM 템플릿: 문제 지적형

안녕하세요 사장님.

현재 인스타에서 가격과 예약 안내를 확인해보니,
손님이 문의 전에 궁금해할 내용들이 여러 게시물에 나뉘어 있는 것 같았습니다.

이런 경우에는
“가격표 + 예약 안내 + FAQ + 카톡 문의”만 정리된 모바일 페이지가 있으면
반복 질문을 줄이는 데 도움이 됩니다.

제가 이런 형태의 소상공인 홍보페이지를 제작하고 있습니다.

부담 없는 기본형은 29만원이고,
제작 후 1개월은 무료로 수정 관리해드립니다.

필요하시면 사장님 매장 기준으로
어떤 구성으로 만들 수 있을지 간단히 정리해서 보내드릴게요.
```

```markdown
# 재연락 메시지

사장님, 며칠 전에 모바일 홍보페이지 관련해서 메시지드렸습니다.

혹시 지금 바로 제작 계획이 없으셔도 괜찮습니다.
다만 가격표, 예약 안내, 자주 묻는 질문을 한 페이지로 정리해두면
인스타/블로그/네이버 플레이스에서 들어온 손님을 카톡 문의로 연결하기가 쉬워집니다.

원하시면 사장님 업종 기준 예시 구성만 짧게 보내드리겠습니다.
```

재연락은 1회까지만 하는 편이 좋습니다.  
반응 없는 고객을 계속 쫓기보다, 후보 리스트를 넓히는 것이 더 효율적입니다.

## 5. 제작 전 진단 체크리스트

상담 전에 진단표를 만들면 고객에게 더 전문가처럼 보입니다.  
또한 제작 범위를 정리하는 기준이 됩니다.

```markdown
# 홍보페이지 필요도 진단 체크리스트

## 기본 정보
- [ ] 매장명, 주소, 전화번호가 명확히 보이는가
- [ ] 카카오톡 문의 링크가 쉽게 보이는가
- [ ] 영업시간과 휴무일이 최신 상태인가
- [ ] 주차/방문 안내가 정리되어 있는가

## 가격/서비스 정보
- [ ] 대표 서비스가 한눈에 보이는가
- [ ] 가격이 확인 가능한가
- [ ] 소요 시간이 안내되어 있는가
- [ ] 첫 방문 고객이 무엇을 선택해야 할지 알 수 있는가

## 문의 전환
- [ ] 문의 버튼이 첫 화면에 있는가
- [ ] 가격 확인 후 바로 문의할 수 있는가
- [ ] 자주 묻는 질문이 정리되어 있는가
- [ ] 예약 전 주의사항이 안내되어 있는가

## 운영 효율
- [ ] 반복 질문이 많은 항목이 있는가
- [ ] 공지/이벤트가 자주 바뀌는가
- [ ] 가격표 수정이 자주 필요한가
- [ ] 사장님이 직접 수정하고 싶은 정보가 명확한가
```

```markdown
# 진단 결과 문구

현재 채널을 보면 사진과 서비스는 잘 보여지고 있지만,
처음 방문하는 고객이 가격, 예약 방법, 위치, 자주 묻는 질문을 한 번에 확인하기는 조금 어렵습니다.

그래서 새 홈페이지를 크게 만드는 것보다,
카카오톡 문의로 이어지는 모바일 홍보페이지를 먼저 만드는 편이 적합해 보입니다.
```

```markdown
페이지 링크:
{페이지링크}

- 위치/주차 안내
```

## 누적 지식에 추가할 메모

- 수익화를 위해서는 “맞춤 홈페이지”가 아니라 업종별 반복 판매가 가능한 홍보페이지 패키지로 정의해야 한다.
- 월 100~150만원은 제작비만으로 만들기보다, 초기 제작비와 월관리비를 조합해야 안정적이다.
- 첫 고객은 무작위 영업보다 가격표, 예약 안내, 카카오톡 링크가 흩어져 있는 매장을 선별하는 방식이 효율적이다.
- 첫 제안 메시지는 “홈페이지 제작”보다 “반복 질문을 줄이고 카톡 문의로 연결하는 페이지”라는 문제 해결 관점으로 써야 한다.
- 제작 완료 후에는 링크만 전달하지 말고, 수정 요청 양식과 무료관리 범위를 함께 안내해야 월관리 전환 가능성이 높아진다.

## 4. 월관리 전환을 위한 운영 리포트 구조

1개월 무료관리 후 월관리로 전환하려면 “계속 관리해야 할 이유”가 보여야 합니다.  
단순히 “수정 필요하면 월 5만원입니다”라고 말하면 전환율이 낮습니다.

무료관리 종료 전에 간단한 운영 리포트를 보내는 편이 좋습니다.

```markdown

사장님, 홍보페이지 무료관리 기간이 곧 종료됩니다.

지난 1개월 동안 페이지 운영 기준으로 보면
아래 항목은 계속 관리하면 좋은 부분입니다.

## 1. 현재 페이지에 정리된 내용
- 대표 서비스/가격
- 예약 전 안내
- 위치/주차 안내
- 자주 묻는 질문
- 카카오톡 문의 버튼

## 2. 앞으로 자주 바뀔 가능성이 있는 항목
- 가격표
- 휴무일
- 이벤트/공지
- 시술 사진
- FAQ 문구
- 예약 안내 문구

## 3. 월관리 시 가능한 작업
- 월 2회 텍스트 수정
- 가격표 일부 수정
- 공지 추가/삭제
- 사진 일부 교체
- 카카오톡 링크 점검
- 모바일 화면 이상 여부 확인

## 4. 추천 관리 방식
사장님 매장은 가격표와 공지 변경 가능성이 있어서
월 5만원 기본 관리형으로 유지하시는 것을 추천드립니다.

항상 최신 정보를 보여줄 수 있습니다.
```

```markdown
# 제작 자료 요청 양식

사장님, 페이지 제작을 위해 아래 내용만 보내주시면 됩니다.
전부 완벽하지 않아도 괜찮고, 비어 있는 항목은 제가 정리하면서 다시 확인드리겠습니다.

## 1. 매장 기본 정보
- 매장명:
- 업종:
- 주소:
- 영업시간:
- 휴무일:
- 주차 가능 여부:
- 카카오톡 문의 링크:
- 인스타그램 주소:
- 네이버플레이스 주소:

## 2. 대표 서비스
아래 형식으로 3~6개 정도 보내주세요.

예시:
- 젤네일 기본: 40,000원
- 속눈썹 펌: 35,000원
- 1:1 체험 수업: 30,000원

## 3. 가격표
현재 사용하는 가격표 이미지가 있으면 보내주세요.
없으면 텍스트로 보내주셔도 됩니다.

## 4. 고객이 자주 묻는 질문
예시:
- 당일 예약 가능한가요?
- 주차 가능한가요?
- 소요 시간은 얼마나 걸리나요?
- 첫 방문 할인 있나요?

## 5. 사진
아래 사진 중 가능한 것만 보내주세요.

- 매장 외부 사진 1장
- 매장 내부 사진 2~3장
- 대표 서비스 사진 3~6장
- 후기 캡처 또는 리뷰 이미지 1~3장

## 6. 꼭 안내해야 하는 내용
예약 전 손님이 알아야 하는 주의사항이 있다면 적어주세요.

예시:
- 예약 변경은 하루 전까지 가능합니다.
- 노쇼 시 재예약이 어려울 수 있습니다.
- 주말은 예약이 빨리 마감됩니다.
```

## 3. 업종별 문제 진단표

DM을 보낼 때 “홈페이지 만들어드립니다”보다 강한 문장은  
“지금 고객이 문의 전에 막히는 지점”을 짚는 문장입니다.

```markdown
# 업종별 홍보페이지 진단표

| 업종 | 고객이 문의 전 확인하는 정보 | 흔한 문제 | 제안 포인트 |
|---|---|---|---|
| 네일샵 | 가격, 디자인 예시, 소요 시간, 위치 | 인스타 사진은 많은데 가격 확인이 어려움 | 가격표와 디자인 예시를 한 페이지에 정리 |
| 속눈썹/왁싱 | 가격, 예약 가능 시간, 주의사항 | 처음 방문 고객이 시술 전 안내를 모름 | 예약 전 안내와 FAQ 강화 |
| 필라테스 | 체험 수업 가격, 위치, 강사, 시간표 | 프로그램 차이를 이해하기 어려움 | 체험 문의용 페이지 구성 |
| 공방 | 클래스 종류, 가격, 소요 시간, 예약 방법 | 인스타 피드에서 클래스 정보를 찾기 어려움 | 클래스별 가격/소요시간 정리 |
| 피부관리샵 | 프로그램, 가격, 전후 안내, 상담 방식 | 메뉴명이 많아 초보 고객이 선택하기 어려움 | 추천 코스와 상담 유도 구조 |
| 미용실 | 가격, 디자이너, 시술 시간, 예약 링크 | 가격표가 이미지로 흩어져 있음 | 대표 시술 가격과 예약 전 안내 정리 |
```

### 진단 질문

```markdown
# 사장님 페이지 진단 질문

1. 인스타그램을 보고 온 고객이 가격을 바로 확인할 수 있나요?
2. 처음 방문하는 고객이 위치와 주차 여부를 쉽게 알 수 있나요?
3. 예약 전 주의사항이 한곳에 정리되어 있나요?
4. 대표 서비스와 가격이 모바일에서 읽기 쉽게 보이나요?
5. 고객이 궁금증을 확인한 직후 바로 카톡 문의를 할 수 있나요?

3개 이상 “아니오”라면
카톡문의 연결형 홍보페이지를 만들었을 때 개선 여지가 있습니다.
```

## 4. 첫 상담용 제안서 구조

상담에서 너무 많은 설명을 하면 고객이 피로해집니다.  
첫 상담 제안서는 1페이지 수준으로 짧게 가져가는 편이 좋습니다.

```markdown
# 첫 상담 제안서 템플릿

## 사장님 매장에 필요한 페이지

현재 고객은 인스타그램/네이버를 보고 들어온 뒤
아래 정보를 확인하려고 합니다.

- 어떤 서비스가 있는지
- 가격이 어느 정도인지
- 위치와 영업시간은 어떤지
- 예약 전에 알아야 할 내용이 있는지
- 어디로 문의하면 되는지

그래서 이번 페이지는 단순 소개용 홈페이지가 아니라
카카오톡 문의로 이어지는 모바일 홍보페이지로 구성하는 것이 좋습니다.

## 제작 구성

- 첫 화면: 매장 핵심 소개와 카톡문의 버튼
- 서비스/가격: 대표 메뉴와 가격 정리
- 예약 전 안내: 주의사항, 소요 시간, 변경 규정
- 사진 영역: 대표 시술/공간 사진
- FAQ: 자주 묻는 질문
- 위치/문의: 주소, 지도, 카카오톡 연결

## 제작 비용

기본 제작비: 29만원  
첫 진행 한정가: 19만원  
포함 사항: 제작 + 카카오톡 문의 연결 + 1개월 무료관리

## 제작 일정

자료 전달 후 3~5일 내 초안 전달  
초안 확인 후 1회 수정  
완료 후 인스타그램 프로필 링크에 연결 가능

## 이후 관리

제작 후 1개월은 무료로 관리해드립니다.  
이후 가격표, 공지, 휴무일, 사진 변경이 필요하면  
월 5만원 관리형으로 이어가실 수 있습니다.
```

## 5. 직접수정형 페이지의 편집 권한 설계

“사장님이 직접 수정 가능”은 강력한 장점이지만, 모든 것을 수정하게 하면 화면이 망가질 수 있습니다.  
따라서 고객이 수정할 수 있는 영역과 제작자가 관리해야 하는 영역을 구분해야 합니다.

```markdown
# 직접수정 권한 설계

## 사장님이 직접 수정 가능한 영역
- 공지 문구
- 영업시간
- 휴무일
- 가격 일부
- FAQ 답변
- 이벤트 안내
- 대표 사진 교체

## 제작자가 관리하는 영역
- 전체 레이아웃
- 버튼 위치
- 모바일 간격
- 섹션 순서
- 카카오톡 문의 링크 구조
- 지도 링크
- 디자인 스타일
- 반응형 화면 점검

## 직접수정 안내 문구

사장님이 자주 바꾸는 공지, 가격, 영업시간은 직접 수정하실 수 있게 구성해드립니다.

다만 버튼 위치, 전체 디자인, 모바일 화면 구조는
문의 전환에 영향을 줄 수 있어서 관리 영역으로 두는 것을 권장드립니다.
```

### 수정 요청 양식

```markdown
# 수정 요청 양식

수정이 필요하실 때는 아래 형식으로 보내주세요.

## 수정 위치
예: 가격표 / 공지 / 영업시간 / FAQ / 사진

## 기존 내용
예: 젤네일 기본 40,000원

## 변경 내용
예: 젤네일 기본 45,000원

## 적용 희망일
예: 오늘 / 이번 주 금요일 / 다음 달 1일

## 참고 이미지
있으면 함께 보내주세요.
```

### 예약 안내 문제형

```markdown

처음 방문하는 고객은 예약 전에

카카오톡 문의로 바로 연결되는 홍보페이지를 제작하고 있습니다.

복잡한 홈페이지가 아니라
인스타 프로필에 붙여두는 모바일 문의 페이지에 가깝습니다.

첫 진행 매장 한정으로 19만원에 제작 가능합니다.
예시 화면 보내드려도 될까요?
```

### 공방/클래스형

```markdown

인스타 피드에 클래스 사진은 잘 보이는데,
처음 보는 고객이 클래스 종류, 가격, 소요 시간, 예약 방법을
한 번에 확인하기는 조금 어려울 수 있어 보였습니다.

클래스별 안내와 카카오톡 문의 버튼을 정리한
모바일 홍보페이지를 제작하고 있습니다.

첫 진행 사례를 만들고 있어서
19만원에 제작 + 1개월 무료관리 조건으로 진행 가능합니다.

간단한 샘플 구성 보내드려도 괜찮을까요?
```

## 7. 제작 완료 후 납품 메시지

납품 시점에 월관리 전환의 씨앗을 심어야 합니다.  
완료 메시지에는 “끝났습니다”가 아니라 “이제 운영 시작입니다”가 들어가야 합니다.

```markdown
# 제작 완료 안내 메시지

사장님, 홍보페이지 제작이 완료되었습니다.

아래 항목까지 확인했습니다.

- 모바일 화면 확인
- 카카오톡 문의 버튼 연결
- 가격/서비스 내용 반영
- 위치/영업시간 반영
- FAQ 정리
- 인스타그램 프로필 링크 연결 가능

오늘부터 1개월 무료관리 기간으로 기록해두겠습니다.

무료관리 기간 동안 아래 내용은 요청 주시면 반영해드릴 수 있습니다.

- 가격 문구 일부 수정
- 영업시간/휴무일 변경
- 공지 추가
- 사진 일부 교체
- FAQ 문구 수정

운영하시면서 손님들이 자주 묻는 질문이 생기면
그 내용을 페이지에 추가해두는 방식으로 개선하면 좋습니다.
```

## 누적 지식에 추가할 메모

- “직접수정 가능”은 모든 영역을 열어주는 것이 아니라, 사장님이 자주 바꾸는 정보와 제작자가 관리해야 할 전환 구조를 분리하는 방식이어야 한다.
- 첫 고객 제작은 디자인보다 자료 수집 병목이 크므로, 제작 전 자료 요청 양식과 필수 자료 기준을 먼저 고정해야 한다.
- DM 성과는 메시지 문장보다 업종별 문제 진단 정확도에 좌우되므로, 후보 매장마다 “가격/예약/위치/FAQ 중 무엇이 부족한지”를 기록해야 한다.
- 납품 메시지는 완료 통보가 아니라 무료관리 시작 안내여야 하며, 이때부터 월관리 전환을 위한 수정 이력과 반복 요청을 축적해야 한다.

```markdown
# 직접수정 가능 항목

## 사장님이 직접 수정 가능한 항목
- 공지 문구
- 대표 상품명
- 가격
- 영업시간
- 휴무일
- 이벤트 안내
- FAQ 문구
- 사진 일부 교체

## 제작자가 관리하는 항목
- 전체 레이아웃
- 모바일 화면 정렬
- 문의 버튼 연결
- 섹션 추가/삭제
- 페이지 구조 변경
- 화면 깨짐 수정
- 카카오톡/네이버/인스타 링크 점검
```

### 고객에게 보여줄 설명 문구

```markdown
이 페이지는 한 번 만들고 끝나는 소개 페이지가 아니라,
사장님이 가격, 공지, 영업시간 같은 기본 정보를 직접 수정할 수 있는 홍보페이지입니다.

다만 전체 구조 변경, 모바일 화면 정리, 문의 버튼 연결 확인처럼
기술적인 부분은 제작자가 관리하는 방식입니다.
```

### 월관리 연결 문구

```markdown
사장님이 간단한 가격이나 공지는 직접 수정하실 수 있고,
화면 정렬, 버튼 연결, 구조 수정, 섹션 추가처럼 손이 많이 가는 부분은
월관리로 맡기실 수 있습니다.
```

## 3. 제작 전 “정보 정리 대행”을 상품 가치로 포함하기

많은 소상공인은 페이지 제작보다 정보 정리에서 막힙니다.  
따라서 오퍼를 “페이지 제작”이 아니라 “흩어진 정보를 손님이 보기 쉽게 정리”로 잡는 것이 좋습니다.

```markdown
# 정보 정리 대행 범위

- 인스타 게시물에 흩어진 가격

- 네이버플레이스의 주소/영업시간
- 카카오톡 문의 전 반복 질문
- 예약 전 주의사항
- 환불/변경 기준
- 이벤트 또는 공지

## 제작자가 해주는 일
- 중복 문구 제거
- 손님 기준으로 순서 재배치
- 문의 전에 봐야 할 내용 분리
- 가격표를 모바일에서 보기 쉽게 정리
- FAQ 형태로 질문/답변 정리
```

### 영업용 문구

```markdown
사장님 인스타에는 정보가 이미 많이 있는데,
처음 보는 손님 입장에서는 가격, 예약 방법, 위치, 안내사항이 조금 흩어져 보일 수 있습니다.
```

## 4. 가격 저항을 줄이는 3단 패키지 구조

처음에는 3단 패키지를 두고, 실제 주력 상품은 가운데에 두는 편이 좋습니다.

```markdown
# 홍보페이지 패키지

## 기본형: 15만원
- 모바일 홍보페이지 1개
- 기본 정보 정리

- 대표 상품 3개까지 정리
- 무료 수정 1회

- 모바일 홍보페이지 1개
- 가격표/서비스 안내 정리
- FAQ 5개 정리
- 카카오톡 문의 버튼 연결
- 네이버플레이스/인스타 링크 연결

## 관리형: 29만원
- 추천형 전체 포함
- 자료 정리 대행 강화
- 공지/이벤트 섹션 추가
- 예약 전 안내문 정리
- 초기 수정 3회
- 1개월 무료관리
```

### 가격 제안 문구

```markdown
보통은 추천형으로 가장 많이 진행합니다.

기본형은 정말 간단한 소개 페이지가 필요할 때 맞고,
추천형은 가격표, FAQ, 문의 버튼까지 정리해서
손님이 문의 전에 필요한 정보를 대부분 확인할 수 있게 만드는 구성입니다.
```

```markdown
# 관리형 제안이 필요한 경우

- 클래스/수업 종류가 많다
- 예약 전 안내사항이 길다
- FAQ가 많다
- 이벤트나 공지가 자주 바뀐다
- 인스타 정보가 많이 흩어져 있다
```

## 5. 첫 고객 확보용 “무료 진단 리포트” 템플릿

무료 진단을 제안했다면, 답장이 왔을 때 바로 보낼 수 있는 짧은 리포트 형식이 있어야 합니다.  
이 리포트는 너무 길면 부담스럽고, 너무 짧으면 전문성이 약합니다.

```markdown
# 인스타 링크 구조 간단 진단

사장님, 처음 보는 고객 입장에서 확인해보니
아래 3가지 부분을 정리하면 문의 전환에 도움이 될 것 같습니다.

## 1. 가격 확인 동선
현재 가격 정보가 여러 게시물 또는 하이라이트에 나뉘어 있어
처음 방문한 고객이 바로 확인하기는 조금 어려울 수 있습니다.

개선 제안:
- 대표 가격 3~5개를 한 화면에 정리
- 상세 가격은 별도 섹션으로 분리
- 문의 버튼을 가격표 아래에 배치

## 2. 예약 전 안내
예약 전 확인해야 할 내용이 있다면
문의 전에 먼저 볼 수 있도록 정리하는 것이 좋습니다.

개선 제안:
- 예약 가능 시간
- 변경/취소 기준
- 시술 또는 수업 전 주의사항
- 자주 묻는 질문

## 3. 문의 버튼 위치
관심이 생긴 고객이 바로 문의할 수 있도록
카카오톡 문의 버튼을 상단과 하단에 반복 배치하는 것이 좋습니다.

개선 제안:
- 첫 화면 문의 버튼
- 가격표 아래 문의 버튼
- 페이지 마지막 문의 버튼

정리하면,
사장님 매장은 새 콘텐츠를 많이 만들기보다
이미 있는 정보를 손님 기준으로 다시 배열하는 것만으로도
문의 전환 동선이 더 좋아질 가능성이 있습니다.
```

### 리포트 후 제안 문구

```markdown
원하시면 위 내용을 바탕으로
가격, 위치, 예약 안내, 문의 버튼이 한 번에 보이는
모바일 홍보페이지 초안으로 정리해드릴 수 있습니다.

제작비는 추천형 기준 19만원이고,
제작 후 1개월 동안은 간단한 수정 관리를 무료로 도와드립니다.
```

## 3. “첫 고객용 포트폴리오”를 실제 고객 없이 만드는 방법

첫 고객이 없을 때는 포트폴리오가 없어서 영업이 막힙니다.

```markdown
## 샘플 1: 네일샵
- 흩어진 가격표를 모바일 가격표로 정리
- 예약 전 안내를 FAQ로 분리

## 샘플 2: 공방
- 클래스 종류를 카드형 목록으로 정리
- 수업 시간과 가격을 한 화면에 배치
- 예약 전 준비물 안내 추가

## 샘플 3: 필라테스
- 체험권 가격 강조
- 프로그램별 차이 정리
- 상담 문의 버튼 고정
```

### 포트폴리오 설명 문구

```markdown
아래 예시는 실제 매장 정보를 그대로 사용한 것이 아니라,
소상공인 인스타에서 자주 흩어지는 정보를
고객이 보기 쉬운 모바일 페이지 구조로 정리한 샘플입니다.
```

### 영업 DM에 넣을 짧은 문구

```markdown
제가 아직 업종별 샘플을 정리해둔 게 있어서,
사장님 매장도 비슷한 방식으로
가격, 예약 안내, 문의 버튼을 한 페이지에 정리해드릴 수 있습니다.
```

- 1개월 무료관리는 월관리 전환을 위한 수정 이력 수집 기간으로 운영해야 한다.
- 첫 10명 영업은 매출 실험이 아니라 업종, 메시지, 문제 가설별 반응 데이터를 모으는 검증 과정으로 기록해야 한다.

## 2. “문의 전 이탈”을 줄이는 FAQ 설계

소상공인 페이지의 FAQ는 친절함을 위한 장식이 아니라,  
**카톡문의 전에 생기는 망설임을 제거하는 장치**입니다.

```markdown
# FAQ 우선순위

## 가격 관련
- 추가 비용이 있나요?
- 현장 결제인가요?
- 예약금이 필요한가요?
- 할인이나 이벤트가 있나요?

## 예약 관련
- 당일 예약이 가능한가요?
- 예약 변경은 언제까지 가능한가요?
- 노쇼 규정이 있나요?
- 상담 후 결정해도 되나요?

## 방문 관련
- 주차가 가능한가요?
- 처음 방문해도 괜찮나요?
- 준비물이 있나요?
- 소요 시간이 얼마나 걸리나요?

## 업종별 추가
- 네일/미용: 제거 비용, 유지 기간, 디자인 변경 가능 여부
- 공방: 초보자 가능 여부, 완성품 수령 방식
- 필라테스/PT: 체험권 포함 내용, 운동복/샤워 가능 여부
- 음식점/카페: 예약 가능 인원, 포장/대관 가능 여부
```

### FAQ 수집 질문

```markdown
사장님, 손님들이 카톡으로 자주 물어보는 질문 5개만 알려주시면

```

### 월관리 연결 문구

```markdown
FAQ는 처음 만든 뒤에도 실제 문의가 쌓이면 계속 좋아집니다.
한 달 동안 반복 질문을 모아서 매월 1~2개씩 보완하면 문의 응대 부담이 줄어듭니다.
```

## 5. 페이지 초안 제안서 와이어프레임

첫 영업에서는 완성 페이지보다 **초안 구조 제안**이 더 빠르게 먹힙니다.  
상대가 “내 매장에도 적용되겠네”라고 느끼게 만드는 것이 목적입니다.

```markdown
# 모바일 홍보페이지 초안 구조

## 1. 첫 화면
- 매장명
- 대표 서비스 한 줄 설명
- 대표 가격 또는 핵심 혜택
- 카카오톡 문의 버튼
- 위치/영업시간 요약

## 2. 대표 상품/서비스
- 상품명
- 짧은 설명
- 가격
- 소요 시간
- 추천 대상

## 3. 예약 전 안내
- 예약 방법
- 변경/취소 기준
- 준비물
- 주차/방문 안내

## 4. 자주 묻는 질문
- 가격 관련 질문
- 예약 관련 질문
- 방문 관련 질문

## 5. 위치와 문의
- 지도 링크
- 영업시간
- 인스타 링크
- 카카오톡 문의 버튼 반복
```

### 제안서 첫 문장

```markdown
사장님 인스타에 있는 정보를 기준으로,
처음 보는 손님이 가격, 위치, 예약 방법을 빠르게 확인할 수 있는 모바일 페이지 구조를 잡아봤습니다.
```

### 제안서 마지막 문장

```markdown
이 구조로 만들면 손님이 카톡을 보내기 전에 기본 정보를 먼저 확인할 수 있어서,
반복 문의를 줄이고 예약 문의까지 더 자연스럽게 이어질 수 있습니다.
```

## 누적 지식에 추가할 메모

- 직접수정형 페이지는 기능 판매가 아니라 수정 권한 설계 판매로 접근해야 한다.
- FAQ는 단순 안내가 아니라 문의 전 이탈과 반복 질문을 줄이는 전환 장치다.
- 첫 상담에는 정보 수집 폼이 필요하며, 이 폼 자체가 제작 프로세스의 신뢰도를 높인다.
- 월 100~150만원 목표는 고가 단발 제작보다 소액 제작비와 월관리 누적 조합이 현실적이다.
- 첫 영업 자료는 완성 포트폴리오보다 “내 매장에 적용된 초안 구조”를 보여주는 쪽이 반응을 만들기 쉽다.
