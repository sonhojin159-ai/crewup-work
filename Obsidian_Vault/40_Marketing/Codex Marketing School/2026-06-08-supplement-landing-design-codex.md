# Round 1. 랜딩페이지 유형/패턴 지도

# 소상공인 직접수정 홍보페이지: 랜딩페이지 유형과 패턴 분류

> 목적: “소상공인용 랜딩페이지”를 하나의 문의형 구조로만 보지 않고, 업종과 판매 방식에 따라 어떤 유형을 선택해야 하는지 구분하기 위한 설계 자료다.  
> 정량 통계는 사용하지 않았고, 한국 소상공인/개인사업자 시장에서 실제 샘플 제작에 바로 쓸 수 있는 구조 중심으로 정리한다.

---

## 1. 유형을 먼저 나눠야 하는 이유

같은 1페이지 홍보페이지라도 고객이 원하는 행동은 다르다.

- 미용실은 “예약”
- 인테리어는 “견적 문의”
- 공방은 “클래스 신청”
- 식당은 “지도 방문”
- PT샵은 “체험 상담”
- 변호사/세무사는 “상담 문의”
- 사진작가는 “포트폴리오 확인 후 문의”
- 네일샵은 “가격표 확인 후 예약”

따라서 샘플을 만들 때도 “소상공인 랜딩페이지” 하나로 만들면 약하다.  
처음부터 유형별 템플릿을 나누면 디자인, 섹션, 관리자 수정 필드, 가격 패키지를 훨씬 쉽게 상품화할 수 있다.

# 2. 랜딩페이지 8가지 유형

## A. 문의형 랜딩페이지

### 적합한 상황

고객이 바로 구매하기보다 먼저 질문하거나 상담해야 하는 업종에 적합하다.  
가격, 일정, 가능 여부, 상세 조건이 사람마다 달라지는 경우다.

### 대표 업종

- 법률 상담
- 세무 상담
- 노무 상담
- 보험 상담
- 학원 상담
- 병원/한의원 상담
- 심리상담
- B2B 서비스
- 홈페이지 제작, 디자인, 마케팅 대행

### 첫 화면 메시지

핵심은 “무엇을 도와주는지”와 “누가 문의하면 좋은지”를 바로 보여주는 것이다.

예시:

> 처음 창업한 개인사업자를 위한 세무 상담  
> 복잡한 신고와 비용 문제를 한 번에 정리해드립니다

또는:

> 우리 아이에게 맞는 수업이 궁금하다면  
> 레벨 확인 후 맞춤 상담을 도와드립니다

### 핵심 섹션

1. 대상 고객 명시
2. 해결 가능한 문제
3. 서비스 범위
4. 상담 진행 방식
5. 자주 묻는 질문
6. 후기 또는 사례
7. 상담 CTA
8. 운영시간/응답 가능 시간

### CTA

- 상담 문의하기
- 카카오톡 상담하기
- 무료 상담 신청
- 전화 상담하기
- 상담 가능 시간 확인하기

### 신뢰 요소

- 상담 가능 분야
- 대표자 이력
- 자격증/면허
- 실제 상담 사례
- 고객 후기
- 응답 시간 안내
- 누적 경력  
  단, 숫자를 넣을 경우 실제 확인 가능한 자료가 있어야 한다.

### 고객 수정 가능 필드

- 대표 문구
- 상담 가능 항목
- 상담 시간
- 상담 CTA 링크
- 전화번호
- 카카오톡 채널 링크
- FAQ
- 후기
- 대표자 프로필
- 공지사항

### 선택 기준

문의형은 “가격을 바로 보여주기 어렵거나, 고객 상황별로 안내가 필요한 업종”에 맞다.  
단순 매장보다는 상담, 진단, 설명이 필요한 서비스에 적합하다.

### 적합한 상황

방문 시간, 시술 시간, 수업 시간처럼 고객이 특정 시간을 잡아야 하는 업종에 적합하다.

- 네일샵

예약형은 첫 화면에서 “어떤 예약을 받을 것인지”가 선명해야 한다.

> 원하는 디자인과 일정에 맞춰 상담 후 확정해드립니다

또는:

> 초보자도 시작하기 쉬운 1:1 필라테스  
> 체형 상담 후 첫 수업을 예약하세요

2. 예약 가능한 서비스
3. 인기 메뉴 또는 대표 프로그램
4. 가격/소요 시간
5. 예약 가능 시간 안내
6. 예약 절차
7. 후기/전후 사진
8. 위치/주차 안내
9. 예약 CTA

### CTA

- 전후 사진

이 유형에서는 예쁜 설명보다 예약 과정의 불안 제거가 더 중요하다.

- 이사
- 입주청소
- 에어컨 청소
- 인테리어
- 도배/장판
- 간판 제작
- 렌탈/설치
- 수리 서비스
- 방충망/중문/커튼
- 홈페이지 제작
- 영상 제작
- 행사 대행

### 첫 화면 메시지

견적형은 “가격이 왜 달라지는지”를 자연스럽게 설명해야 한다.

예시:

> 우리 집 조건에 맞는 입주청소 견적  
> 평수, 오염도, 옵션을 확인한 뒤 정확히 안내드립니다

또는:

> 매장 분위기에 맞는 간판 제작 견적  
> 사이즈와 설치 환경에 따라 맞춤 견적을 드립니다

1. 서비스 범위
2. 견적이 달라지는 기준
3. 작업 사례
4. 진행 절차
5. 기본 가격 또는 시작가

7. 작업 가능 지역

### 대표 업종

- 네일샵 오픈 이벤트
- 피부관리실 첫 방문 할인

- 헬스장 등록 이벤트
- 학원 방학 특강
- 카페 신메뉴 이벤트

- 웨딩/돌잔치/스튜디오 프로모션

  의료 광고는 별도 규정 확인 필요

### 첫 화면 메시지

이벤트형은 혜택, 기간, 대상이 첫 화면에서 바로 보여야 한다.

또는:

> 여름방학 수학 집중반 모집  
> 소수 정원으로 4주간 약점 단원만 집중 관리합니다

1. 이벤트 혜택

3. 기간/마감 조건

5. 정상가 대비 이벤트가

9. 신청 CTA

### 대표 업종

- 네일샵
- 미용실

### 첫 화면 메시지

예시:

# 4. 유형 선택 기준

## 1. 가격이 고정되어 있으면 가격표형

예를 들어 네일, 펫미용, 사진관, 공방처럼 상품 구성이 어느 정도 정해져 있으면 가격표형이 좋다.  
가격을 감추는 것보다 “기본가, 옵션가, 추가 조건”을 명확히 보여주는 편이 문의 품질을 높인다.

## 2. 시간이 핵심이면 예약형

시술, 수업, 촬영, 상담처럼 특정 시간대를 잡아야 하면 예약형이 맞다.  
이 경우 관리자에서 가장 자주 바뀌는 필드는 예약 링크, 휴무일, 가능 시간, 공지사항이다.

## 3. 조건별 가격 차이가 크면 견적형

이사, 청소, 인테리어, 설치, 제작 서비스는 견적형이 맞다.  
첫 화면에서 가격을 단정하기보다 “정확한 견적을 위해 필요한 정보”를 자연스럽게 받는 구조가 중요하다.

## 4. 신규 고객 유입이 목적이면 이벤트형

오픈 초기, 비수기, 시즌 행사, 체험권 판매는 이벤트형이 적합하다.  
이 유형은 월관리 상품과 잘 맞는다. 이벤트 문구와 기간을 계속 바꿔야 하기 때문이다.

## 5. 첫 구매 장벽이 높으면 체험신청형

운동, 학원, 상담, 관리 서비스처럼 고객이 바로 결제하기 부담스러운 업종은 체험신청형이 적합하다.  
“무료/저가 체험 → 상담 → 정규 상품” 흐름을 전제로 설계한다.

## 6. 결과물이 설득력이면 포트폴리오형

사진, 디자인, 인테리어, 네일, 헤어, 케이크, 꽃처럼 결과물이 중요한 업종은 포트폴리오형이 맞다.  
이미지 업로드, 카테고리, 정렬 기능이 관리자 툴의 핵심이 된다.

## 7. 오프라인 방문이 목적이면 지도방문형

카페, 음식점, 꽃집, 반찬가게, 소품샵처럼 방문이 곧 전환인 업종은 지도방문형이 맞다.  
첫 화면에 위치, 영업시간, 오늘의 메뉴, 길찾기 CTA가 있어야 한다.

# 6. 샘플 제작 우선순위

처음부터 8개를 모두 만들 필요는 없다.  
판매 가능성과 재사용성을 기준으로 보면 다음 순서가 현실적이다.

## 1순위: 예약형 + 가격표형 혼합

추천 업종:

- 네일샵
- 피부관리실
- 필라테스
- 펫미용

이유:

- 사진이 중요하다.
- 가격표가 필요하다.
- 예약 CTA가 명확하다.
- 수정 필드가 많아 직접수정 기능을 보여주기 좋다.
- 월관리로 이어질 요소가 많다.

## 2순위: 견적형

추천 업종:

- 입주청소
- 이사
- 인테리어
- 에어컨 청소

이유:

- 견적 폼 설계가 차별점이 된다.
- 업체가 반복적으로 받는 질문을 줄일 수 있다.
- 작업 사례와 후기 추가가 월관리로 연결된다.

## 3순위: 지도방문형

추천 업종:

- 카페
- 음식점
- 꽃집
- 반찬가게

이유:

- 소상공인 수요가 넓다.
- 오늘의 메뉴, 공지, 영업시간 수정 니즈가 크다.
- 다만 단가를 높이려면 사진 퀄리티와 메뉴 관리 구조가 필요하다.

## 4순위: 포트폴리오형

추천 업종:

- 사진관
- 네일아트
- 인테리어
- 케이크 공방
- 플로리스트

이유:

- 샘플이 예쁘게 나오기 쉽다.
- 하지만 이미지 관리 UX를 잘 만들어야 한다.
- 결과물 사진이 부족한 고객에게는 제작이 지연될 수 있다.

# 7. 결론

소상공인 직접수정 홍보페이지는 “문의 버튼이 있는 1페이지”로만 보면 상품성이 약하다.  
실제로는 업종별 전환 행동에 따라 다음처럼 나눠야 한다.

- 상담이 필요하면 문의형
- 시간을 잡아야 하면 예약형
- 조건별 금액이 다르면 견적형
- 단기 매출을 만들려면 이벤트형
- 첫 구매 장벽을 낮추려면 체험신청형
- 가격 확인이 중요하면 가격표형
- 결과물이 중요하면 포트폴리오형
- 방문 유도가 목적이면 지도방문형

첫 샘플은 **예약형 + 가격표형 + 포트폴리오형이 섞인 네일샵/피부관리실/필라테스** 중 하나가 좋다.  
이 조합은 가격, 사진, 후기, 공지, 예약 링크, 이벤트 문구를 모두 수정 필드로 보여줄 수 있어서 “직접수정 홍보페이지”라는 상품의 장점이 가장 잘 드러난다.

```css
  --border-soft: 1px solid rgba(15, 23, 42, 0.1);
```

페이지 전체는 다음처럼 잡는다.

```css
.page {
```

## 2. 타이포그래피 시스템

### 폰트

한국어 기준 추천:

```css
font-family:
  Pretendard,
  -apple-system,
  BlinkMacSystemFont,
  "Apple SD Gothic Neo",
  "Noto Sans KR",
  sans-serif;
```

Pretendard를 쓰면 숫자, 한글, 버튼 텍스트가 안정적으로 보인다. 단, 속도를 위해 웹폰트를 과하게 많이 불러오지 않는다. 가능하면 Regular, Medium, Bold 3개 이하만 사용한다.

### 크기 체계

```css
:root {
  --text-xs: 12px;
  --text-sm: 14px;
  --text-md: 16px;
  --text-lg: 18px;
  --text-xl: 22px;
  --text-2xl: 28px;
  --text-3xl: 34px;
}
```

권장 사용:

| 요소 | 크기 | 줄간격 | 굵기 |
|---|---:|---:|---:|
| 히어로 제목 | 30~34px | 1.18~1.25 | 700~800 |
| 섹션 제목 | 24~28px | 1.25~1.32 | 700 |
| 카드 제목 | 18~20px | 1.35 | 700 |
| 본문 | 16px | 1.6~1.7 | 400~500 |
| 보조 설명 | 14px | 1.5~1.6 | 400 |
| 캡션/라벨 | 12~13px | 1.4 | 500~600 |
| 버튼 | 15~17px | 1.2 | 700 |

한국어 모바일 본문은 `15px` 이하로 내려가면 신뢰감이 떨어지고 노년층/중장년 고객에게 불리하다. 기본 본문은 `16px`를 유지한다.

```css
h1 {
  font-size: clamp(30px, 8vw, 34px);
  line-height: 1.2;
  font-weight: 800;
  letter-spacing: 0;
}

h2 {
  font-size: 26px;
  line-height: 1.3;
  font-weight: 750;
  letter-spacing: 0;
}

p {
  font-size: 16px;
  line-height: 1.65;
  letter-spacing: 0;
}
```

주의할 점:

- 한국어는 영문보다 줄이 빨리 길어지므로 제목 한 줄은 **12~16자 내외**가 보기 좋다.
- 가격, 운영시간, 주소는 문장형보다 **표/행 구조**로 분리한다.
- `letter-spacing: -0.05em` 같은 과한 자간 축소는 모바일에서 답답해 보인다. 기본은 `0`.

## 4. 버튼 시스템

### 버튼 크기

모바일 CTA 버튼은 손가락으로 누르기 쉬워야 한다.

```css
.button {
  min-height: 52px;
  padding: 15px 18px;
  border-radius: 12px;
  font-size: 16px;
  font-weight: 750;
  line-height: 1.2;
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: 8px;
}
```

권장 크기:

| 버튼 유형 | 높이 | 용도 |
|---|---:|---|
| 메인 CTA | 52~56px | 카카오톡 문의, 예약 |
| 보조 CTA | 48~52px | 전화, 지도 |
| 아이콘 버튼 | 44~48px | 상단/하단 보조 액션 |
| 고정 하단 CTA | 56~64px | 가장 중요한 전환 |

### 버튼 우선순위

버튼은 항상 1순위와 2순위를 분리한다.

예시:

```html
<div class="cta-row">
  <a class="button button-primary">카카오톡 문의</a>
  <a class="button button-secondary">전화하기</a>
</div>
```

```css
.cta-row {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 10px;
}

.button-primary {
  background: #111827;
  color: #fff;
}

.button-secondary {
  background: #fff;
  color: #111827;
  border: 1px solid #d1d5db;
}
```

한국 소상공인 페이지에서는 카카오톡, 전화, 네이버지도, 예약 버튼을 모두 같은 강조도로 두면 오히려 선택이 느려진다.

권장 우선순위:

- 예약/상담형 업종: `카카오톡 문의` 1순위, `전화하기` 2순위
- 즉시 방문형 업종: `네이버지도 보기` 1순위, `전화하기` 2순위
- 병원/학원/필라테스: `예약하기` 1순위, `전화 상담` 2순위
- 배달/출장/수리: `전화하기` 1순위, `카카오톡 문의` 2순위

## 5. 고정 CTA 설계

하단 고정 CTA는 모바일 전환에서 매우 중요하다. 단, 콘텐츠를 가리면 안 된다.

```css
.sticky-cta {
  position: fixed;
  left: 50%;
  bottom: 0;
  transform: translateX(-50%);
  width: min(100%, 480px);
  padding: 10px 16px calc(10px + env(safe-area-inset-bottom));
  background: rgba(255, 255, 255, 0.96);
  border-top: 1px solid rgba(15, 23, 42, 0.1);
  backdrop-filter: blur(12px);
  z-index: 50;
}

body {
  padding-bottom: 84px;
}
```

권장 구조:

```html
<div class="sticky-cta">
  <div class="sticky-cta__grid">
    <a class="sticky-cta__sub">전화</a>
    <a class="sticky-cta__main">카카오톡 문의</a>
  </div>
</div>
```

```css
.sticky-cta__grid {
  display: grid;
  grid-template-columns: 0.42fr 1fr;
  gap: 8px;
}

.sticky-cta__main,
.sticky-cta__sub {
  min-height: 52px;
  border-radius: 12px;
  display: flex;
  align-items: center;
  justify-content: center;
  font-weight: 750;
}
```

고정 CTA 문구는 짧게 쓴다.

좋은 예:

- `카카오톡 문의`
- `예약하기`
- `전화 상담`
- `길찾기`

피해야 할 예:

- `지금 바로 무료 상담 신청하고 혜택 받기`
- `자세한 예약 문의는 카카오톡으로 남겨주세요`

버튼 안의 긴 문장은 모바일에서 신뢰감보다 부담을 준다.

## 7. 이미지 비율과 사용 규칙

### 권장 비율

| 이미지 용도 | 비율 | 설명 |
|---|---:|---|
| 히어로 대표 이미지 | 4:5 또는 1:1 | 모바일 첫 화면에서 안정적 |
| 시술/작업 사례 | 1:1 | 그리드 구성에 적합 |
| 공간/매장 사진 | 4:3 | 내부 분위기 전달 |
| Before/After | 1:1 두 장 또는 2분할 | 비교가 쉬움 |
| 지도 썸네일 | 16:9 | 위치 섹션에서 충분 |
| 프로필/원장 사진 | 1:1 또는 4:5 | 신뢰 요소 |

```css
.image-frame {
  overflow: hidden;
  border-radius: 16px;
  background: #f3f4f6;
}

.image-frame img {
  width: 100%;
  height: 100%;
  object-fit: cover;
  display: block;
}
```

```css
.hero-image {
  aspect-ratio: 4 / 5;
}

.gallery-image {
  aspect-ratio: 1 / 1;
}

.map-image {
  aspect-ratio: 16 / 9;
}
```

### 이미지 원칙

- 첫 화면 이미지는 업종이 즉시 보이는 실제 이미지가 좋다.
- 네일/피부/미용은 결과물 클로즈업이 유리하다.
- 필라테스/PT/공방은 공간 + 사람의 사용 장면이 유리하다.
- 병원/상담형 업종은 인물보다 공간, 장비, 절차 안내 이미지가 더 부담이 적다.
- 너무 어두운 이미지, 흐릿한 배경 이미지, 의미 없는 감성 이미지는 전환에 불리하다.
- 고객이 직접 사진을 바꿀 수 있으므로 업로드 이미지는 자동 크롭 기준을 정해둔다.

관리자 필드 예시:

```ts
type ImageField = {
  src: string;
  alt: string;
  ratio: "1:1" | "4:5" | "4:3" | "16:9";
  focalPoint?: "center" | "top" | "bottom";
};
```

## 8. 가격표 표현 원칙

가격표는 예쁘게 꾸미는 영역이 아니라 불안을 줄이는 영역이다.

### 가격표 기본 구조

```html
<section class="pricing-section">
  <header class="section-header">
    <h2>가격 안내</h2>
    <p>방문 전 확인이 필요한 대표 상품입니다.</p>
  </header>

  <div class="price-list">
    <article class="price-item">
      <div>
        <h3>기본 케어</h3>
        <p>손톱 정리 + 큐티클 케어</p>
      </div>
      <strong>30,000원</strong>
    </article>
  </div>
</section>
```

```css
.price-item {
  display: grid;
  grid-template-columns: 1fr auto;
  gap: 16px;
  padding: 16px 0;
  border-bottom: 1px solid #e5e7eb;
}

.price-item h3 {
  margin: 0 0 4px;
  font-size: 17px;
  line-height: 1.35;
}

.price-item p {
  margin: 0;
  font-size: 14px;
  color: #6b7280;
}

.price-item strong {
  white-space: nowrap;
  font-size: 17px;
  font-weight: 800;
}
```

### 가격표 원칙

- `상담 후 결정`만 반복하면 이탈할 수 있다. 최소 시작가를 제공한다.
- 변동 가격은 `30,000원~`, `상태에 따라 추가`, `방문 전 상담 필요`처럼 표현한다.
- 할인 가격은 원가/할인가를 명확히 구분한다.
- 너무 많은 상품은 접어두기보다 `대표 상품 5개 + 전체 가격표 보기` 구조가 낫다.
- 월관리 고객이 직접 수정할 수 있도록 가격 항목을 배열 데이터로 분리한다.

```ts
type PriceItem = {
  name: string;
  description?: string;
  price: string;
  badge?: "인기" | "추천" | "신규";
  note?: string;
};
```

### 피해야 할 가격 UI

- 이미지로 된 가격표만 제공
- 가격이 카드 안에서 줄바꿈되어 읽기 어려운 구조
- `문의`, `문의`, `문의`만 반복되는 표
- 이벤트가 끝났는데 날짜가 없는 할인 배너
- 부가세/재료비/출장비 기준이 없는 가격 안내

## 9. 섹션 길이와 순서

이번 문서는 기본 목적 반복을 피하기 위해 구조 이름보다 각 섹션의 디자인 밀도를 기준으로 정리한다.

### 권장 섹션 밀도

| 섹션 | 모바일 길이 | 구성 |
|---|---:|---|
| 히어로 | 1.0~1.3 화면 | 제목, 짧은 설명, 이미지, CTA |
| 핵심 장점 | 0.7~1.2 화면 | 3개 카드 또는 체크리스트 |
| 대표 상품 | 1.2~1.8 화면 | 3~5개 카드 |
| 가격 | 1.0~1.8 화면 | 대표 가격 4~7개 |
| 사례/사진 | 1.0~2.0 화면 | 2열 그리드 4~6장 |
| 후기 | 1.2~2.0 화면 | 후기 3개 |
| 예약 과정 | 0.8~1.2 화면 | 3단계 |
| 위치/운영시간 | 1.0~1.5 화면 | 지도, 주소, 시간, 주차 |
| FAQ | 0.8~1.5 화면 | 4~6개 |

모바일에서 한 섹션이 3화면 이상 길어지면 쪼갠다. 특히 가격표와 후기 섹션은 길어지기 쉽다.

### 섹션 컴포넌트 기본형

```tsx
type SectionProps = {
  eyebrow?: string;
  title: string;
  description?: string;
  children: React.ReactNode;
};

function Section({ eyebrow, title, description, children }: SectionProps) {
  return (
    <section className="section">
      <header className="section-header">
        {eyebrow && <p className="section-eyebrow">{eyebrow}</p>}
        <h2 className="section-title">{title}</h2>
        {description && <p className="section-desc">{description}</p>}
      </header>
      {children}
    </section>
  );
}
```

## 10. 카카오톡/전화/네이버지도/예약 버튼 배치

## 한국 사용자 기준 버튼 우선순위

### 1순위: 카카오톡 문의

적합한 경우:

- 네일, 피부, 속눈썹, 미용실
- 필라테스, PT, 학원
- 공방, 클래스
- 견적이 필요한 서비스
- 사진을 보내고 상담해야 하는 업종

배치:

- 히어로 CTA 첫 번째
- 하단 고정 CTA 메인 버튼
- 가격표 하단
- FAQ 하단
- 페이지 마지막

문구:

- `카카오톡 문의`
- `카톡으로 예약 문의`
- `사진 보내고 상담하기`
- `체험권 문의하기`

### 2순위: 전화

적합한 경우:

- 음식점, 병원, 수리, 출장 서비스
- 당일 예약/긴급 문의
- 중장년층 고객 비중이 높은 업종

배치:

- 히어로 보조 버튼
- 하단 고정 CTA 보조 버튼
- 운영시간 옆
- 위치 섹션

문구:

- `전화하기`
- `전화 상담`
- `예약 전화`
- `당일 문의`

전화 링크:

```html
<a href="tel:0212345678">전화하기</a>
```

전화번호 표시는 가독성을 위해 하이픈을 넣는다.

```html
<span>02-1234-5678</span>
```

### 네이버지도

네이버지도는 위치 기반 업종에서 강한 신뢰 요소다.

적합한 경우:

- 오프라인 매장 전체
- 주차/역세권/찾아오는 길이 중요한 업종
- 네이버 플레이스 후기가 있는 업장

배치:

- 위치 섹션 메인 버튼
- 하단 고정 CTA에는 필요할 때만 보조 액션으로 배치
- 히어로에는 `길찾기`보다 문의/예약을 우선

문구:

- `네이버지도 보기`
- `길찾기`
- `주차 안내 보기`

```html
<a href="https://map.naver.com/..." target="_blank" rel="noopener">
  네이버지도 보기
</a>
```

지도 섹션 구조:

```html
<section class="location">
  <div class="map-image"></div>
  <div class="info-list">
    <div><span>주소</span><strong>서울시 ...</strong></div>
    <div><span>운영시간</span><strong>평일 10:00-20:00</strong></div>
    <div><span>주차</span><strong>건물 뒤 2대 가능</strong></div>
  </div>
  <div class="cta-row">
    <a class="button button-primary">길찾기</a>
    <a class="button button-secondary">전화하기</a>
  </div>
</section>
```

### 예약 버튼

예약 버튼은 사용하는 예약 시스템에 따라 다르다.

가능한 링크:

- 네이버예약
- 카카오톡 채널
- 자체 폼
- 구글폼
- 타입폼/토스폼 등 외부 폼
- 병원/학원 자체 예약 링크

배치 원칙:

- 예약 링크가 확실하면 `예약하기`를 1순위 CTA로 둔다.
- 예약 전에 상담이 필요한 업종은 `상담 후 예약` 흐름으로 둔다.
- 예약 버튼을 눌렀을 때 외부 앱/브라우저로 이동한다면 버튼 아래에 짧게 안내한다.

예시:

```html
<a class="button button-primary" href="{bookingUrl}">
  예약하기
</a>
<p class="button-note">네이버예약 화면으로 이동합니다.</p>
```

## 12. 접근성 체크리스트

### 터치 접근성

- 모든 버튼 높이는 최소 `44px`, 주요 CTA는 `52px` 이상.
- 버튼 사이 간격은 최소 `8px`.
- 하단 고정 CTA가 휴대폰 홈 인디케이터와 겹치지 않도록 `env(safe-area-inset-bottom)` 적용.
- 지도, 전화, 카카오톡 버튼은 텍스트 라벨을 반드시 포함한다.
- 아이콘만 있는 버튼에는 `aria-label`을 넣는다.

```html
<a href="tel:0212345678" aria-label="전화 문의하기">
  전화하기
</a>
```

### 색상 대비

- 본문 텍스트는 흰 배경 기준 `#111827` 또는 유사한 진한 색.
- 보조 텍스트는 `#4b5563` 이상 권장.
- 연회색 텍스트 `#9ca3af`는 캡션 외에는 피한다.
- 노란 카카오톡 버튼을 쓸 경우, 텍스트는 검정 계열로 둔다.

```css
.kakao-button {
  background: #fee500;
  color: #111827;
}
```

### 포커스 상태

키보드 접근을 위해 focus-visible을 둔다.

```css
a:focus-visible,
button:focus-visible {
  outline: 3px solid rgba(37, 99, 235, 0.35);
  outline-offset: 3px;
}
```

### 이미지 대체 텍스트

좋은 alt:

- `속눈썹 펌 시술 후 정면 사진`
- `필라테스 개인 레슨 공간 내부`
- `공방 원데이 클래스 완성 작품`

나쁜 alt:

- `이미지`
- `사진1`
- `예쁜 사진`

## 13. 가독성 체크리스트

- 제목은 모바일 기준 2~3줄을 넘기지 않는다.
- 본문 문단은 2~3문장 이하로 나눈다.
- 가격, 주소, 시간, 주차 정보는 문장보다 행 구조로 보여준다.
- CTA 버튼 문구는 8자 내외가 가장 안정적이다.
- 가격표의 금액은 오른쪽 정렬 또는 별도 강조로 고정한다.
- 한 화면에 강조 색을 2개 이상 경쟁시키지 않는다.
- 후기 문장은 긴 원문보다 핵심 2~4줄을 먼저 보여준다.
- `자세히 보기`, `더보기` 버튼은 정보가 정말 많을 때만 쓴다.

## 14. 속도 체크리스트

소상공인 랜딩페이지는 속도가 느리면 신뢰가 떨어진다. 특히 인스타/카카오톡 인앱 브라우저에서 체감이 중요하다.

### 이미지

- 히어로 이미지는 WebP 또는 AVIF 사용.
- 모바일 기준 대표 이미지는 보통 800~1200px 폭이면 충분하다.
- 갤러리 썸네일은 원본 고해상도 이미지를 그대로 넣지 않는다.
- `loading="lazy"`는 히어로 아래 이미지에 적용한다.
- 히어로 이미지는 가능하면 우선 로드한다.

```html
<img
  src="/images/hero.webp"
  width="960"
  height="1200"
  alt="매장 대표 시술 사진"
/>
```

```html
<img
  src="/images/gallery-01.webp"
  loading="lazy"
  decoding="async"
  alt="시술 사례 사진"
/>
```

### CSS/JS

- 랜딩페이지 하나에 무거운 애니메이션 라이브러리를 넣지 않는다.
- 카카오톡/지도/예약 외부 스크립트는 꼭 필요한 경우만 사용한다.
- 지도는 실제 지도 iframe보다 이미지/정적 썸네일 + 네이버지도 링크가 빠르다.
- 폰트 weight를 많이 불러오지 않는다.
- 관리자 수정 데이터는 JSON/config로 분리하되 클라이언트 번들을 과하게 키우지 않는다.

## 15. 신뢰감 체크리스트

### 반드시 보여줄 정보

- 사업장명
- 대표 서비스
- 가격 또는 시작가
- 운영시간
- 주소
- 문의 방법
- 예약 가능 여부
- 주차/대중교통 안내
- 후기 또는 실제 사례
- 취소/변경 기준이 중요한 업종은 간단한 안내

### 신뢰를 높이는 UI 표현

```html
<ul class="trust-list">
  <li>1:1 예약제로 운영</li>
  <li>시술 전 충분한 상담 진행</li>
  <li>네이버지도에서 위치 확인 가능</li>
</ul>
```

```css
.trust-list {
  display: grid;
  gap: 10px;
  padding: 0;
  margin: 0;
  list-style: none;
}

.trust-list li {
  padding-left: 28px;
  position: relative;
  font-size: 15px;
  line-height: 1.55;
}

.trust-list li::before {
  content: "✓";
  position: absolute;
  left: 0;
  top: 0;
  font-weight: 800;
  color: #16a34a;
}
```

주의:

- 검증되지 않은 `1위`, `최고`, `100% 만족` 같은 표현은 피한다.
- 실제 근거가 없으면 수상/인증/후기 수치를 쓰지 않는다.
- 의료, 법률, 금융, 건강 관련 업종은 과장 표현을 피하고 추가 확인이 필요하다.
- 후기는 실제 고객 동의 여부와 개인정보 노출 여부를 확인해야 한다.

## 16. 컴포넌트 설계안

### 데이터 구조

고객 직접수정을 위해 텍스트와 링크는 컴포넌트 안에 박지 않는다.

```ts
type LandingPageData = {
  business: {
    name: string;
    category: string;
    phone: string;
    address: string;
    hours: string;
    parking?: string;
  };

  hero: {
    headline: string;
    subcopy: string;
    image: ImageField;
    primaryCta: CtaLink;
    secondaryCta?: CtaLink;
  };

  services: ServiceItem[];
  prices: PriceItem[];
  gallery: ImageField[];
  reviews: ReviewItem[];
  location: LocationInfo;
  faq: FaqItem[];
  stickyCta: {
    primary: CtaLink;
    secondary?: CtaLink;
  };
};

type CtaLink = {
  label: string;
  href: string;
  type: "kakao" | "phone" | "map" | "booking" | "form";
};
```

### CTA 컴포넌트

```tsx
function CtaButton({ cta, variant = "primary" }: {
  cta: CtaLink;
  variant?: "primary" | "secondary";
}) {
  return (
    <a
      className={`button button-${variant} button-${cta.type}`}
      href={cta.href}
      target={cta.href.startsWith("http") ? "_blank" : undefined}
      rel={cta.href.startsWith("http") ? "noopener" : undefined}
    >
      {cta.label}
    </a>
  );
}
```

### 가격 컴포넌트

```tsx
function PriceList({ items }: { items: PriceItem[] }) {
  return (
    <div className="price-list">
      {items.map((item) => (
        <article className="price-item" key={item.name}>
          <div>
            {item.badge && <span className="badge">{item.badge}</span>}
            <h3>{item.name}</h3>
            {item.description && <p>{item.description}</p>}
            {item.note && <small>{item.note}</small>}
          </div>
          <strong>{item.price}</strong>
        </article>
      ))}
    </div>
  );
}
```

### 하단 고정 CTA 컴포넌트

```tsx
function StickyCta({ primary, secondary }: {
  primary: CtaLink;
  secondary?: CtaLink;
}) {
  return (
    <aside className="sticky-cta" aria-label="빠른 문의">
      <div className="sticky-cta__grid">
        {secondary && <CtaButton cta={secondary} variant="secondary" />}
        <CtaButton cta={primary} variant="primary" />
      </div>
    </aside>
  );
}
```

## 17. 색상 시스템

업종별로 색을 완전히 바꾸기보다 토큰만 교체한다.

```css
:root {
  --color-bg: #ffffff;
  --color-surface: #f8fafc;
  --color-text: #111827;
  --color-muted: #4b5563;
  --color-border: #e5e7eb;

  --color-primary: #111827;
  --color-primary-text: #ffffff;

  --color-accent: #2563eb;
  --color-success: #16a34a;
  --color-kakao: #fee500;
}
```

업종별 추천 방향:

| 업종 | 색상 방향 |
|---|---|
| 네일/피부/미용 | 흰 배경 + 차분한 포인트 컬러 |
| 필라테스/PT | 밝은 배경 + 선명한 액센트 |
| 학원/공부방 | 신뢰감 있는 블루/그린 계열 |
| 공방/클래스 | 작품 사진 중심, UI 색은 절제 |
| 병원/한의원 | 흰색, 옅은 회색, 신뢰감 있는 포인트 |
| 청소/수리/출장 | 명확한 대비, 전화 CTA 강조 |

페이지 전체가 한 가지 색으로만 보이면 저가 템플릿처럼 보일 수 있다. 배경, 텍스트, 보더, 포인트, 상태색을 분리한다.

## 18. 관리자 수정 필드 기준

직접수정 상품으로 팔려면 관리자에서 수정 가능한 항목을 처음부터 제한해야 한다.

### 고객이 수정 가능해야 하는 필드

- 매장명
- 대표 문구
- 공지/이벤트
- 대표 이미지
- 갤러리 이미지
- 가격표
- 운영시간
- 휴무일
- 전화번호
- 카카오톡 링크
- 네이버지도 링크
- 예약 링크
- 후기
- FAQ

### 고객이 수정하지 않는 편이 좋은 필드

- 전체 레이아웃
- 폰트 크기
- 버튼 크기
- 섹션 여백
- 컬러 팔레트 전체
- 애니메이션
- 카드 구조

즉, 고객은 내용을 바꾸고 제작자는 디자인 시스템을 유지한다.

관리자 입력 검증 예시:

```ts
type EditableNotice = {
  title: string;
  body: string;
  active: boolean;
  startsAt?: string;
  endsAt?: string;
};
```

공지/이벤트에는 종료일 필드를 두는 것이 좋다. 종료된 이벤트가 계속 노출되면 신뢰가 떨어진다.

## 19. 최종 제작 체크리스트

### 디자인

- 첫 화면에서 업종, 대표 서비스, 문의 버튼이 보인다.
- 주요 CTA는 페이지 전체에 반복 배치되어 있다.
- 하단 고정 CTA가 콘텐츠를 가리지 않는다.
- 가격표는 이미지가 아니라 텍스트/데이터로 구성되어 있다.
- 카드 안의 제목, 설명, 가격이 명확히 분리되어 있다.
- 섹션 하나가 지나치게 길지 않다.
- 실제 사진이 충분히 크고 선명하게 보인다.

### 접근성

- 주요 버튼 높이가 52px 이상이다.
- 모든 링크/버튼에 텍스트 라벨이 있다.
- 색 대비가 충분하다.
- focus-visible 상태가 있다.
- 이미지 alt가 비어 있지 않다.
- 전화 링크는 `tel:` 형식이다.
- 외부 링크는 새 창 처리와 `rel="noopener"`를 적용했다.

### 속도

- 이미지는 WebP/AVIF로 최적화했다.
- 히어로 아래 이미지는 lazy loading을 적용했다.
- 지도 iframe을 무겁게 넣지 않았다.
- 웹폰트 weight를 최소화했다.
- 불필요한 애니메이션/스크립트를 제거했다.

### 신뢰

- 주소, 운영시간, 문의 방법이 정확하다.
- 가격 또는 시작가가 있다.
- 후기/사례가 실제 정보로 보인다.
- 이벤트에는 기간이 있다.
- 과장 표현과 검증되지 않은 수치를 쓰지 않았다.
- 업종상 법적 표현 제한이 필요한 경우 추가 확인 대상으로 표시했다.

# 샘플 제작 우선순위

첫 샘플은 **네일샵** 또는 **필라테스/PT**가 좋다.  
이유는 사진, 가격표, 후기, 예약 CTA, 운영시간, 직접 수정 필드가 모두 들어가서 “직접수정 홍보페이지”의 장점이 잘 보인다.

두 번째 샘플은 **청소/이사**가 좋다.  
견적형 구조라서 네일샵과 완전히 다른 판매 사례를 보여줄 수 있다.

세 번째 샘플은 **학원/공부방** 또는 **공방/원데이클래스**가 좋다.  
예약형, 상담형, 시간표형 구조를 확장할 수 있다.

# Round 4. 고객용 수정툴 UX와 데이터 필드 설계

# 고객 직접수정 관리자 툴 UX / 데이터 구조 설계

## 1. 설계 원칙

이 관리자 툴의 핵심은 “고객이 자주 바꾸는 정보만 안전하게 수정하게 하는 것”이다.

소상공인 고객에게 전체 홈페이지 편집기를 주면 오히려 망가질 가능성이 높다. 따라서 자유 편집형 빌더가 아니라, **정해진 섹션 안에서 텍스트, 이미지, 가격, 링크만 수정하는 폼 기반 관리자**가 적합하다.

권장 방향:

- 고객은 내용만 수정한다.
- 디자인, 레이아웃, 여백, 폰트, 버튼 스타일은 잠근다.
- 저장과 게시를 분리한다.
- 모바일 미리보기를 기본으로 제공한다.
- 수정 가능한 항목은 월관리 상품과 연결한다.

# 3. 메뉴별 UX 설계

## 3-1. 대시보드

### 목적

고객이 관리자에 들어왔을 때 현재 페이지 상태를 한눈에 확인하게 한다.

### 표시 항목

- 현재 게시 상태: 게시 중 / 비공개 / 수정본 있음
- 마지막 저장일
- 마지막 게시일
- 대표 문의 버튼 상태
- 필수 정보 누락 여부
- 페이지 바로가기
- 모바일 미리보기 버튼
- “수정 후 게시 필요” 알림

### 예시 화면 구성

```text
[현재 상태]
게시 중

[수정 필요]
- 이벤트 종료일이 지났습니다.
- 대표 이미지가 권장 비율보다 작습니다.

[빠른 수정]
기본정보 수정
가격표 수정
공지 추가
후기 추가
문의 버튼 수정

[페이지 확인]
모바일 미리보기
실제 페이지 열기
```

### 유효성 체크

- 필수 CTA 링크가 없는 경우 경고
- 전화번호 형식 오류 경고
- 사업장 주소 누락 경고
- 대표 이미지 누락 경고
- 공지 종료일이 지난 경우 표시

## 3-2. 기본정보

### 고객이 수정 가능한 필드

| 필드 | 설명 | 필수 여부 | 예시 |
|---|---|---:|---|
| 매장명 / 사업명 | 페이지 상단, 공유 제목 등에 사용 | 필수 | 홍대 라온네일 |
| 한 줄 소개 | 히어로 문구 또는 소개 영역 | 필수 | 손상 적은 젤네일 전문샵 |
| 짧은 설명 | 2~3문장 소개 | 선택 | 1:1 예약제로 꼼꼼하게 시술합니다. |
| 업종 카테고리 | 내부 분류용 | 필수 | 네일샵, 필라테스, 공방 |
| 운영 상태 | 정상 영업 / 임시 휴무 / 예약 마감 | 선택 | 정상 영업 |
| 운영시간 요약 | 페이지 표시용 | 필수 | 평일 11:00-20:00, 토요일 11:00-18:00 |
| 휴무일 | 페이지 표시용 | 선택 | 매주 일요일 |
| 대표 전화번호 | 전화 CTA에 사용 가능 | 선택 | 010-1234-5678 |
| 사업자명 | 푸터 표시용 | 선택 | 라온뷰티 |
| 사업자등록번호 | 필요 시 푸터 표시 | 선택 | 000-00-00000 |

### 유효성 검증

- 매장명: 2~30자
- 한 줄 소개: 10~45자 권장
- 짧은 설명: 300자 이하
- 전화번호: 숫자, 하이픈 허용
- 운영시간: 자유 입력 가능하되 100자 이하
- 사업자등록번호: `000-00-00000` 형식 권장, 필수는 아님

### 미리보기 방식

- 우측 또는 하단에 모바일 프레임 제공
- 수정 중인 내용이 히어로, 푸터, 연락처 영역에 어떻게 반영되는지 즉시 표시
- 저장 전에는 “미저장 변경사항” 배지 표시

### 저장 / 게시 흐름

```text
입력 수정 → 임시저장 → 미리보기 확인 → 게시하기
```

기본정보는 여러 섹션에 영향을 주기 때문에 저장 즉시 게시하지 않는 것이 좋다.

## 3-3. 대표 이미지

### 고객이 수정 가능한 필드

| 필드 | 설명 | 필수 여부 |
|---|---|---:|
| 히어로 대표 이미지 | 첫 화면 배경 또는 대표 사진 | 필수 |
| 모바일용 대표 이미지 | 세로형 이미지 별도 등록 | 선택 |
| 매장 내부 사진 | 신뢰 섹션 또는 갤러리용 | 선택 |
| 서비스 사진 | 서비스/가격 카드에 연결 | 선택 |
| 이미지 대체 텍스트 | SEO 및 접근성용 | 선택 |

### 이미지 권장 크기

| 이미지 유형 | 권장 크기 | 비율 | 용도 |
|---|---:|---:|---|
| 히어로 대표 이미지 | 1600×1000px 이상 | 16:10 또는 4:3 | 첫 화면 |
| 모바일 히어로 이미지 | 1080×1350px | 4:5 | 모바일 첫 화면 |
| 정사각형 썸네일 | 1000×1000px | 1:1 | 서비스 카드 |
| 갤러리 이미지 | 1200×900px | 4:3 | 사례, 매장 사진 |
| 공유 이미지 | 1200×630px | 1.91:1 | 카카오톡, SNS 공유 |

### 업로드 제한

- 파일 형식: JPG, PNG, WebP
- 최대 용량: 이미지당 5MB 권장
- 자동 압축: WebP 변환 권장
- 최대 등록 수: MVP 기준 10~20장

### 유효성 검증

- 너무 작은 이미지 업로드 시 경고
- 가로/세로 비율이 맞지 않으면 크롭 도구 제공
- 파일 용량 초과 시 압축 안내
- HEIC 파일은 업로드 전 변환 필요. 구현 가능하면 서버에서 변환

### 미리보기 방식

- 데스크톱 / 모바일 크롭 미리보기
- 어둡게 오버레이가 들어가는 히어로라면 실제 텍스트가 올라간 상태로 미리보기
- 이미지 변경 후 기존 이미지로 되돌리기 버튼 제공

### 주의할 UX

고객에게 “이미지를 아무거나 올리세요”라고 하면 첫 화면 품질이 무너질 수 있다.  
따라서 대표 이미지는 다음처럼 제한하는 것이 좋다.

```text
대표 이미지는 첫 화면에 크게 노출됩니다.
가로 사진 또는 밝은 매장 사진을 권장합니다.
문자가 많이 들어간 포스터 이미지는 권장하지 않습니다.
```

## 3-4. 서비스 / 가격

### 고객이 수정 가능한 필드

서비스는 반복 가능한 카드 구조가 적합하다.

| 필드 | 설명 | 필수 여부 |
|---|---|---:|
| 서비스명 | 상품명 또는 프로그램명 | 필수 |
| 짧은 설명 | 1~2줄 설명 | 선택 |
| 가격 표시 | 숫자 또는 문구 | 필수 |
| 가격 단위 | 원 / 회 / 월 / 상담 후 안내 | 선택 |
| 소요시간 | 시술, 수업, 상담 시간 | 선택 |
| 추천 배지 | 인기 / 신규 / 이벤트 / 대표 | 선택 |
| 노출 여부 | 페이지에 보일지 여부 | 필수 |
| 정렬 순서 | 카드 순서 | 필수 |
| 서비스 이미지 | 카드 이미지 | 선택 |
| CTA 연결 | 이 서비스 전용 문의 링크 | 선택 |

### 가격 입력 방식

고객이 가장 많이 실수하는 부분이 가격이다.  
자유 입력만 주면 `5~` 또는 `별도문의`처럼 제각각 입력될 수 있으므로 두 가지 모드를 제공한다.

#### 가격 모드

1. **정가 표시**
   - 숫자 입력: `50000`
   - 화면 표시: `50,000원`

2. **문구 표시**
   - 자유 문구: `상담 후 안내`, `월 12만원부터`, `체험권 1회 30,000원`

### 유효성 검증

- 서비스명: 2~30자
- 설명: 100자 이하
- 가격 숫자: 0 이상
- 가격 문구: 30자 이하
- 서비스 개수: MVP 기준 3~12개
- 추천 배지: 한 페이지에 1~3개까지만 권장
- 노출 서비스가 0개이면 게시 불가

### 미리보기 방식

- 가격 카드가 실제 모바일 폭에서 어떻게 보이는지 표시
- 긴 서비스명이 줄바꿈되는지 확인 가능
- “대표 상품” 배지가 붙은 상태 확인
- 카드 순서 드래그 앤 드롭

### 저장 / 게시 흐름

서비스/가격은 매출과 직결되므로 변경 이력을 남기는 것이 좋다.

```text
가격 수정 → 변경사항 저장 → 미리보기 → 게시하기 → 변경 이력 기록
```

변경 이력 예시:

```text
2026-06-08 14:20
젤네일 기본 가격 45,000원 → 50,000원
수정자: 고객
```

# 4. 저장 / 게시 흐름 설계

## 권장 흐름

```text
수정
↓
임시저장
↓
미리보기
↓
게시하기
↓
실제 페이지 반영
```

## 저장과 게시를 분리해야 하는 이유

- 고객이 작성 중인 미완성 내용을 바로 노출하지 않게 한다.
- 제작자가 월관리 중 검수할 수 있다.
- 실수로 가격이나 전화번호를 잘못 올리는 문제를 줄인다.
- SaaS화 시 버전 관리가 쉬워진다.

## 상태값

| 상태 | 의미 |
|---|---|
| draft | 수정 중 |
| published | 게시 중 |
| unpublished | 비공개 |
| scheduled | 예약 게시 |
| archived | 보관 |

## 게시 전 체크리스트

게시 버튼을 누르면 아래 항목을 자동 점검한다.

```text
필수 정보 확인
- 매장명 있음
- 대표 이미지 있음
- 서비스/가격 1개 이상 있음
- 대표 문의 버튼 있음
- 지도 링크 있음
- SEO 제목/설명 있음

주의 확인
- 종료일 지난 이벤트 있음
- 전화번호 형식 이상
- 너무 긴 버튼 문구 있음
- 이미지 권장 크기 미달
```

# 6. 고객이 건드리면 안 되는 항목

## 잠가야 할 항목

| 항목 | 잠그는 이유 |

| 섹션 순서 | 문의 전환 흐름이 깨짐 |
| 폰트 | 브랜드 일관성 및 가독성 유지 |
| 컬러 팔레트 | 디자인 품질 저하 방지 |

| 버튼 위치 | CTA 전환 구조 유지 |
| 여백 / 간격 | 화면 완성도 유지 |
| 반응형 브레이크포인트 | 모바일 레이아웃 붕괴 방지 |
| 코드 삽입 | 보안 위험 |
| 지도 임베드 코드 | 깨진 iframe, 추적 코드 삽입 위험 |
| SEO index 설정 | 검색 노출 실수 방지 |

| 템플릿 구조 | SaaS화 시 유지보수 어려움 |

# 9. 권한 설계

## 권장 권한 3단계

| 권한 | 대상 | 가능 작업 |
|---|---|---|
| 고객 관리자 | 소상공인 대표 | 내용 수정, 이미지 업로드, 게시 요청 |
| 제작자 관리자 | 본인 / 운영자 | 템플릿 수정, 디자인 설정, 게시 승인 |
| 슈퍼 관리자 | SaaS 운영자 | 결제, 도메인, 계정, 전체 설정 |

초기에는 고객 관리자와 제작자 관리자만 있어도 된다.

## 게시 승인 방식

상품 운영 방식에 따라 두 가지 모델을 선택할 수 있다.

### 1안: 고객 즉시 게시

- 고객이 수정 후 바로 게시 가능
- 운영 부담 적음
- 실수 가능성 높음

적합한 경우:

- 저가형 상품
- 텍스트/가격만 수정 가능
- 디자인 잠금이 강한 경우

### 2안: 고객 게시 요청 → 제작자 승인

- 고객이 수정 후 게시 요청
- 제작자가 확인 후 게시
- 월관리 상품과 연결하기 좋음

적합한 경우:

- 월관리 고객
- 의료, 교육, 금융성 표현 등 문구 리스크가 있는 업종
- 이벤트 배너나 SEO 문구까지 관리하는 상품

추천은 MVP에서는 **고객 즉시 게시 + 주요 위험 항목 잠금**이다.  
월관리 고객에게는 “게시 전 검수”를 부가 가치로 제공하면 된다.

# 10. 실제 상품화에 바로 쓸 수 있는 관리자 툴 범위

## 1차 MVP 범위

처음부터 완전한 SaaS를 만들 필요는 없다.

```text
- 최근 변경 이력
```

모바일 관리자까지 완벽하게 만들 필요는 낮다.  
소상공인 고객이 휴대폰으로 수정할 가능성은 높지만, MVP에서는 모바일에서도 폼이 깨지지 않는 수준이면 충분하다.

---

# 12. 결론

이 관리자 툴은 홈페이지 빌더가 아니라 **소상공인용 콘텐츠 관리 폼**으로 설계해야 한다.

가장 중요한 구분은 다음이다.

- 고객이 바꿔야 하는 것: 가격, 공지, 사진, 후기, 문의 링크, 운영시간
- 고객이 바꾸면 안 되는 것: 레이아웃, CTA 위치, 폰트, 색상, 반응형 구조, 코드, SEO 노출 설정

SaaS화를 고려한다면 처음부터 `Site`, `Profile`, `ServiceItem`, `Notice`, `Review`, `CTA`, `Location`, `SeoMeta`, `PublishVersion` 정도의 개념은 분리해두는 것이 좋다.  
구현은 JSON 기반으로 가볍게 시작하되, 데이터 이름과 구조는 나중에 DB로 옮기기 쉬운 형태로 잡는 것이 현실적이다.

# Round 5. 첫 번째 포트폴리오 샘플 제작 명세

# 첫 번째 실제 샘플 랜딩페이지 제작 명세

## 1. 추천 첫 샘플 업종: 1:1 필라테스/PT 스튜디오

첫 샘플은 **“동네 1:1 필라테스/PT 스튜디오 체험권 예약 랜딩페이지”**로 만드는 것을 추천한다.

이유는 다음과 같다.

- **구매 전 확인 정보가 많다.** 가격, 체험권, 강사, 시설, 위치, 운동 대상, 예약 방식이 한 페이지에 정리될수록 문의 전환에 유리하다.
- **사진과 신뢰 요소가 중요하다.** 센터 내부, 기구, 강사 프로필, 수업 장면, 후기 이미지를 넣기 좋아 포트폴리오 시각 품질을 보여주기 쉽다.
- **수정 가능 필드가 명확하다.** 체험권 가격, 이벤트 기간, 시간표, 강사, 후기, 공지, 상담 링크를 고객이 직접 바꾸는 구조가 잘 맞는다.
- **월관리 전환 명분이 있다.** 매월 체험권 이벤트, 시간표 변경, 신규 강사 추가, 후기 업데이트가 발생할 가능성이 높다.
- **크몽에서 설명하기 쉽다.** “인스타/네이버플레이스에 흩어진 정보를 체험권 예약 페이지로 정리해드립니다”라는 상품 메시지가 바로 통한다.

단, 샘플 카피에서는 “살 빠진다”, “통증 치료”, “교정 보장” 같은 표현은 피한다. 운동 효과는 개인차가 있으므로 **상담, 습관, 자세, 컨디션, 체험 수업** 중심으로 표현한다.

---

## 2. 샘플 콘셉트

샘플명:

**바디핏 스튜디오 - 1:1 필라테스 체험권 예약 페이지**

목표 행동:

- 1순위: 카카오톡 체험 상담
- 2순위: 네이버지도 확인
- 3순위: 전화 예약

페이지 톤:

- 깔끔하고 신뢰감 있는 피트니스/웰니스 분위기
- 과한 병원 느낌은 피하고, 따뜻하지만 전문적인 스튜디오 느낌
- 색상은 화이트, 차콜, 세이지 그린, 라이트 그레이 조합 추천
- 모바일 기준으로 섹션을 짧게 나누고, 사진과 정보가 번갈아 나오게 구성

관리자에서 직접 수정할 핵심 필드:

- 체험권 가격
- 이벤트 마감일
- 수업 가능 시간
- 강사 프로필
- 후기
- 시설 사진
- 공지
- 카카오톡/전화/지도 링크

---

## 3. 전체 와이어프레임

### 섹션 1. 상단 히어로

**목적**  
처음 들어온 사람이 “어떤 스튜디오인지, 지금 무엇을 신청할 수 있는지, 어디를 누르면 되는지”를 5초 안에 이해하게 만든다.

**화면 구성**

- 상단 작은 바: `바디핏 스튜디오 | 1:1 필라테스`
- 메인 문구
- 체험권 제안
- 대표 내부 사진 또는 수업 장면
- 핵심 배지 3개
- CTA 버튼 2개

**예시 카피**

```markdown
처음 운동을 시작하는 분을 위한
1:1 필라테스 체험 상담

내 몸 상태와 목표에 맞춰
수업 방향, 가능 시간, 체험권을 안내드립니다.

신규 체험권 1회 29,000원
6월 상담 신청 고객 한정
```

**배지 예시**

- 1:1 맞춤 상담
- 초보자 가능
- 예약제 운영

**이미지 자리**

- 모바일 첫 화면 기준 세로형 이미지 1장
- 추천 이미지: 밝은 스튜디오 내부, 리포머 기구, 강사가 자세를 봐주는 장면
- 피해야 할 이미지: 얼굴이 너무 크게 나온 사진, 어두운 헬스장 느낌, 과도한 보정 이미지

**CTA**

- 주요 버튼: `카카오톡으로 체험 상담`
- 보조 버튼: `위치 먼저 보기`

**수정 가능 필드**

- 스튜디오명
- 메인 헤드라인
- 서브 문구
- 체험권 가격
- 이벤트 문구
- 배지 3개
- 대표 이미지
- 카카오톡 링크
- 지도 링크

**디자인 의도**

첫 화면은 정보보다 “신뢰감 있는 첫인상”이 중요하다. 이미지는 넓게 쓰되 CTA가 접히지 않게 하고, 가격/혜택은 작은 배지가 아니라 본문 안에 명확히 넣는다.

---

### 섹션 2. 방문자 상황 체크

**목적**  
고객이 “내 얘기다”라고 느끼게 만드는 섹션이다. 필라테스를 처음 시작하는 사람의 망설임을 짚는다.

**예시 카피**

```markdown
이런 분께 체험 상담을 추천합니다

운동을 시작하고 싶은데 무엇부터 해야 할지 모르겠는 분
그룹 수업보다 내 속도에 맞는 수업이 필요한 분
오래 앉아 있어 몸이 자주 뻐근한 분
무리한 운동보다 꾸준한 루틴을 만들고 싶은 분
```

**이미지 자리**

- 아이콘 4개 또는 짧은 체크리스트형 카드
- 별도 사진 없이 텍스트 중심도 가능

**CTA**

- 섹션 하단 작은 텍스트 링크: `내 상황에 맞는 수업 상담하기`

**수정 가능 필드**

- 섹션 제목
- 체크 항목 4~6개
- 보조 CTA 문구

**디자인 의도**

이 섹션은 세일즈 느낌보다 상담 전 자기진단처럼 보여야 한다. 문장은 짧게, 체크 아이콘은 작게 사용한다.

---

### 섹션 3. 체험권/대표 상품

**목적**  
방문자가 가격을 보고 이탈하지 않도록 대표 상품만 단순하게 보여준다.

**예시 구성**

```markdown
신규 체험 상담 패키지

1:1 체험 수업
29,000원
상담 10분 + 체험 수업 40분
처음 방문하는 고객 전용

1:1 정규 수업
월 4회 / 8회 상담 가능
목표와 가능 시간에 맞춰 안내
방문 상담 후 등록 가능

2:1 듀엣 수업
친구, 가족과 함께 수업 가능
가능 시간 별도 문의
```

**가격 안내 문구**

```markdown
정확한 수업 구성과 비용은 상담 후 안내드립니다.
체험권은 신규 고객 1회만 이용 가능합니다.
```

**이미지 자리**

- 가격 카드 옆 또는 아래에 기구 사진 1장
- 가능하면 “체험권” 카드에는 추천 라벨 부착

**CTA**

- `체험권 가능 시간 문의`
- `정규 수업 상담`

**수정 가능 필드**

- 상품명
- 가격
- 설명
- 추천 상품 여부
- 안내 문구
- 이벤트 종료일

**디자인 의도**

가격표는 복잡하면 안 된다. 첫 샘플에서는 3개 카드만 보여주고, 자세한 요금표 전체 공개보다 “상담으로 연결되는 대표 가격”에 집중한다.

---

### 섹션 4. 수업 방식 소개

**목적**  
필라테스/PT가 낯선 고객에게 수업 과정을 보여줘 불안을 낮춘다.

**예시 카피**

```markdown
처음 방문해도 어렵지 않게 진행됩니다

1. 현재 몸 상태와 운동 경험을 확인합니다.
2. 목표에 맞춰 필요한 움직임을 가볍게 테스트합니다.
3. 리포머와 소도구를 활용해 기본 수업을 체험합니다.
4. 이후 가능한 시간과 수업 방향을 안내드립니다.
```

**이미지 자리**

- 4단계 중 2개에 작은 사진 삽입 가능
- 예: 상담 테이블, 리포머 기구, 소도구, 스트레칭 공간

**CTA**

- `처음 방문 상담 예약`

**수정 가능 필드**

- 단계 제목 4개
- 단계 설명
- 단계별 이미지 선택 여부

**디자인 의도**

고객이 “가면 뭘 하지?”를 알게 해주는 섹션이다. 이 부분은 전문성보다 예측 가능성이 핵심이다.

---

### 섹션 5. 강사 프로필

**목적**  
1:1 수업은 사람이 상품이므로 강사 신뢰를 보여준다.

**예시 카피**

```markdown
대표 강사 김서연

처음 운동을 시작하는 분도 부담 없이 따라올 수 있도록
몸 상태와 생활 패턴을 함께 확인하며 수업을 설계합니다.

- 필라테스 지도 경력 6년
- 1:1 개인 레슨 중심 수업
- 초보자, 직장인, 산전/산후 상담 가능
```

**주의 문구**

- 자격증, 경력, 수료 내용은 실제 고객 자료를 받아 정확히 넣어야 한다.
- 의료적 치료 표현은 사용하지 않는다.

**이미지 자리**

- 강사 프로필 사진 1장
- 수업 중 뒷모습/측면 사진 1장 선택 가능

**CTA**

- `강사에게 수업 상담하기`

**수정 가능 필드**

- 강사명
- 직함
- 소개 문구
- 경력 항목
- 자격/수료 항목
- 프로필 사진

**디자인 의도**

프로필은 이력서처럼 길게 쓰지 않는다. “이 강사가 나를 편하게 봐줄 것 같다”는 느낌이 중요하다.

---

### 섹션 6. 공간/시설 사진

**목적**  
방문 전 환경을 확인하게 해서 예약 장벽을 낮춘다.

**예시 카피**

```markdown
프라이빗하게 집중할 수 있는 예약제 스튜디오

1:1 수업에 맞춘 기구와 조용한 공간에서
처음 오시는 분도 편하게 상담받을 수 있습니다.
```

**이미지 자리**

- 사진 4장 그리드
- 추천 사진:
  - 입구
  - 리포머/캐딜락 기구
  - 탈의 공간
  - 상담 공간
- 모바일에서는 2열 그리드 또는 가로 스와이프

**CTA**

- `스튜디오 위치 보기`

**수정 가능 필드**

- 섹션 제목
- 소개 문구
- 사진 4~8장
- 사진 설명

**디자인 의도**

이 섹션은 포트폴리오에서 시각적으로 가장 강하게 보여줄 부분이다. 크몽 미리보기 이미지로도 활용하기 좋다.

---

### 섹션 7. 후기

**목적**  
처음 방문하는 고객의 불안을 낮춘다. 과장된 전후 변화보다 상담 경험, 친절함, 수업 만족을 중심으로 쓴다.

**예시 카피**

```markdown
“운동을 오래 쉬어서 걱정했는데, 첫날부터 무리하지 않게 봐주셔서 좋았어요.”

“가격과 수업 방향을 먼저 설명해주셔서 등록 여부를 편하게 결정할 수 있었습니다.”

“센터가 조용하고 예약제로 운영돼서 집중하기 좋았습니다.”
```

**이미지 자리**

- 네이버 리뷰 캡처형 이미지 자리 1~2개
- 텍스트 후기 카드 3개
- 실제 사용 시 개인정보 가림 필요

**CTA**

- `후기 보고 상담하기`

**수정 가능 필드**

- 후기 내용
- 작성자 표시명
- 별점
- 후기 이미지
- 네이버플레이스 리뷰 링크

**디자인 의도**

후기는 길게 나열하지 않는다. 첫 샘플에서는 짧은 카드 3개와 리뷰 링크 버튼이면 충분하다.

---

### 섹션 8. 운영시간/위치/주차

**목적**  
방문 가능 여부를 즉시 판단하게 만든다.

**예시 카피**

```markdown
오시는 길

서울시 마포구 ○○로 12, 3층
홍대입구역 2번 출구 도보 5분

운영시간
평일 10:00 - 21:00
토요일 10:00 - 17:00
일요일 휴무

주차
건물 뒤편 유료주차장 이용 가능
```

**이미지 자리**

- 지도 캡처 또는 지도 임베드 영역
- 건물 입구 사진 1장

**CTA**

- `네이버지도 열기`
- `전화로 문의하기`

**수정 가능 필드**

- 주소
- 가까운 역/랜드마크
- 운영시간
- 휴무일
- 주차 안내
- 지도 링크
- 전화번호

**디자인 의도**

지도 섹션은 디자인보다 정확성이 중요하다. 주소, 층수, 주차 안내가 빠지면 문의가 반복된다.

---

### 섹션 9. 자주 묻는 질문

**목적**  
반복 문의를 줄이고, 체험 신청 전 마지막 불안을 해소한다.

**예시 FAQ**

```markdown
Q. 운동을 처음 해도 가능한가요?
A. 가능합니다. 체험 상담에서 운동 경험과 현재 상태를 먼저 확인합니다.

Q. 체험 수업 후 바로 등록해야 하나요?
A. 아닙니다. 수업 방향과 가능 시간을 안내받은 뒤 결정하시면 됩니다.

Q. 준비물이 필요한가요?
A. 편한 운동복을 준비해주세요. 자세한 안내는 예약 확정 후 보내드립니다.

Q. 예약 변경이 가능한가요?
A. 예약 변경 기준은 스튜디오 운영 정책에 따라 안내드립니다.
```

**이미지 자리**

- 없음
- 아코디언 UI 추천

**CTA**

- `궁금한 점 카카오톡으로 묻기`

**수정 가능 필드**

- 질문
- 답변
- 노출 여부
- 순서 변경

**디자인 의도**

FAQ는 관리자 수정 기능을 보여주기 좋은 섹션이다. 크몽 설명에서 “자주 묻는 질문도 직접 수정 가능”이라고 말할 수 있다.

### 섹션 10. 최종 신청 영역

**목적**  
페이지를 끝까지 본 사람에게 다시 체험권 제안을 보여준다.

**예시 카피**

```markdown
이번 달 체험 상담 가능 시간을 확인해보세요

원하는 요일과 시간을 남겨주시면
가능한 수업 시간과 체험권 이용 방법을 안내드립니다.
```

**CTA**

- 주요 버튼: `카카오톡으로 체험권 예약`
- 보조 버튼: `전화 상담하기`

**수정 가능 필드**

- 최종 CTA 제목
- 설명 문구
- 버튼 문구
- 카카오톡 링크
- 전화번호
- 이벤트 문구

**디자인 의도**

마지막 섹션은 새 정보를 넣기보다 행동을 다시 정리한다. 버튼은 크게, 문장은 짧게 둔다.

## 4. 하단 고정 CTA 명세

모바일 하단에 고정되는 버튼 영역을 넣는다.

구성:

```markdown
[카카오톡 상담] [전화]
```

또는 더 단순하게:

```markdown
[체험권 상담하기]
```

추천은 첫 샘플에서는 2버튼 구조다. 크몽 미리보기에서 “모바일 하단 고정 문의 버튼”을 명확히 보여주기 좋다.

수정 가능 필드:

- 버튼 1 라벨
- 버튼 1 링크
- 버튼 2 라벨
- 버튼 2 링크
- 버튼 노출 여부

- 예약 변경/취소 안내 문구

- 가상 스튜디오명
- 가상 가격표
- 무료 이미지 또는 직접 생성한 스튜디오 이미지
- 가상 후기 문구
- 샘플 카카오톡 링크

주의할 점:

- 실제 업체처럼 오해될 수 있으므로 샘플에는 `데모 페이지입니다` 문구를 작게 넣는다.
- 후기, 경력, 자격증은 실제 판매 페이지에서 임의로 만들어주면 안 된다.
- 운동 효과 보장, 치료, 교정 표현은 피한다.

## 7. 크몽 상품 페이지 연결 포인트

### 상품 제목 예시

```markdown
소상공인 체험권 예약 랜딩페이지 제작 | 직접 수정 가능한 관리자 포함
```

또는

```markdown
필라테스/PT/공방/샵 전용 문의형 홍보페이지 제작해드립니다
```

### 썸네일 문구

```markdown
가격표·후기·사진·예약버튼
직접 수정 가능한 홍보페이지
```

### 미리보기 이미지 구성

크몽 상품 이미지 5장을 만든다면 다음 순서가 좋다.

1. **완성 페이지 전체 미리보기**  
   모바일 화면 3개를 나란히 배치: 첫 화면, 가격 섹션, 위치/CTA

2. **직접 수정 가능 화면**  
   관리자에서 체험권 가격, 공지, 사진, 후기 수정하는 장면

3. **문의 전환 구조 설명**  
   인스타 링크 → 랜딩페이지 → 카카오톡/전화/지도 연결 흐름

4. **업종별 적용 예시**  
   필라테스, PT, 네일샵, 공방, 학원으로 문구만 바꿔 적용 가능하다는 이미지

5. **제공 범위 안내**  
   섹션 수, 수정 가능 항목, CTA 연결, 모바일 최적화, 1개월 무료 관리

### 상품 설명 핵심 문구

```markdown
인스타그램, 네이버플레이스, 블로그에 흩어진 정보를
고객이 보기 쉬운 한 페이지로 정리해드립니다.

가격표, 공지, 사진, 후기, 문의 버튼은
제작 후 직접 수정할 수 있도록 구성합니다.
```

### 포함 항목

- 모바일 중심 랜딩페이지 1개
- 첫 화면/가격/후기/사진/위치/FAQ/문의 CTA 구성
- 카카오톡, 전화, 네이버지도, 폼 링크 연결
- 고객 직접 수정 가능한 기본 관리자
- 기본 카피 정리
- 샘플 기준 디자인 적용
- 제작 후 1개월 무료 관리

### 제외 항목

- 로고 제작
- 전문 촬영
- 상세 브랜딩 가이드
- 복잡한 예약 시스템 개발
- 결제 기능
- 회원가입 기능
- 광고 운영 대행

### 월관리 전환 문구

```markdown
제작 후 1개월 동안 가격 수정, 공지 변경, 후기 추가를 도와드립니다.
이후 매월 이벤트/사진/가격표 업데이트가 필요하신 경우 월관리로 전환할 수 있습니다.
```

## 8. 첫 샘플 제작 우선순위

가장 먼저 만들 화면은 다음 순서가 좋다.

1. 모바일 첫 화면
2. 체험권/가격 카드
3. 시설 사진 그리드
4. 강사 프로필
5. 후기
6. 위치/운영시간
7. 관리자 수정 화면

이 샘플의 핵심은 “예쁜 운동 페이지”가 아니라 **고객이 직접 수정할 수 있는 문의형 체험권 페이지**를 보여주는 것이다. 크몽에서는 완성 랜딩페이지보다도 `수정 가능 필드`와 `문의 연결 구조`를 함께 보여줘야 상품 차별점이 분명해진다.

## 2. 제작 중 체크리스트

### 페이지 구조

- [ ] 첫 화면에서 업종, 핵심 혜택, 문의 버튼이 바로 보이는가?

- [ ] 섹션 순서가 문의 전환 흐름을 따르는가?

권장 순서:

1. 첫 화면: 매장명, 핵심 제안, CTA
2. 고객이 겪는 불편 또는 니즈
3. 대표 서비스

## 6. 장기 기억해야 할 메모 10개

1. 이 사업은 “홈페이지 제작”보다 “문의가 들어오게 정리된 모바일 홍보페이지”로 팔아야 한다.

2. 한국 소상공인 시장에서는 카카오톡, 전화, 네이버지도, 폼 문의 연결이 전환의 핵심이다.

3. 첫 샘플은 범용성이 높은 업종 하나에 집중해야 한다. 네일샵, 필라테스, 피부관리실, 공방, 학원, 청소/이사 서비스가 우선 후보군이다.

4. 고객이 직접 수정할 수 있어야 하는 항목은 가격, 공지, 사진, 후기, 운영시간, CTA 링크다.

5. 관리자 툴은 기능이 많은 것보다 고객 언어로 단순해야 한다. 고객은 `히어로 타이틀`보다 `첫 화면 제목`을 이해한다.

6. 월관리 전환은 제작 품질만으로 생기지 않는다. 이벤트, 후기, 사진, 가격표처럼 매달 바뀔 이유가 페이지 구조 안에 있어야 한다.

7. 상품 범위는 반드시 제한해야 한다. 페이지 1개, 섹션 7~9개, 수정 1~2회, 무료 관리 1개월처럼 고정해야 납품이 흔들리지 않는다.

8. 샘플은 포트폴리오가 아니라 판매 도구다. 고객이 샘플을 보는 즉시 “내 매장 정보도 이렇게 정리되면 좋겠다”고 느껴야 한다.

9. 모바일 QA가 데스크톱 QA보다 먼저다. 소상공인 홍보페이지의 실제 사용자는 대부분 모바일에서 가격, 위치, 예약 버튼을 확인한다.

10. 목표 현금흐름 월 100~150만원은 제작비와 월관리비를 섞어 만들어야 한다. 단발 제작만으로는 매달 안정성이 낮고, 월관리만으로는 초기 고객 수가 부족하다.
