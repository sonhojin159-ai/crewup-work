## 핵심

## 데모 구조

왼쪽 입력 패널:

1. 첫 화면 질문 / Hero
2. 고객 고민 질문 / Problem
3. 해결 방식 질문 / Solution
4. 증거 질문 / Proof
5. 오퍼 질문 / Offer
6. 예약 행동 질문 / Action

오른쪽 출력:

1. 모바일 랜딩페이지 미리보기
2. 6개 블록으로 압축된 양식 구조 설명
3. 실시간 JSON 데이터 표시

## 중요한 제품 판단

- 고객에게는 Sabri 17단계라는 말을 기본 화면에서 과하게 보여주지 않는다.
- 기본 상품은 17단계를 `Hero`, `Problem`, `Solution`, `Proof`, `Offer`, `Action` 6개 블록으로 압축한다.
- 단, 고객이 원하면 17단계 전체를 바로 노출하기보다 추가 가능한 내용도 먼저 `추가 압축 블록`으로 묶어 보여준다.
- 추가 압축 블록 예시: `예약 망설임 압축 블록(5+6)`, `진행 방식 압축 블록(8+9)`, `신뢰 증거 압축 블록(10+11+12)`, `안심 오퍼 압축 블록(14+15)`, `예약 행동 압축 블록(16+17)`.
- 더 세밀한 고객 요청이 있을 때만 17단계 개별 섹션을 따로 추가할 수 있게 한다.
- 사장님에게는 쉬운 질문으로 받는다.
- 내부 JSON에는 `industry`, `template_goal`, `visual_style`, `base_blocks`, `optional_sales_blocks`, `optional_sales_steps`, `sales_story`로 정규화한다.
- 같은 JSON을 다른 스킨으로 렌더링할 수 있게 해야 한다.
- 최종 양식 관리 단위는 `업종 x 목적 x 스킨`이다.

예시:

```text
네일샵 x 예약 전환형 x 소프트 프리미엄
네일샵 x 첫 방문 이벤트형 x 포스터
청소업 x 문제해결형 x 전후사진
필라테스 x 체험권 전환형 x 프리미엄
```
