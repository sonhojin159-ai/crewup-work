# 사비 편집기 양식 클릭 오류 수정

날짜: 2026-06-12

## 문제

사용자 피드백: 최종 편집기에서 상단 양식 버튼을 눌러도 양식이 바뀌지 않는 것처럼 보임.

대상 파일:

- `/home/ubuntu/crewup-work/landing-samples/data-driven-landing-template/final-sabri-editor-site/index.html`

## 수정

상단 양식 버튼 클릭 로직을 더 견고하게 수정했다.

- 양식 버튼에 `type="button"` 추가.
- 양식 버튼에 직접 `onclick="setFormat(...)"` 추가해 개별 이벤트 리스너/이벤트 위임이 실패해도 버튼 자체에서 포맷 변경이 실행되게 함.
- 기존 개별 버튼 이벤트 리스너에 더해 `document.addEventListener('click', ...)` 기반 이벤트 위임 추가.
- `setFormat(format)` 함수를 만들어 포맷 변경과 미리보기 갱신을 한 곳에서 처리.
- `history.replaceState()`는 로컬 파일/일부 브라우저에서 클릭 핸들러를 중단시킬 수 있어, `update()` 이후 http/https에서만 try/catch로 실행하도록 변경.
- 캐시/이전 첨부 혼동을 피하기 위해 별도 파일 `final-sabri-editor-clickfix-v4.html`과 `final-sabri-editor-clickfix-v4.zip` 생성.

## 검증

브라우저에서 직접 검증:

- `poster`, `instagram`, `calendar`, `case`, `lookbook` 순서로 클릭 시 모두 정상 변경.
- `formatLabel`, active button, JSON `template.format`, 미리보기 `.page` class가 모두 일치.
- 브라우저 콘솔 오류 없음.

검증 결과 예시:

```json
[
  {"fmt":"poster","label":"포스터형","active":"poster","json":"poster","pageClass":"page poster"},
  {"fmt":"instagram","label":"인스타형","active":"instagram","json":"instagram","pageClass":"page instagram"},
  {"fmt":"calendar","label":"예약형","active":"calendar","json":"calendar","pageClass":"page calendar"},
  {"fmt":"case","label":"사례형","active":"case","json":"case","pageClass":"page case"},
  {"fmt":"lookbook","label":"룩북형","active":"lookbook","json":"lookbook","pageClass":"page lookbook"}
]
```
