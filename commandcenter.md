## 프로젝트 목표

**완전 자동화된** 파일 기반 멀티에이전트 협업 시스템 구축.
사람이 개입하지 않아도 Claude(기획) → Codex(코딩) → AntiGravity(검수) 파이프라인이 자동으로 순환한다.

## 핵심 아키텍처

```
[commandcenter.md 변경 감지]
        │
        ▼
   tracker.py (오케스트레이터)
        │
        ├─→ subprocess로 Codex CLI 자동 호출
        │       → Codex가 코딩 수행
        │       → CodexResult.py + CodexReport.md 작성
        │
        ├─→ [CodexReport.md 변경 감지]
        │       → subprocess로 AntiGravity CLI 자동 호출
        │       → AntiGravity가 검수 수행
        │       → AntiReport.md 작성
        │
        └─→ [AntiReport.md 변경 감지]
                → 팀장(Claude)에게 결과 알림
                → 승인/재작업 판단
```

### [지시 1] Codex에게 - tracker.py 오케스트레이터로 업그레이드

**담당:** Codex
**우선순위:** 최상
**상태:** 🔴 미착수 → 즉시 수행하라
**산출물:** tracker.py 전면 재작성

#### ⚠️ 긴급: tracker.py 한글 인코딩 깨짐 복구

현재 tracker.py의 한글이 전부 깨져 있다. 재작성 시 반드시 **UTF-8 인코딩**으로 저장하라.

#### 요구사항

tracker.py를 단순 알림이 아닌 **자동 실행 오케스트레이터**로 만들어라:

**상단 설정 (config):**
```python
CONFIG = {
    "work_dir": r"C:\Users\sonho\Desktop\MA",
    "codex_cmd": [
        "codex", "exec",
        "-C", r"C:\Users\sonho\Desktop\MA",
        "--skip-git-repo-check",
        "--sandbox", "workspace-write"
    ],
    "agy_cmd": [
        "agy", "--print", "--print-timeout", "10m"
    ],
    "codex_prompt": "commandcenter.md를 읽고 새로운 지시사항을 수행하라. 코드는 CodexResult.py에, 보고는 CodexReport.md에 작성하라.",
    "agy_prompt": "CodexReport.md와 CodexResult.py를 읽고 검수하라. 결과는 AntiReport.md에 작성하라."
}
```

**기능:**

1. **파일 감시 + 자동 CLI 호출**
   - commandcenter.md 변경 감지 → `codex exec ...` 자동 호출
   - CodexReport.md 변경 감지 → `agy --print ...` 자동 호출
   - AntiReport.md 변경 감지 → 상태를 "완료"로 전환 + 터미널 알림

2. **워크플로우 상태 관리**
   - 상태: 대기 → 코딩중 → 검수중 → 완료
   - `status.json` 실시간 저장
   - 내용: `{ "phase": "코딩중", "last_updated": "...", "last_file": "...", "current_task": "..." }`

3. **로그 기록**
   - 모든 상태 변화 + CLI 호출 결과를 `logs/` 폴더에 기록
   - 로그 파일명: `log_YYYYMMDD_HHMMSS.txt`
   - `logs/` 폴더가 없으면 자동 생성

4. **안전장치**
   - tracker.py 자체가 만든 파일 변경은 무시 (무한 루프 방지)
   - CLI 호출 실패 시 재시도 1회 후 에러 로그 기록
   - 동시 실행 방지 (이미 에이전트가 실행 중이면 대기)
   - 짧은 시간 내 중복 이벤트 무시 (debounce 2초)

**코딩 규칙:**
- Python 3.10+ / UTF-8 인코딩 필수
- 외부 라이브러리: watchdog만 (subprocess는 내장)
- 에러 핸들링 필수
- 한글 print 메시지 유지

**완료 후:** CodexReport.md에 보고
```
## 작업 보고
- 작업 내용: (요약)
- 변경 파일: (목록)
- 테스트 여부: (Y/N, 방법)
- 특이사항: (있으면 기재)
```

### ~~[지시 4] 폐기됨~~ → [지시 4-R] Codex에게 - 게임 프로토타입 구현 (팀전)

**담당:** Codex
**우선순위:** 최상
**상태:** 🔴 미착수 → 즉시 수행하라
**산출물:** `Game/prototype.py` (단일 파일, 콘솔 기반)
**참조:** `Game/게임기획서_최종.md` 를 반드시 먼저 읽고 전체 규칙을 숙지하라 (v2.0 팀전 버전)

#### 개요

「용사 vs 마왕: 시련의 모험」 콘솔 프로토타입을 구현하라.
- **유저 = 용사 팀 중 1명** (직접 조작, 나머지 용사는 AI)
- **마왕 = 전원 AI 봇** (자동 플레이)
- 콘솔 입출력 기반, Python 단일 파일

#### 구현 범위

**1. 게임 모드 선택**
- 시작 시 모드 선택: 2v2 / 3v3
- 2v2: 2라운드 × 시련 3개 (용사2명 vs 마왕2명)
- 3v3: 3라운드 × 시련 2개 (용사3명 vs 마왕3명)

**2. 라운드 배정**
- 양쪽 모두 라운드 배정 **랜덤** (게임 시작 시 결정)
- 유저가 몇 라운드에 배정되었는지 표시
- 양쪽 모두 팀원 간 **상의 불가** (AI는 독립 판단)

**3. 카드 시스템**
- 시련 카드: A, B, C, D × 2장 = 총 8장 (전체 카드풀)
- 용기 카드: a, b, c, d (매 라운드 시련 수만큼 선택, 중복 가능)
- 라운드마다 사용된 시련 카드는 카드풀에서 제거

**4. 동료 시스템**
- 게임 시작 시 용사 팀 동료 타입 선택: 특정 동료 / 만능 동료
- **특정 동료:** 매 라운드 해당 용사가 A/B/C/D 중 1명 선택
  - **팀 내 중복 불가**: 이전 라운드에서 팀원이 사용한 동료 재선택 불가
  - 해당 시련 등장 시 자동 정화 (발동 시 정체 공개)
- **만능 동료:**
  - R1: 슬롯 번호 지정 → 해당 슬롯 무조건 정화 + 나머지 용기 선택
  - 2v2 R2: 마왕 선택 후 → 용사 1장 밴 → 적중 시 마왕 재선택
  - 3v3 R2~R3: 한 라운드에서만 2장 밴 사용 가능 (해당 용사가 독립 결정), 미사용 라운드는 용기만

**5. AI 봇 (마왕 + 용사 팀원)**
- **마왕 AI:**
  - R1: 카드풀에서 랜덤 선택 (블라인드)
  - R2 이후: 이전 라운드 결과 기반 전략 (발동된 동료 시련 회피 70%)
  - 마왕별 독립 AI (서로의 전략 모름)
- **용사 팀원 AI:**
  - 유저가 아닌 용사는 AI가 대신 플레이
  - 동료: 사용 가능한 동료 중 랜덤 선택 (팀 내 중복 불가 준수)
  - 용기: 이전 라운드 결과 기반 간단한 전략 (발동된 시련 타입 우선 선택 60%)

**6. 라운드 진행 흐름**
```
매 라운드:
  1. 해당 라운드 마왕 AI가 시련 카드 배치 (비공개)
  2. 해당 라운드 용사가 동료 선택 + 용기 카드 선택
     - 유저 라운드: 직접 입력
     - AI 라운드: 자동 선택 후 결과만 표시
     - 만능 동료 밴 라운드인 경우: 마왕 선택 후 밴 처리
  3. 동시 공개 → 결과 출력
     - 동료 발동 여부, 각 시련별 정화/타락 판정, 누적 점수
  4. 카드풀 갱신
  5. 양쪽 모두 결과 확인 → 다음 라운드 담당자에게 전달
```

**7. 승패 판정 + 보상**
- 정화 >= 타락 → 용사 팀 승리 / 타락 > 정화 → 마왕 팀 승리
- 판돈 100 고정
- 각 플레이어 개인별 보상 계산 (기획서 §7 참조)

**8. UI (콘솔)**
```
===== 용사 vs 마왕: 시련의 모험 =====
모드를 선택하세요:
  1) 2 vs 2
  2) 3 vs 3
> 

동료 타입을 선택하세요:
  1) 특정 동료
  2) 만능 동료
>

라운드 배정 결과:
  R1: 용사A(AI) vs 마왕A(AI)
  R2: 용사B(당신) vs 마왕B(AI)

--- 라운드 1 (용사A vs 마왕A) ---
[AI 자동 진행]
  시련: B  C  A
  용사A 동료: C → 시련 C 발동! 자동 정화!
  용기: a  b  d
  슬롯 1: 시련 B vs 용기 b → ✓ 정화!
  슬롯 2: 시련 C → 동료 자동 정화!
  슬롯 3: 시련 A vs 용기 d → ✗ 타락...
정화: 2  타락: 1

--- 라운드 2 (당신의 차례!) ---
카드풀: A×1  B×1  C×1  D×2  (5장)
사용 가능 동료: [A] [B] [D]  (C는 팀원이 사용함)
동료를 선택하세요 > 
```

#### 코딩 규칙
- Python 3.10+, UTF-8 인코딩 (`# -*- coding: utf-8 -*-` 선언)
- 외부 라이브러리 없음 (표준 라이브러리만)
