멀티 에이전트 프로젝트 협업 가이드

1. 에이전트별 역할 (Roles)

에이전트 역할 주요 임무
지휘관 (Claude) PM / 설계 기획서 작성, 업무 배분, 최종 승인
메인 코더 (Codex) 개발 / 디자인 코드 생성, 이미지 프롬프트 작성
기술 검토 (Anti) QA / 리서칭 코드 버그 검수, 최신 기술 서칭

2. 표준 작업 흐름 (Standard Workflow)

[Step 1] 기휘 및 명령
Claude가 commandcenter.md 에 업무 스펙과 지시사항을 기록합니다.

[Step 2] 개발 진행
Codex가 지휘서를 읽고 CodexResult.py 에 코드를 작성합니다.

[Step 3] 기술 검수
Antigravity가 생성된 코드를 검토하고 안티리포트.txt 를 작성합니다.

[Step 4] 최종 승인
Claude가 리포트를 읽고 최종 통과 여부를 결정합니다. (통과 시 다음 단계로)

3. 관제탑 알림 규칙
• 파일이 저장되는 순간 관제탑(Python)이 터미널에 실시간 알림을 띄웁니다.

모든 에이전트는 본인의 차례가 오면 공유된 텍스트 파일들을 최신 상태로 읽어야 합니다.

4. 주의사항
- Claude는 직접 코드를 길게 짜지 않고 지휘에 집중한다.
- 모든 진행 상황은 반드시 지정된 텍스트 파일에 기록한다.
